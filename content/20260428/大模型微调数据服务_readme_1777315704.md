## Overview

**SoloAI 大模型微调数据服务** 是一个开源项目模板与服务说明文档，面向需要构建或优化大模型训练数据管线的团队。  
项目强调**数据质量、自动化处理、结构化交付、隐私合规与可扩展性**，适用于通用大模型、行业模型及垂直场景 AI 应用的数据准备工作。

---

## Features

- **多源数据接入**  
  支持文本、问答、客服对话、知识库文档、网页内容、表格及半结构化数据的统一导入与转换。

- **数据清洗与标准化**  
  提供去重、脱敏、格式归一化、脏数据过滤、分段切片、标签规范化等能力，提升训练数据可用性。

- **面向微调的样本构建**  
  支持 SFT 指令数据、对话数据、多轮问答、偏好对数据、分类与抽取样本等格式化生成。

- **数据质检与评估机制**  
  包含规则校验、抽样复核、一致性检查、标签质量分析与可追踪审计流程，降低低质量数据风险。

- **可扩展的数据交付格式**  
  支持 JSONL、CSV、Parquet 等常见训练格式，便于接入 Hugging Face、DeepSpeed、LLaMA Factory 等训练框架。

- **企业级服务支持**  
  可根据行业需求定制数据 schema、标注规范、交付标准与 SLA，适配金融、医疗、教育、电商、制造等场景。

## Quick Start

以下示例展示如何将原始客服对话转换为适用于大模型监督微调（SFT）的 JSONL 数据。

### 1. Prepare Raw Data

创建一个原始数据文件 `data/raw/customer_support.json`：

```json
[
  {
    "instruction": "用户咨询订单物流状态",
    "input": "订单号 123456，什么时候发货？",
    "output": "您的订单已进入仓库处理阶段，预计 24 小时内发货。"
  },
  {
    "instruction": "用户申请退款",
    "input": "商品有破损，我想退款。",
    "output": "非常抱歉给您带来不便，请提供订单号和破损照片，我们将优先为您处理退款申请。"
  }
]
```

### 2. Run Data Conversion

```bash
python scripts/prepare_sft.py \
  --input data/raw/customer_support.json \
  --output data/processed/sft_train.jsonl
```

### 3. Example Output

```json
{"messages":[{"role":"system","content":"你是专业客服助手。"},{"role":"user","content":"用户咨询订单物流状态\n订单号 123456，什么时候发货？"},{"role":"assistant","content":"您的订单已进入仓库处理阶段，预计 24 小时内发货。"}]}
{"messages":[{"role":"system","content":"你是专业客服助手。"},{"role":"user","content":"用户申请退款\n商品有破损，我想退款。"},{"role":"assistant","content":"非常抱歉给您带来不便，请提供订单号和破损照片，我们将优先为您处理退款申请。"}]}
```

## Usage Examples

### 1. Data Cleaning

```bash
python scripts/clean_data.py \
  --input data/raw/source.jsonl \
  --output data/clean/cleaned.jsonl \
  --dedup \
  --normalize \
  --remove-empty \
  --mask-pii
```

### 2. Build Instruction Tuning Dataset

```bash
python scripts/build_instruction_dataset.py \
  --input data/clean/cleaned.jsonl \
  --output data/processed/instruction_data.jsonl \
  --template chat
```

### 3. Generate Preference Pairs for DPO/RLHF

```bash
python scripts/build_preference_pairs.py \
  --input data/raw/preferences.jsonl \
  --output data/processed/dpo_pairs.jsonl
```

示例输出：

```json
{
  "prompt": "请总结以下合同条款重点。",
  "chosen": "该合同重点包括付款周期、违约责任、知识产权归属和保密条款。",
  "rejected": "这是一个合同。"
}
```

### 4. Python API Example

```python
from pathlib import Path
import json

def convert_to_sft(records, system_prompt="你是专业领域助手。"):
    output = []
    for item in records:
        output.append({
            "messages": [
                {"role": "system", "content": system_prompt},
                {
                    "role": "user",
                    "content": f"{item['instruction']}\n{item['input']}".strip()
                },
                {"role": "assistant", "content": item["output"]}
            ]
        })
    return output

raw_records = [
    {
        "instruction": "提取发票关键信息",
        "input": "发票号码: 10001, 金额: 2300元, 日期: 2024-01-08",
        "output": "发票号码为10001，金额为2300元，日期为2024-01-08。"
    }
]

dataset = convert_to_sft(raw_records)
Path("data/processed").mkdir(parents=True, exist_ok=True)

with open("data/processed/invoice_sft.jsonl", "w", encoding="utf-8") as f:
    for row in dataset:
        f.write(json.dumps(row, ensure_ascii=False) + "\n")
```
