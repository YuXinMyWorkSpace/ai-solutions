## Overview

**SoloAI Data Service for LLM Fine-Tuning** is designed for AI teams, data engineers, and model developers who need reliable, scalable, and auditable data pipelines for instruction tuning, supervised fine-tuning (SFT), preference data preparation, and domain-specific dataset delivery.

This project is optimized for:
- LLM fine-tuning dataset processing
- Instruction / dialogue data formatting
- Data quality validation and filtering
- Enterprise AI data workflow automation
- Scalable preprocessing for domain adaptation

---

## Features

- **Multi-format dataset ingestion**  
  Import JSON, JSONL, CSV, Parquet, and custom structured data for LLM training pipelines.

- **Data cleaning and normalization**  
  Standardize prompts, responses, roles, metadata, and schema fields across heterogeneous sources.

- **Quality validation and filtering**  
  Detect empty samples, duplicated records, malformed conversations, token-length outliers, and schema violations.

- **Fine-tuning dataset transformation**  
  Convert raw data into common training formats for SFT, instruction tuning, chat tuning, and alignment workflows.

- **Pipeline automation and reproducibility**  
  Run repeatable preprocessing jobs with config-driven workflows suitable for CI/CD and MLOps environments.

- **Traceability and auditability**  
  Track source files, transformation steps, validation results, and dataset versions for enterprise-grade governance.

## Quick Start

Prepare a raw dataset and transform it into a fine-tuning-ready JSONL file.

### Example input: `data/raw/sample.jsonl`

```json
{"instruction":"解释什么是向量数据库","input":"","output":"向量数据库用于存储和检索高维向量，常用于语义搜索、推荐和RAG系统。"}
{"instruction":"将下面句子翻译成英文","input":"人工智能正在改变软件开发。","output":"Artificial intelligence is changing software development."}
```

### Run preprocessing

```bash
soloai-data preprocess \
  --input data/raw/sample.jsonl \
  --output data/processed/train.jsonl \
  --task sft \
  --schema instruction
```

### Example output: `data/processed/train.jsonl`

```json
{"messages":[{"role":"user","content":"解释什么是向量数据库"},{"role":"assistant","content":"向量数据库用于存储和检索高维向量，常用于语义搜索、推荐和RAG系统。"}],"meta":{"task":"sft"}}
{"messages":[{"role":"user","content":"将下面句子翻译成英文\n\n人工智能正在改变软件开发。"},{"role":"assistant","content":"Artificial intelligence is changing software development."}],"meta":{"task":"sft"}}
```

## Usage

### 1. Validate dataset quality

Use the validator to identify invalid rows, missing values, schema mismatches, or duplicate records.

```bash
soloai-data validate \
  --input data/raw/sample.jsonl \
  --schema instruction \
  --report reports/validation.json
```

Example validation report:

```json
{
  "total_records": 10000,
  "valid_records": 9821,
  "invalid_records": 179,
  "duplicate_records": 43,
  "empty_output_records": 27
}
```

---

### 2. Clean and normalize data

Normalize fields, trim whitespace, remove broken samples, and unify conversation format.

```bash
soloai-data clean \
  --input data/raw/sample.jsonl \
  --output data/clean/sample.cleaned.jsonl \
  --drop-empty \
  --deduplicate \
  --normalize-unicode
```

---

### 3. Convert to chat-style fine-tuning format

Transform instruction-style records into a chat dataset suitable for modern LLM training.

```bash
soloai-data transform \
  --input data/clean/sample.cleaned.jsonl \
  --output data/processed/chat_train.jsonl \
  --from-schema instruction \
  --to-schema chat
```

Example transformed record:

```json
{
  "messages": [
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user", "content": "解释什么是向量数据库"},
    {"role": "assistant", "content": "向量数据库用于存储和检索高维向量，常用于语义搜索、推荐和RAG系统。"}
  ]
}
```

---

### 4. Run with YAML configuration

Create a config file for reproducible data workflows.

#### `config/pipeline.yaml`

```yaml
input: data/raw/domain_dataset.jsonl
output: data/processed/domain_train.jsonl
task: sft
schema: instruction
clean:
  drop_empty: true
  deduplicate: true
  normalize_unicode: true
validate:
  enabled: true
  report: reports/domain_validation.json
transform:
  to_schema: chat
```

Run:

```bash
soloai-data run --config config/pipeline.yaml
```

---

### 5. Python API example

Use the project as a library inside your existing AI data pipeline.

```python
from soloai_data_service.pipeline import DataPipeline

pipeline = DataPipeline(
    input_path="data/raw/sample.jsonl",
    output_path="data/processed/train.jsonl",
    task="sft",
    schema="instruction"
)

result = pipeline.run()

print(result.summary())
```
