# SoloAI 大模型微调数据服务

![Build Status](https://img.shields.io/badge/build-passing-brightgreen)
![License](https://img.shields.io/badge/license-MIT-blue)

**SoloAI 大模型微调数据服务** 是一个面向企业与研发团队的开源数据服务项目，专注于为大语言模型（LLM）微调提供高质量、可扩展、可审计的数据处理能力。  
本项目覆盖从原始数据清洗、指令构造、质量评估到训练集导出的完整流程，帮助团队更高效地构建可用于 SFT、DPO、分类、问答与对话优化的数据资产。

---

## 项目简介

在大模型微调场景中，数据质量往往直接决定模型效果。SoloAI 提供一套标准化、工程化的数据服务能力，支持多源数据接入、自动化清洗、结构化标注、数据质检与格式导出，适用于企业级 LLM 微调数据生产与管理。

---

## 核心特性

- **多源数据接入**：支持 JSON、JSONL、CSV、Excel、API 拉取等多种数据输入方式
- **数据清洗与标准化**：自动去重、脱敏、格式规范化、低质量样本过滤
- **指令微调数据构建**：支持生成 `instruction / input / output`、`messages` 等主流微调格式
- **质量评估与审核**：支持规则校验、样本评分、人工复核与可追踪审计
- **多任务数据导出**：适配 SFT、对话、分类、问答、偏好优化等训练任务
- **工程化可扩展**：模块化架构设计，便于集成到企业现有 AI 数据流水线

---

## 适用场景

- 大语言模型指令微调（Instruction Tuning）
- 企业知识问答数据构建
- 客服/销售/运营对话数据整理
- 行业语料标准化与高质量训练集生产
- 偏好数据、评估集与监督微调数据服务

## 快速开始

以下示例展示如何将原始问答数据转换为适用于大模型监督微调（SFT）的训练数据格式。

### 示例输入

`data/raw/sample.jsonl`

```json
{"question": "什么是大模型微调？", "answer": "大模型微调是指在预训练模型基础上，使用特定任务数据继续训练，以提升模型在目标场景下的表现。"}
{"question": "如何提升训练数据质量？", "answer": "可以通过清洗、去重、标注规范、人工审核和自动质检等方式提升数据质量。"}
```

### 运行处理流程

```bash
python -m soloai.pipeline \
  --input data/raw/sample.jsonl \
  --output data/processed/sft.jsonl \
  --task sft
```

### 示例输出

`data/processed/sft.jsonl`

```json
{"instruction": "请回答以下问题", "input": "什么是大模型微调？", "output": "大模型微调是指在预训练模型基础上，使用特定任务数据继续训练，以提升模型在目标场景下的表现。"}
{"instruction": "请回答以下问题", "input": "如何提升训练数据质量？", "output": "可以通过清洗、去重、标注规范、人工审核和自动质检等方式提升数据质量。"}
```

## 使用示例

### 1. 数据清洗

```bash
python -m soloai.cleaner \
  --input data/raw/source.jsonl \
  --output data/clean/cleaned.jsonl \
  --dedup \
  --normalize \
  --mask-sensitive
```

### 2. 转换为对话格式

```bash
python -m soloai.converter \
  --input data/clean/cleaned.jsonl \
  --output data/processed/chat_messages.jsonl \
  --format chat
```

输出示例：

```json
{
  "messages": [
    {"role": "user", "content": "请解释什么是监督微调。"},
    {"role": "assistant", "content": "监督微调是利用标注数据对预训练模型进行进一步训练，使其更好地完成特定任务。"}
  ]
}
```

### 3. 生成分类任务数据

```bash
python -m soloai.builder \
  --input data/raw/tickets.csv \
  --output data/processed/classification.jsonl \
  --task classification
```

输出示例：

```json
{"text": "用户反馈 App 登录失败并提示验证码错误", "label": "登录问题"}
```

### 4. Python API 调用

```python
from soloai.pipeline import DataPipeline

pipeline = DataPipeline(
    input_path="data/raw/sample.jsonl",
    output_path="data/processed/output.jsonl",
    task="sft"
)

pipeline.run()
print("Data processing completed.")
```

## 最佳实践

- 在微调前优先进行 **去重、脱敏、格式统一**
- 通过抽样审核确保 **指令一致性** 与 **回答质量**
- 对不同任务类型分别维护训练集、验证集与测试集
- 使用统一 schema，确保数据能平滑接入训练框架
- 建议记录每批数据版本，支持追踪与回滚
