# SoloAI RAG 检索增强生成

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/rag-retrieval-augmented-generation/ci.yml?branch=main)](https://github.com/SoloAI/rag-retrieval-augmented-generation/actions)
[![License](https://img.shields.io/github/license/SoloAI/rag-retrieval-augmented-generation)](./LICENSE)

面向生产环境的 **RAG（Retrieval-Augmented Generation，检索增强生成）** 开源项目，帮助开发者快速构建高质量、可扩展、可观测的 AI 知识问答系统。  
该项目聚焦文档检索、向量索引、上下文增强和大模型生成流程，适用于企业知识库、智能客服、内部搜索和垂直领域问答场景。

---

## Overview

**SoloAI RAG 检索增强生成** 提供从数据摄取、文本切分、向量化、检索、重排到生成回答的完整工作流。项目设计强调模块化、可扩展性与工程可维护性，适合构建现代 AI 应用、LLM 知识库系统和企业级检索增强问答平台。

### 适用场景
- 企业知识库问答
- 私有文档检索与智能搜索
- AI 客服与 FAQ 自动应答
- 垂直行业专家助手
- 多文档语义检索与内容增强生成

---

## Features

- **端到端 RAG Pipeline**：支持文档加载、切分、Embedding、向量存储、召回、重排和生成。
- **模块化架构**：可自由替换 LLM、Embedding 模型、Vector Store 与 Retriever 组件。
- **多数据源接入**：支持 Markdown、PDF、TXT、HTML、JSON 等常见知识源格式。
- **高质量语义检索**：支持 Top-K 召回、相似度搜索、混合检索和上下文拼接。
- **生产可观测性**：便于接入日志、Tracing、评估指标和性能监控。
- **面向企业落地**：适合私有化部署、行业知识库构建和 AI 应用二次开发。

## Installation

### Requirements

- Python 3.10+
- `pip` or `poetry`
- 可选：OpenAI / compatible LLM API key
- 可选：向量数据库，如 FAISS、Chroma、Milvus、Weaviate

### Install via pip

```bash
git clone https://github.com/SoloAI/rag-retrieval-augmented-generation.git
cd rag-retrieval-augmented-generation
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -U pip
pip install -r requirements.txt
```

### Environment Variables

创建 `.env` 文件：

```bash
OPENAI_API_KEY=your_api_key
EMBEDDING_MODEL=text-embedding-3-large
CHAT_MODEL=gpt-4o-mini
VECTOR_STORE=faiss
```

### Optional: Install with Poetry

```bash
poetry install
poetry shell
```

## Quick Start

下面示例展示如何快速建立一个基础的 RAG 检索增强生成问答流程：

```python
from soloai_rag import RAGPipeline
from soloai_rag.loaders import DirectoryLoader
from soloai_rag.embeddings import OpenAIEmbeddings
from soloai_rag.vectorstores import FAISSStore
from soloai_rag.llms import ChatModel

# 1. 加载文档
documents = DirectoryLoader("./data").load()

# 2. 初始化组件
embeddings = OpenAIEmbeddings(model="text-embedding-3-large")
vectorstore = FAISSStore.from_documents(documents, embeddings)
llm = ChatModel(model="gpt-4o-mini")

# 3. 创建 RAG Pipeline
pipeline = RAGPipeline(
    vectorstore=vectorstore,
    llm=llm,
    top_k=4
)

# 4. 提问
response = pipeline.query("什么是 RAG 检索增强生成？它适合哪些业务场景？")

print(response.answer)
print(response.sources)
```

运行：

```bash
python examples/quickstart.py
```

## Usage Examples

### 1. Build an Index from Local Documents

```python
from soloai_rag.loaders import DirectoryLoader
from soloai_rag.splitters import RecursiveTextSplitter
from soloai_rag.embeddings import OpenAIEmbeddings
from soloai_rag.vectorstores import FAISSStore

docs = DirectoryLoader("./knowledge_base").load()

splitter = RecursiveTextSplitter(
    chunk_size=800,
    chunk_overlap=120
)
chunks = splitter.split_documents(docs)

embeddings = OpenAIEmbeddings(model="text-embedding-3-large")
store = FAISSStore.from_documents(chunks, embeddings)

store.save_local("./storage/faiss_index")
```

### 2. Query with Source Attribution

```python
from soloai_rag import RAGPipeline
from soloai_rag.vectorstores import FAISSStore
from soloai_rag.embeddings import OpenAIEmbeddings
from soloai_rag.llms import ChatModel

embeddings = OpenAIEmbeddings(model="text-embedding-3-large")
store = FAISSStore.load_local("./storage/faiss_index", embeddings)
llm = ChatModel(model="gpt-4o-mini")

rag = RAGPipeline(
    vectorstore=store,
    llm=llm,
    top_k=5,
    return_sources=True
)

result = rag.query("请总结该知识库中的核心技术架构。")

print("Answer:")
print(result.answer)
print("\nSources:")
for source in result.sources:
    print(f"- {source}")
```

### 3. Advanced Configuration

```python
from soloai_rag import RAGPipeline
from soloai_rag.retrievers import HybridRetriever
from soloai_rag.rerankers import CrossEncoderReranker
from soloai_rag.llms import ChatModel

retriever = HybridRetriever(
    vector_weight=0.7,
    keyword_weight=0.3,
    top_k=10
)

reranker = CrossEncoderReranker(
    model_name="bge-reranker-large"
)

llm = ChatModel(model="gpt-4o-mini")

rag = RAGPipeline(
    retriever=retriever,
    reranker=reranker,
    llm=llm,
    top_k=5,
    temperature=0.2
)

response = rag.query("如何提升企业知识库问答的准确率与可解释性？")
print(response.answer)
```

## Best Practices

为获得更好的 RAG 检索增强生成效果，建议：

1. **优化文本切分策略**：根据文档结构设置合理的 `chunk_size` 与 `chunk_overlap`。  
2. **选择合适的 Embedding 模型**：针对中文、行业术语和长文档场景进行评估。  
3. **增加重排步骤**：使用 reranker 提升召回结果相关性。  
4. **保留来源信息**：提高答案可追溯性与业务可审计性。  
5. **建立评估机制**：使用准确率、召回率、上下文覆盖率和响应延迟指标持续优化。
