## Installation

### Prerequisites

- Python 3.10+
- CUDA 11.8+ recommended for GPU acceleration
- Docker optional for containerized deployment
- Linux recommended for production environments

### Option 1: Install from source

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
python -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
```

### Option 2: Install with editable mode for development

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
pip install -e .
```

### Option 3: Docker

```bash
docker build -t soloai-llm-opt .
docker run --gpus all -p 8000:8000 soloai-llm-opt
```

## Quick Start

Below is a simple example for launching an optimized LLM inference server.

```bash
python -m soloai_llm_opt.serve \
  --model /models/your-llm \
  --host 0.0.0.0 \
  --port 8000 \
  --device cuda \
  --precision fp16 \
  --max-batch-size 16
```

Test the API:

```bash
curl -X POST "http://127.0.0.1:8000/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "your-llm",
    "messages": [
      {"role": "user", "content": "Explain how to reduce LLM inference latency."}
    ],
    "temperature": 0.2
  }'
```

## Usage Examples

### 1. Enable quantized inference

Use lower-precision or quantized weights to reduce GPU memory usage and improve cost efficiency.

```bash
python -m soloai_llm_opt.serve \
  --model /models/your-llm \
  --quantization int8 \
  --device cuda \
  --port 8000
```

### 2. Optimize throughput with dynamic batching

Dynamic batching can improve GPU utilization under concurrent traffic.

```bash
python -m soloai_llm_opt.serve \
  --model /models/your-llm \
  --enable-dynamic-batching \
  --max-batch-size 32 \
  --batch-timeout-ms 20
```

### 3. Kubernetes deployment example

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: soloai-llm-opt
spec:
  replicas: 1
  selector:
    matchLabels:
      app: soloai-llm-opt
  template:
    metadata:
      labels:
        app: soloai-llm-opt
    spec:
      containers:
        - name: llm-server
          image: soloai-llm-opt:latest
          ports:
            - containerPort: 8000
          resources:
            limits:
              nvidia.com/gpu: 1
```

### 4. Python client example

```python
import requests

url = "http://127.0.0.1:8000/v1/chat/completions"
payload = {
    "model": "your-llm",
    "messages": [
        {"role": "system", "content": "You are a performance optimization assistant."},
        {"role": "user", "content": "Give me 3 ways to improve LLM deployment efficiency."}
    ],
    "temperature": 0.3
}

response = requests.post(url, json=payload, timeout=60)
print(response.json())
```
