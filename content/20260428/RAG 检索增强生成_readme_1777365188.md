**SoloAI RAG 检索增强生成** is an open-source framework for building high-accuracy, production-ready **Retrieval-Augmented Generation (RAG)** systems. It helps developers connect large language models with private or domain-specific knowledge bases to improve factuality, traceability, and response quality.

---

## Overview

RAG（检索增强生成） combines **semantic retrieval**, **document chunking**, **vector indexing**, and **LLM generation** to deliver grounded answers based on your own data. This project is designed for teams building AI assistants, enterprise knowledge search, document QA, and industry-specific copilots.

## Features

- **Modular RAG pipeline**: ingestion, chunking, embedding, indexing, retrieval, reranking, and generation
- **Multi-source knowledge ingestion**: support for Markdown, PDF, TXT, HTML, and structured data
- **Vector database integration**: plug into popular vector stores for scalable semantic search
- **Prompt orchestration**: optimize answer quality with configurable templates and retrieval context control
- **Source-grounded responses**: return citations or matched passages for better explainability
- **Production-friendly architecture**: environment-based configuration, API integration, and extensible components

## Installation

- Python 3.10+

## Usage

### Python API example

```python
from soloai_rag import RAGPipeline, Settings

settings = Settings(
    chat_model="gpt-4o-mini",
    embedding_model="text-embedding-3-large",
    vector_db="faiss",
    index_dir="./index"
)

rag = RAGPipeline(settings=settings)

response = rag.ask(
    question="如何构建企业内部知识库问答系统？",
    top_k=5,
    return_sources=True
)

print(response.answer)
for source in response.sources:
    print(source)
```

### Build an index programmatically

```python
from soloai_rag import IndexBuilder

builder = IndexBuilder(
    input_dir="./data",
    index_dir="./index",
    chunk_size=500,
    chunk_overlap=50
)

builder.build()
```

### REST API example

Start the API service:

Query the API:
