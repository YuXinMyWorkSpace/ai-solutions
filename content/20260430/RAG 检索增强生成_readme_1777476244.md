# 🧠 SoloRAG: Enterprise-Grade Retrieval-Augmented Generation Framework
![Build Status](https://img.shields.io/badge/build-passing-brightgreen) ![License](https://img.shields.io/badge/License-Apache%202.0-blue.svg) ![Python](https://img.shields.io/badge/Python-3.9%2B-blue) ![Version](https://img.shields.io/badge/version-1.0.0-orange)

SoloRAG is a high-performance, modular framework engineered to streamline Retrieval-Augmented Generation (RAG) pipelines for production AI applications. It unifies advanced vector retrieval, context-aware document chunking, and LLM orchestration to deliver accurate, low-latency, and hallucination-resistant generative responses at scale.

## ✨ Key Features
- **🔍 Hybrid Retrieval Engine**: Combines dense embeddings, sparse BM25, and metadata filtering with dynamic re-ranking for precision recall.
- **📐 Adaptive Chunking & Routing**: Semantic-aware document splitting, cross-window context preservation, and query-type routing (factual, analytical, procedural).
- **🔄 Multi-LLM Orchestration**: Seamless provider switching (OpenAI, Anthropic, vLLM, Ollama) with automatic fallback, token budgeting, and streaming support.
- **📊 Built-in Evaluation & Observability**: Native metrics for faithfulness, answer relevance, and context recall, plus OpenTelemetry-compatible tracing.
- **🛡️ Enterprise Security**: Role-based access control, data isolation, PII redaction, and on-prem deployment readiness.
- **🔌 Plug-and-Play Adapters**: Drop-in connectors for ChromaDB, Milvus, Qdrant, Weaviate, Pinecone, and custom embedding models.

## 📦 Installation
Requires Python 3.9+. Install via pip:
```bash
pip install solo-rag

# Optional: Install with specific vector DB & LLM provider support
pip install solo-rag[milvus,openai,anthropic]
```
> 💡 **Tip**: For GPU-accelerated embedding inference, install `solo-rag[gpu]` and ensure CUDA 11.8+ drivers are present.

## 🚀 Quick Start
Initialize a production-ready RAG pipeline in under 10 lines:
```python
from solo_rag import RAGPipeline, VectorStore, LLMAdapter

# 1. Configure pipeline components
rag = RAGPipeline(
    vector_store=VectorStore(provider="chromadb", collection="company_knowledge"),
    llm=LLMAdapter(provider="openai", model="gpt-4o-mini"),
    embedding_model="text-embedding-3-small",
    reranker="cross-encoder/ms-marco-MiniLM-L-6-v2"
)

# 2. Ingest & index documents
rag.ingest(["./docs/product_manual.pdf", "./docs/api_reference.md"])

# 3. Query with source attribution
response = rag.query("How do I configure the retry policy for failed API calls?")
print(f"✅ Answer: {response.answer}")
print(f"📚 Sources: {[src.metadata['file'] for src in response.sources]}")
```

## 🛠️ Usage Examples

### 🔹 Metadata-Filtered Retrieval
```python
results = rag.query(
    prompt="Explain the authentication flow",
    filters={"department": "engineering", "doc_version": ">=2.5.0"},
    top_k=5,
    strategy="hybrid"
)
```

### 🔹 Streaming & Real-Time Context Tracing
```python
for chunk in rag.query_stream("Summarize the deployment architecture", stream=True):
    print(chunk, end="", flush=True)

# Access trace ID for observability dashboards
print(f"\n🔍 Trace ID: {results.trace_id}")
```

### 🔹 Automated Evaluation Pipeline
```python
from solo_rag.eval import RAGEvaluator

evaluator = RAGEvaluator(pipeline=rag)
report = evaluator.run(
    dataset="test_qa_set.json",
    metrics=["faithfulness", "answer_relevance", "context_precision"]
)
report.export_markdown("evaluation_report.md")
```
