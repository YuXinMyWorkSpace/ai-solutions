## Overview

**SoloAI LLM 部署优化** 提供一套围绕大语言模型（LLM）推理部署的工程化工具与最佳实践，帮助开发者快速实现：

- 更低的推理延迟（low latency inference）
- 更高的吞吐能力（high throughput serving）
- 更优的显存与算力利用（GPU memory / compute efficiency）
- 更稳定的生产级服务部署（production-ready model serving）

该项目面向以下场景：

- LLM inference optimization
- 模型量化与加速部署
- 批处理与动态调度
- 多 GPU / 分布式部署
- Kubernetes / Docker / 云环境推理服务

---

## Features

- **高性能推理优化**：支持批处理、并发调度、请求合并与延迟优化，提升 LLM 在线服务效率。
- **量化与压缩支持**：兼容量化部署策略，如 INT8 / FP16 / 低精度推理，降低显存占用。
- **多后端部署能力**：支持本地、Docker、Kubernetes 以及常见 GPU 推理环境。
- **可观测性与监控**：集成日志、性能指标与健康检查，便于生产环境运维。
- **可扩展架构**：模块化设计，便于接入自定义模型、推理引擎与调度策略。
- **面向生产环境**：提供配置化部署、服务化接口与自动化流程，适合企业级 AI 基础设施。

## Installation

### Requirements

- Python 3.10+
- CUDA 11.8+（如需 GPU 推理）
- `pip` or `poetry`
- 推荐：Linux x86_64 / Ubuntu 20.04+

### Install from Source

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization

python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

pip install --upgrade pip
pip install -r requirements.txt
pip install -e .
```

### Optional: Install with Extra Dependencies

```bash
pip install -e ".[gpu,metrics,dev]"
```

### Docker Installation

```bash
docker build -t soloai/llm-deployment-optimization:latest .
docker run --gpus all -p 8080:8080 soloai/llm-deployment-optimization:latest
```

## Quick Start

以下示例展示如何启动一个基础的 LLM 推理优化服务：

```bash
python -m soloai_llm_opt.serve \
  --model /models/llama-7b \
  --host 0.0.0.0 \
  --port 8080 \
  --precision fp16 \
  --max-batch-size 16 \
  --enable-kv-cache \
  --workers 2
```

服务启动后，可通过 HTTP 接口进行推理请求：

```bash
curl -X POST http://localhost:8080/v1/generate \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Explain LLM deployment optimization in simple terms.",
    "max_tokens": 128,
    "temperature": 0.7
  }'
```

示例响应：

```json
{
  "id": "req_001",
  "object": "text_completion",
  "model": "llama-7b",
  "output": "LLM deployment optimization focuses on making large language models faster, cheaper, and more reliable in production...",
  "usage": {
    "prompt_tokens": 9,
    "completion_tokens": 27,
    "total_tokens": 36
  }
}
```

## Usage Examples

### 1. Python API

```python
from soloai_llm_opt import OptimizedEngine

engine = OptimizedEngine(
    model_path="/models/llama-7b",
    precision="fp16",
    max_batch_size=8,
    enable_kv_cache=True,
)

result = engine.generate(
    prompt="What are the key strategies for optimizing LLM inference?",
    max_tokens=100,
    temperature=0.5,
)

print(result.text)
```

### 2. Enable Quantized Inference

```bash
python -m soloai_llm_opt.serve \
  --model /models/llama-7b \
  --quantization int8 \
  --max-batch-size 32 \
  --enable-kv-cache
```

### 3. Run with Metrics Enabled

```bash
python -m soloai_llm_opt.serve \
  --model /models/llama-7b \
  --port 8080 \
  --enable-metrics \
  --metrics-port 9090
```

Prometheus metrics endpoint:

```bash
curl http://localhost:9090/metrics
```

### 4. Example Configuration File

```yaml
server:
  host: 0.0.0.0
  port: 8080
  workers: 2

model:
  path: /models/llama-7b
  precision: fp16
  quantization: int8
  max_batch_size: 16
  enable_kv_cache: true

monitoring:
  enable_metrics: true
  metrics_port: 9090
  log_level: info
```

使用配置文件启动：

```bash
python -m soloai_llm_opt.serve --config configs/production.yaml
```
