## 项目简介

**SoloAI 大模型微调数据服务** 是一个专注于 **LLM Fine-tuning Data Pipeline** 的开源项目，用于管理和生产高质量微调数据集。它支持多种数据格式与任务类型，帮助研究团队与企业快速搭建面向大模型训练的数据工程体系。

该项目特别适用于以下场景：

- 指令微调（Instruction / SFT）
- 对话数据构建（Multi-turn Conversation）
- 偏好数据构建（Preference / DPO / RLHF）
- 数据去重、脱敏、过滤与质量评估
- 面向训练框架的数据导出（JSONL / Parquet / Alpaca / ShareGPT 等）

---

## 核心特性

- **多源数据接入**：支持从本地文件、对象存储、数据库、API 等多渠道接入训练数据。
- **标准化数据处理管线**：内置清洗、去重、脱敏、切分、标签规范化等能力，提升数据一致性与训练效果。
- **多格式数据转换**：支持将数据导出为 `JSONL`、`Parquet`、`Alpaca`、`ShareGPT`、自定义 schema 等格式。
- **质量评估与可追踪性**：支持数据版本管理、样本审计、质量规则校验和处理日志追踪。
- **面向偏好学习的数据支持**：支持 `chosen/rejected`、pairwise preference、ranking 数据结构，适用于 DPO / RLHF 场景。
- **可扩展架构设计**：插件化处理器与任务编排机制，便于接入自定义规则、模型评估器和企业内部系统。

## 安装说明

### 环境要求

- Python 3.10+
- pip 23+
- 推荐使用 Linux / macOS
- 可选：Docker / Kubernetes 用于服务化部署

### 方式一：通过 pip 安装

```bash
pip install soloai-finetune-data-service
```

### 方式二：从源码安装

```bash
git clone https://github.com/SoloAI/llm-finetune-data-service.git
cd llm-finetune-data-service
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -U pip
pip install -e .[dev]
```

### 方式三：使用 Docker

```bash
docker build -t soloai/finetune-data-service:latest .
docker run --rm -p 8080:8080 soloai/finetune-data-service:latest
```

## 快速开始

下面示例演示如何将原始对话数据转换为适用于大模型监督微调（SFT）的标准数据集。

### 1. 准备输入数据

创建 `data/raw.jsonl`：

```json
{"id":"1","messages":[{"role":"user","content":"什么是大模型微调？"},{"role":"assistant","content":"大模型微调是基于预训练模型，使用领域数据继续训练以适配特定任务。"}]}
{"id":"2","messages":[{"role":"user","content":"请解释 SFT 和 DPO 的区别。"},{"role":"assistant","content":"SFT 使用监督信号训练，而 DPO 使用偏好对优化模型输出偏好。"}]}
```

### 2. 执行数据处理任务

```bash
soloai-data build \
  --input data/raw.jsonl \
  --output data/processed/sft.jsonl \
  --task sft \
  --format sharegpt \
  --deduplicate \
  --sanitize \
  --validate
```

### 3. 启动服务 API

```bash
soloai-data serve --host 0.0.0.0 --port 8080
```

### 4. 访问健康检查接口

```bash
curl http://localhost:8080/health
```

返回示例：

```json
{
  "status": "ok",
  "service": "soloai-finetune-data-service"
}
```

## 使用示例

### 1. Python SDK 示例

```python
from soloai_data import DataPipeline

pipeline = DataPipeline(
    task="sft",
    input_path="data/raw.jsonl",
    output_path="data/out/sft.jsonl",
    output_format="alpaca",
)

pipeline.enable_dedup()
pipeline.enable_sanitize()
pipeline.enable_validation()
pipeline.run()

print("Dataset build completed.")
```

### 2. 构建 DPO 偏好数据

输入示例 `data/preference.jsonl`：

```json
{"prompt":"介绍一下 Transformer。","chosen":"Transformer 是一种基于自注意力机制的神经网络架构。","rejected":"Transformer 是一种数据库查询语言。"}
```

命令行执行：

```bash
soloai-data build \
  --input data/preference.jsonl \
  --output data/out/dpo.jsonl \
  --task dpo \
  --format jsonl \
  --validate
```

### 3. 使用 REST API 提交数据转换任务

```bash
curl -X POST http://localhost:8080/v1/jobs \
  -H "Content-Type: application/json" \
  -d '{
    "task": "sft",
    "input": "s3://datasets/raw/customer-support.jsonl",
    "output": "s3://datasets/processed/customer-support-sft.jsonl",
    "format": "sharegpt",
    "options": {
      "deduplicate": true,
      "sanitize": true,
      "validate": true
    }
  }'
```

返回示例：

```json
{
  "job_id": "job_01HXYZ123ABC",
  "status": "queued"
}
```

### 4. 自定义数据质量规则

```python
from soloai_data import DataPipeline, Rule

class MaxLengthRule(Rule):
    name = "max_length"

    def validate(self, sample):
        text = " ".join([m["content"] for m in sample.get("messages", [])])
        return len(text) < 4000

pipeline = DataPipeline(task="sft")
pipeline.add_rule(MaxLengthRule())
pipeline.run()
```
