## 项目简介

**SoloAI LLM 部署优化** 是一个开源项目，聚焦于 **LLM inference optimization、model serving、quantization、batch scheduling、GPU memory optimization** 等核心问题。  
它为大模型推理部署提供统一的优化入口，帮助团队在保证服务质量的前提下提升吞吐、降低显存占用，并缩减整体基础设施成本。

### 适用场景

- 企业级 LLM API 服务部署
- 私有化大模型推理平台建设
- GPU/CPU 混合推理资源调度
- 多模型服务编排与弹性扩缩容
- LLM 量化、缓存与批处理优化

---

## 核心特性

- **推理性能优化**：支持批处理、并发调度与请求聚合，提升 tokens/sec 与整体吞吐。
- **显存与资源管理**：优化 KV Cache、显存分配和模型加载策略，降低 GPU OOM 风险。
- **量化与压缩支持**：集成常见量化工作流，支持降低模型体积与推理成本。
- **多后端部署兼容**：适配主流推理后端与服务框架，便于接入现有 MLOps / AI Infra。
- **可观测性与监控**：暴露延迟、吞吐、显存、错误率等关键指标，便于生产排障。
- **可扩展架构**：模块化设计，支持自定义优化策略、调度器与部署插件。

## 安装

### 环境要求

- Python 3.10+
- CUDA 11.8+（如使用 GPU）
- Linux/macOS
- 推荐使用 `venv` 或 `conda`

### 使用 pip 安装

```bash
pip install llm-deploy-optimizer
```

```bash

cd llm-deploy-optimizer
pip install -U pip
pip install -e .
```

### 安装可选依赖

```bash
pip install -e ".[gpu,monitoring,quantization]"
```

## 快速开始

以下示例展示如何使用 SoloAI LLM 部署优化工具加载模型配置，并启动一个经过优化的推理服务。

```python
from soloai_llm_opt import OptimizedServer, OptimizationConfig

config = OptimizationConfig(
    model="meta-llama/Llama-3-8B-Instruct",
    device="cuda",
    quantization="int4",
    max_batch_size=16,
    enable_kv_cache=True,
    tensor_parallel=1,
    metrics=True
)

server = OptimizedServer(config)
server.launch(host="0.0.0.0", port=8080)
```

启动后可通过 HTTP 接口发送推理请求：

```bash
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "meta-llama/Llama-3-8B-Instruct",
    "messages": [
      {"role": "user", "content": "解释一下 LLM 部署优化的关键点"}
    ],
    "max_tokens": 128
  }'
```

## 使用示例

### 1. 启用量化优化

通过量化减少模型显存占用，并提高部署密度。

```python
from soloai_llm_opt import OptimizationConfig

config = OptimizationConfig(
    model="Qwen/Qwen2-7B-Instruct",
    quantization="int8",
    device="cuda",
    max_batch_size=8
)
```

---

### 2. 启用动态批处理

动态批处理可以提升高并发场景下的吞吐表现。

```python
from soloai_llm_opt import OptimizationConfig

config = OptimizationConfig(
    model="mistralai/Mistral-7B-Instruct-v0.2",
    dynamic_batching=True,
    max_batch_size=32,
    batch_timeout_ms=20
)
```

---

### 3. 启用监控与指标导出

暴露 Prometheus 风格指标，方便接入 Grafana 与告警系统。

```python
from soloai_llm_opt import OptimizedServer, OptimizationConfig

config = OptimizationConfig(
    model="meta-llama/Llama-3-8B-Instruct",
    metrics=True,
    metrics_port=9090
)

server = OptimizedServer(config)
server.launch()
```

抓取指标：

```bash
curl http://localhost:9090/metrics
```

---

### 4. CLI 示例

项目同时提供命令行方式快速部署优化后的 LLM 服务。

```bash
soloai-llm-opt serve \
  --model meta-llama/Llama-3-8B-Instruct \
  --device cuda \
  --quantization int4 \
  --max-batch-size 16 \
  --enable-kv-cache \
  --port 8080
```

## 配置说明

| 配置项 | 类型 | 默认值 | 说明 |
|---|---|---:|---|
| `model` | `str` | - | 模型名称或本地模型路径 |
| `device` | `str` | `cpu` | 推理设备，如 `cpu` / `cuda` |
| `quantization` | `str` | `none` | 量化模式，如 `int8` / `int4` |
| `max_batch_size` | `int` | `1` | 最大批处理大小 |
| `dynamic_batching` | `bool` | `False` | 是否启用动态批处理 |
| `batch_timeout_ms` | `int` | `0` | 批处理聚合等待时间 |
| `enable_kv_cache` | `bool` | `True` | 是否启用 KV Cache |
| `tensor_parallel` | `int` | `1` | 张量并行数量 |
| `metrics` | `bool` | `False` | 是否启用指标监控 |
| `metrics_port` | `int` | `9090` | 指标暴露端口 |
