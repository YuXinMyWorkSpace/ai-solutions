# SoloAI 大模型微调数据服务

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/llm-finetune-data-service/ci.yml?branch=main&style=flat-square)](https://github.com/SoloAI/llm-finetune-data-service/actions)
[![License](https://img.shields.io/github/license/SoloAI/llm-finetune-data-service?style=flat-square)](./LICENSE)

**SoloAI 大模型微调数据服务** is an open-source data service platform for **LLM fine-tuning datasets**, designed to streamline data ingestion, cleaning, annotation, transformation, validation, and export workflows. It helps AI teams build high-quality, scalable, and production-ready datasets for training and fine-tuning large language models.

---

## Overview

Large model performance depends heavily on data quality. SoloAI 大模型微调数据服务 provides a standardized pipeline for managing the full lifecycle of fine-tuning data, including structured data processing, instruction dataset generation, schema validation, and multi-format export for popular AI training frameworks.

---

## Features

- **统一数据接入**
  - Ingest data from JSON, JSONL, CSV, Parquet, databases, and object storage.
- **数据清洗与标准化**
  - Normalize text, remove duplicates, filter low-quality samples, and enforce schema consistency.
- **指令微调数据构建**
  - Build instruction-following datasets for SFT, chat tuning, and domain adaptation.
- **质量校验与规则引擎**
  - Validate dataset fields, token length, language distribution, and content safety rules.
- **多格式导出**
  - Export datasets to formats compatible with Hugging Face, OpenAI-style chat format, Alpaca, and custom training pipelines.
- **可扩展服务架构**
  - API-first design with modular processors for enterprise AI data engineering workflows.

## Installation

### Requirements

- Python 3.10+
- pip 23+
- Optional:
  - Docker 24+
  - PostgreSQL / MySQL / SQLite
  - Redis

### Install from source

```bash
git clone https://github.com/SoloAI/llm-finetune-data-service.git
cd llm-finetune-data-service
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -U pip
pip install -r requirements.txt
```

### Install with Docker

```bash
docker build -t soloai-finetune-data-service .
docker run -p 8080:8080 soloai-finetune-data-service
```

## Quick Start

Start the service locally:

```bash
python -m app.main
```

Example API request to create a cleaned fine-tuning dataset:

```bash
curl -X POST http://localhost:8080/api/v1/datasets/process \
  -H "Content-Type: application/json" \
  -d '{
    "name": "customer-support-sft",
    "source_format": "jsonl",
    "target_format": "openai_chat",
    "pipeline": ["deduplicate", "normalize", "validate", "export"],
    "data": [
      {
        "instruction": "解释什么是向量数据库",
        "input": "",
        "output": "向量数据库是一种专门用于存储和检索向量嵌入的数据库系统。"
      }
    ]
  }'
```

Example response:

```json
{
  "dataset_id": "ds_8f2c91",
  "status": "processed",
  "records_in": 1,
  "records_out": 1,
  "export_url": "/exports/ds_8f2c91.jsonl"
}
```

## Usage Examples

### 1. Python SDK Example

```python
from soloai_data_service import DataClient

client = DataClient(base_url="http://localhost:8080")

dataset = client.process_dataset(
    name="finance-domain-sft",
    source_format="jsonl",
    target_format="huggingface",
    pipeline=["normalize", "quality_filter", "schema_validate", "export"],
    data=[
        {
            "instruction": "什么是资产负债表？",
            "input": "",
            "output": "资产负债表是反映企业在某一特定日期财务状况的会计报表。"
        }
    ]
)

print(dataset)
```

### 2. Input Dataset Example

```json
[
  {
    "instruction": "将下面文本总结为一句话",
    "input": "大模型微调通常依赖高质量、结构化和经过校验的数据集。",
    "output": "大模型微调需要高质量且结构化的数据集。"
  }
]
```

### 3. Export to OpenAI Chat Format

```json
{
  "messages": [
    {
      "role": "system",
      "content": "你是一个专业的AI助手。"
    },
    {
      "role": "user",
      "content": "什么是监督微调？"
    },
    {
      "role": "assistant",
      "content": "监督微调是利用带标签的数据对预训练模型进行进一步训练，以适应特定任务或领域。"
    }
  ]
}
```

### 4. CLI Example

```bash
python -m cli.dataset build \
  --input ./data/raw/customer_support.jsonl \
  --output ./data/processed/customer_support_openai.jsonl \
  --source-format jsonl \
  --target-format openai_chat \
  --pipeline deduplicate,normalize,validate
```

## Architecture

```text
Raw Data Sources
   ├── JSON / JSONL / CSV / Parquet
   ├── DB / Object Storage / APIs
   ▼
Ingestion Layer
   ▼
Cleaning & Normalization
   ▼
Quality Validation & Rule Engine
   ▼
Instruction / Chat Dataset Transformation
   ▼
Dataset Export Layer
   ├── Hugging Face
   ├── OpenAI Chat
   ├── Alpaca
   └── Custom Schema
```

This architecture supports scalable **LLM fine-tuning data pipelines**, **instruction tuning dataset preparation**, and **enterprise AI data governance**.
