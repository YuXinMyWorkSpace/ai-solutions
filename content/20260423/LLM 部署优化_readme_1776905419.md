# SoloAI LLM 部署优化

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/llm-deploy-optimization/ci.yml?branch=main)](https://github.com/SoloAI/llm-deploy-optimization/actions)
[![License](https://img.shields.io/github/license/SoloAI/llm-deploy-optimization)](./LICENSE)

面向生产环境的大语言模型（LLM）部署优化工具链，帮助开发者在 **推理性能、资源利用率、延迟控制与部署稳定性** 之间取得最佳平衡。  
该项目专注于 **LLM inference optimization、quantization、batch scheduling、serving orchestration** 等核心场景，适用于本地、边缘和云端部署。

---

## Overview

**SoloAI LLM 部署优化** 是一个用于提升大模型推理部署效率的开源项目，聚焦以下问题：

- 如何降低 LLM 在线推理延迟
- 如何在有限 GPU / CPU 资源下提升吞吐量
- 如何通过量化、缓存和调度优化部署成本
- 如何构建可观测、可扩展、可维护的 LLM Serving 系统

无论你是在构建 AI SaaS、企业知识助手、推理 API 服务，还是边缘端模型推理平台，该项目都可以作为 **LLM deployment optimization** 的基础设施组件。

---

## Features

- **高性能推理优化**  
  支持针对 LLM serving 的延迟优化、吞吐优化与资源调度策略配置。

- **量化与模型压缩支持**  
  提供常见量化流程集成能力，帮助降低显存占用并提升部署效率。

- **动态批处理与请求调度**  
  支持 batch 合并、并发控制、队列调度等能力，提升多请求场景吞吐表现。

- **KV Cache 与内存优化**  
  面向长上下文和高并发服务，优化缓存管理与显存利用率。

- **可观测性与性能分析**  
  集成日志、指标与 profiling 接口，便于定位瓶颈并持续优化。

- **多环境部署兼容**  
  支持本地开发、Docker、Kubernetes 及云原生推理服务部署方式。

## Installation

### Requirements

- Python 3.10+
- CUDA 11.8+（如需 GPU 推理）
- Linux/macOS（生产环境推荐 Linux）
- `pip` 或 `poetry`

### Option 1: Install from source

```bash
git clone https://github.com/SoloAI/llm-deploy-optimization.git
cd llm-deploy-optimization
python -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
pip install -e .
```

### Option 2: Install with Poetry

```bash
git clone https://github.com/SoloAI/llm-deploy-optimization.git
cd llm-deploy-optimization
poetry install
```

### Optional dependencies

```bash
# GPU acceleration
pip install -e ".[gpu]"

# Quantization toolchain
pip install -e ".[quant]"

# Development tools
pip install -e ".[dev]"
```

## Quick Start

以下示例展示如何使用 SoloAI 对 LLM 服务进行基础部署优化配置。

```python
from soloai_llm_opt import OptimizedLLMServer, OptimizationConfig

config = OptimizationConfig(
    model_name="meta-llama/Llama-3-8B-Instruct",
    device="cuda",
    quantization="int8",
    max_batch_size=16,
    enable_kv_cache=True,
    tensor_parallel_size=1,
    request_timeout=30,
)

server = OptimizedLLMServer(config)
server.load_model()

response = server.generate(
    prompt="请简要说明 LLM 部署优化的核心目标。",
    max_tokens=128,
    temperature=0.7,
)

print(response.text)
```

启动服务：

```bash
soloai-serve --model meta-llama/Llama-3-8B-Instruct \
             --device cuda \
             --quantization int8 \
             --max-batch-size 16 \
             --enable-kv-cache
```

## Usage

### 1. 启动推理 API 服务

```bash
soloai-serve \
  --model mistralai/Mistral-7B-Instruct-v0.2 \
  --host 0.0.0.0 \
  --port 8080 \
  --device cuda \
  --max-batch-size 32 \
  --scheduler dynamic \
  --enable-kv-cache
```

### 2. 发送推理请求

```bash
curl -X POST http://localhost:8080/v1/generate \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "解释一下为什么量化可以降低 LLM 部署成本。",
    "max_tokens": 120,
    "temperature": 0.6
  }'
```

示例响应：

```json
{
  "id": "gen_01HXYZ...",
  "model": "mistralai/Mistral-7B-Instruct-v0.2",
  "text": "量化通过降低模型权重与激活表示精度，显著减少显存占用和带宽需求，从而提升吞吐率并降低硬件成本。",
  "usage": {
    "prompt_tokens": 18,
    "completion_tokens": 42,
    "total_tokens": 60
  },
  "latency_ms": 138
}
```

### 3. 使用 YAML 配置部署

创建配置文件 `config.yaml`：

```yaml
model:
  name: meta-llama/Llama-3-8B-Instruct
  device: cuda
  quantization: int4

serving:
  host: 0.0.0.0
  port: 8000
  max_batch_size: 24
  scheduler: dynamic
  request_timeout: 60

optimization:
  enable_kv_cache: true
  enable_prefix_cache: true
  memory_fraction: 0.9
  enable_metrics: true
```

使用配置启动：

```bash
soloai-serve --config config.yaml
```

### 4. 性能基准测试

```bash
soloai-bench \
  --endpoint http://localhost:8000/v1/generate \
  --concurrency 32 \
  --requests 500 \
  --input-tokens 256 \
  --output-tokens 128
```

示例输出：

```text
Benchmark Summary
-----------------
Avg Latency:      142.8 ms
P95 Latency:      231.4 ms
Throughput:       78.2 req/s
Token Throughput: 12450 tokens/s
Error Rate:       0.2%
```
