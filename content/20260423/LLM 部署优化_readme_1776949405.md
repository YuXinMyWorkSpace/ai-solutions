## 简介

**LLM 部署优化** 是一个面向生产环境的大语言模型部署与性能调优项目，聚焦于推理加速、显存优化、弹性扩缩容和服务稳定性。  
它为 AI 工程团队、平台团队和企业研发部门提供可复用的最佳实践、工具链和部署模板。

## 核心特性

- **高性能推理部署**：支持低延迟、高吞吐的 LLM 推理服务架构

- **多环境部署支持**：适配本地、GPU 服务器、容器平台及云环境

- **弹性扩展能力**：支持并发调度、横向扩容与服务治理
- **工程化最佳实践**：提供标准化配置、部署脚本与可扩展示例

## 安装

### 环境要求

- Python 3.10+
- CUDA 11.8+（如使用 NVIDIA GPU）
- Docker 20.10+（可选）
- Linux/macOS，推荐 Ubuntu 20.04+

### 方式一：从源码安装

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization

python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

pip install -U pip
pip install -r requirements.txt
```

### 方式二：开发模式安装

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
pip install -e .
```

### 方式三：Docker 部署

```bash
docker build -t soloai/llm-deployment-optimization:latest .
docker run --gpus all -p 8080:8080 soloai/llm-deployment-optimization:latest
```

## 快速开始

以下示例展示如何使用默认配置启动一个基础的 LLM 推理优化服务：

```bash
python -m app.server \
  --model meta-llama/Llama-3-8B-Instruct \
  --host 0.0.0.0 \
  --port 8080 \
  --max-batch-size 16 \
  --dtype float16
```

服务启动后，你可以通过 HTTP 请求进行推理测试：

```bash
curl -X POST "http://127.0.0.1:8080/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "meta-llama/Llama-3-8B-Instruct",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user", "content": "请解释 LLM 部署优化的核心指标。"}
    ],
    "temperature": 0.2,
    "max_tokens": 256
  }'
```

## 用法示例

### 1. 使用 YAML 配置启动服务

```yaml
# configs/production.yaml
model: meta-llama/Llama-3-8B-Instruct
host: 0.0.0.0
port: 8080
dtype: float16
tensor_parallel_size: 2
max_batch_size: 32
enable_metrics: true
enable_flash_attention: true
quantization: int8
```

启动命令：

```bash
python -m app.server --config configs/production.yaml
```

---

### 2. 开启量化优化

适用于显存受限或希望降低推理成本的场景：

```bash
python -m app.server \
  --model mistralai/Mistral-7B-Instruct-v0.2 \
  --quantization int8 \
  --max-batch-size 32 \
  --gpu-memory-utilization 0.9
```

---

### 3. 运行性能基准测试

```bash
python -m benchmarks.run \
  --model meta-llama/Llama-3-8B-Instruct \
  --concurrency 32 \
  --requests 500 \
  --input-tokens 512 \
  --output-tokens 256
```

示例输出指标：

```text
Latency P50: 142 ms
Latency P95: 288 ms
Throughput: 124.6 tokens/s
GPU Memory: 18.2 GB
Error Rate: 0.00%
```

---

### 4. Python API 集成示例

```python

client = LLMClient(base_url="http://127.0.0.1:8080")

resp = client.chat(
    model="meta-llama/Llama-3-8B-Instruct",
    messages=[
        {"role": "system", "content": "You are an inference optimization assistant."},
        {"role": "user", "content": "如何降低大模型推理延迟？"}
    ],
    temperature=0.1,
    max_tokens=200,
```

## 性能优化建议

为便于 AI 工程团队快速落地，建议重点关注以下指标：

- **Latency**：P50 / P95 / P99 响应时间
- **Throughput**：每秒请求数（RPS）或 tokens/s
- **GPU Utilization**：GPU 利用率与空转比例
- **Memory Footprint**：显存占用与峰值内存
- **Cost per Request**：单次请求成本与资源折算成本
- **Stability**：错误率、超时率、重试率

在生产场景中，通常需要结合以下方法进行优化：

- 动态批处理（Dynamic Batching）
- 模型量化（INT8 / 4-bit）
- 张量并行与流水并行
- 请求路由与多实例负载均衡
- KV Cache 复用与上下文管理
- 推理引擎优化，如 vLLM、TensorRT-LLM、TGI 等
