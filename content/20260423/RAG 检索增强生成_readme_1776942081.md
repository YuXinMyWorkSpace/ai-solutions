# SoloAI RAG 检索增强生成

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/rag-retrieval-augmented-generation/ci.yml?branch=main&style=flat-square)](https://github.com/SoloAI/rag-retrieval-augmented-generation/actions)
[![License](https://img.shields.io/github/license/SoloAI/rag-retrieval-augmented-generation?style=flat-square)](./LICENSE)

**SoloAI RAG 检索增强生成** is an open-source framework for building production-ready **Retrieval-Augmented Generation (RAG)** systems. It helps developers connect private knowledge bases with LLMs to deliver more accurate, grounded, and explainable AI answers.

---

## Overview

RAG（检索增强生成） combines **semantic retrieval** with **large language model generation** to reduce hallucinations, improve factual accuracy, and enable enterprise AI applications over proprietary data. This project provides a clean, extensible foundation for document ingestion, embedding, vector search, context assembly, and answer generation.

## Features

- **Modular RAG pipeline** for ingestion, chunking, embedding, retrieval, reranking, and generation
- **Multi-source document support** including Markdown, PDF, TXT, HTML, and structured data
- **Vector database integration** with pluggable backends for scalable semantic search
- **Prompt orchestration and context grounding** for higher-quality, source-aware responses
- **API-first architecture** for easy integration into chatbots, knowledge assistants, and enterprise workflows
- **Production-friendly design** with configuration management, logging, and extensibility

## Installation

### Requirements

- Python 3.10+
- `pip` or `poetry`
- Optional: running vector database such as FAISS, Milvus, or Chroma

### Install from source

```bash
git clone https://github.com/SoloAI/rag-retrieval-augmented-generation.git
cd rag-retrieval-augmented-generation
pip install -r requirements.txt
```

### Optional environment setup

```bash
python -m venv .venv
source .venv/bin/activate   # Linux / macOS
# .venv\Scripts\activate    # Windows

pip install --upgrade pip
pip install -r requirements.txt
```

### Environment variables

Create a `.env` file in the project root:

```env

EMBEDDING_MODEL=text-embedding-3-large
LLM_MODEL=gpt-4o-mini
VECTOR_DB=faiss
DATA_DIR=./data
```

## Quick Start

Below is a minimal example showing how to ingest documents and query the RAG pipeline.

```python
from soloai_rag import RAGPipeline

pipeline = RAGPipeline(
    embedding_model="text-embedding-3-large",
    llm_model="gpt-4o-mini",
    vector_store="faiss"
)

pipeline.ingest("./data")

response = pipeline.query(
    question="什么是 RAG 检索增强生成？",
    top_k=5
)

print(response.answer)
print(response.sources)
```

Run it:

```bash
python examples/quickstart.py
```

## Usage Examples

### 1. Document Ingestion

Load local knowledge files into the retrieval index.

```python
from soloai_rag import DocumentIngestor

ingestor = DocumentIngestor(
    chunk_size=800,
    chunk_overlap=120
)

ingestor.load("./knowledge_base")
ingestor.build_index(vector_store="faiss")
```

### 2. Semantic Retrieval

Retrieve the most relevant chunks before generation.

```python
from soloai_rag import Retriever

retriever = Retriever(index_path="./indexes/faiss")

results = retriever.search(
    query="How does retrieval augmentation improve LLM accuracy?",
    top_k=3
)

for item in results:
    print(item.score, item.text[:200])
```

### 3. RAG Question Answering

Generate answers grounded in retrieved context.

```python
from soloai_rag import RAGPipeline

pipeline = RAGPipeline.from_config("configs/default.yaml")

result = pipeline.query(
    question="请根据知识库总结企业知识助手的核心架构",
    top_k=5,
    return_sources=True
)

print("Answer:", result.answer)
print("Sources:", result.sources)
```

### 4. API Service

Example FastAPI service for exposing a RAG endpoint.

```python
from fastapi import FastAPI
from pydantic import BaseModel
from soloai_rag import RAGPipeline

app = FastAPI(title="SoloAI RAG API")
pipeline = RAGPipeline.from_config("configs/default.yaml")

class QueryRequest(BaseModel):
    question: str
    top_k: int = 5

@app.post("/query")
def query_rag(req: QueryRequest):
    result = pipeline.query(req.question, top_k=req.top_k, return_sources=True)
    return {
        "answer": result.answer,
        "sources": result.sources
    }
```

Run the API:

```bash
uvicorn app:app --reload --host 0.0.0.0 --port 8000
```
