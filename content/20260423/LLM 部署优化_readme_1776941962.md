A professional open-source toolkit by **SoloAI** for **LLM 部署优化** (LLM deployment optimization), focused on improving inference performance, reducing serving cost, and simplifying production rollout across cloud, edge, and on-prem environments.  
It helps teams optimize large language model serving with practical tools for quantization, acceleration, observability, and scalable deployment workflows.

---

## Features

- **High-performance inference optimization** for LLM serving pipelines
- **Quantization and model compression** to reduce memory footprint and hardware cost
- **Multi-environment deployment support** for cloud, Kubernetes, bare metal, and edge scenarios
- **Observability and performance profiling** for latency, throughput, and resource utilization
- **Scalable API serving architecture** compatible with modern model serving stacks
- **Production-oriented configuration templates** for fast rollout and repeatable deployment

## Installation

### Requirements

- Python 3.9+
- CUDA 11.8+ recommended for GPU acceleration
- Docker optional, but recommended for production deployment

### Clone the repository

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
```

### Install with pip

```bash
pip install -r requirements.txt
```

### Optional: install development dependencies

```bash
pip install -r requirements-dev.txt
```

### Optional: Docker setup

```bash
docker build -t soloai/llm-deployment-optimization:latest .
docker run --gpus all -p 8000:8000 soloai/llm-deployment-optimization:latest
```

## Quick Start

Below is a minimal example to launch an optimized LLM inference service.

```bash
python serve.py \
  --model meta-llama/Llama-2-7b-chat-hf \
  --host 0.0.0.0 \
  --port 8000 \
  --quantization int8 \
  --max-batch-size 16
```

Send a test request:

```bash
curl -X POST "http://localhost:8000/generate" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Explain tensor parallelism in simple terms.",
    "max_tokens": 128,
    "temperature": 0.7
  }'
```

Example response:

```json
{
  "text": "Tensor parallelism splits model computation across multiple GPUs so large layers can be processed faster and fit into available memory."
}
```

## Usage Examples

### 1. Run with 4-bit quantization

```bash
python serve.py \
  --model mistralai/Mistral-7B-Instruct-v0.2 \
  --quantization int4 \
  --device cuda \
  --port 8000
```

### 2. Enable metrics and profiling

```bash
python serve.py \
  --model meta-llama/Llama-2-7b-chat-hf \
  --enable-metrics \
  --metrics-port 9090 \
  --enable-profiler
```

### 3. Deploy with Docker Compose

```yaml
version: "3.9"

services:
  llm-server:
    image: soloai/llm-deployment-optimization:latest
    ports:
      - "8000:8000"
    environment:
      MODEL_ID: meta-llama/Llama-2-7b-chat-hf
      QUANTIZATION: int8
      MAX_BATCH_SIZE: 16
    deploy:
      resources:
        reservations:
          devices:
            - capabilities: [gpu]
```

Run it:

```bash
docker compose up -d
```

### 4. Python client example

```python
import requests

payload = {
    "prompt": "What are the key trade-offs between latency and throughput in LLM serving?",
    "max_tokens": 150,
    "temperature": 0.3
}

response = requests.post("http://localhost:8000/generate", json=payload, timeout=60)
print(response.json())
```

### 5. Kubernetes deployment snippet

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: llm-server
spec:
  replicas: 1
  selector:
    matchLabels:
      app: llm-server
  template:
    metadata:
      labels:
        app: llm-server
    spec:
      containers:
        - name: llm-server
          image: soloai/llm-deployment-optimization:latest
          ports:
            - containerPort: 8000
          env:
            - name: MODEL_ID
              value: "meta-llama/Llama-2-7b-chat-hf"
            - name: QUANTIZATION
              value: "int8"
```
