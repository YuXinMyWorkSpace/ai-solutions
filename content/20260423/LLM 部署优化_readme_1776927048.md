# SoloAI LLM 部署优化

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/llm-deployment-optimizer/ci.yml?branch=main&style=flat-square)](https://github.com/SoloAI/llm-deployment-optimizer/actions)
[![License](https://img.shields.io/github/license/SoloAI/llm-deployment-optimizer?style=flat-square)](./LICENSE)

A high-performance open-source toolkit by **SoloAI** for **LLM 部署优化** (LLM deployment optimization), designed to improve inference efficiency, reduce latency, and lower serving costs across local, cloud, and edge environments.  
It provides practical workflows for model quantization, runtime tuning, batching, hardware-aware deployment, and observability for production-grade large language model serving.

---

## Features

- **Inference Performance Optimization**  
  Optimize token throughput, latency, and memory utilization for production LLM workloads.

- **Quantization & Compression Support**  
  Enable low-bit deployment strategies such as INT8/INT4 and other model compression techniques.

- **Hardware-Aware Deployment**  
  Tune deployment for GPUs, CPUs, and heterogeneous inference environments.

- **Dynamic Batching & Concurrency Control**  
  Improve serving efficiency under variable traffic patterns with intelligent batching strategies.

- **Observability & Benchmarking**  
  Track latency, throughput, memory footprint, and deployment metrics with built-in monitoring hooks.

- **Production-Ready Configuration**  
  Standardize deployment pipelines with reproducible configs, CLI workflows, and environment profiles.

## Installation

### Requirements

- Python 3.10+
- Linux/macOS recommended
- CUDA-enabled GPU optional but recommended for high-throughput inference

### Install from source

```bash
git clone https://github.com/SoloAI/llm-deployment-optimizer.git
cd llm-deployment-optimizer
pip install -U pip
pip install -e .
```

### Install with optional dependencies

```bash
pip install -e ".[gpu,benchmark,monitoring]"
```

### Verify installation

```bash
soloai-llm-opt --help
```

## Quick Start

Use the CLI to benchmark and optimize an LLM deployment profile:

```bash
soloai-llm-opt optimize \
  --model meta-llama/Llama-2-7b-hf \
  --device cuda \
  --quantization int8 \
  --batch-size 8 \
  --max-input-length 2048 \
  --max-output-length 512
```

Example output:

```text
[INFO] Loading model: meta-llama/Llama-2-7b-hf
[INFO] Applying quantization: int8
[INFO] Target device: cuda
[INFO] Running benchmark...
[INFO] Avg latency: 82.4 ms
[INFO] Throughput: 121.7 tokens/s
[INFO] Peak memory: 9.8 GB
[INFO] Optimization profile saved to: ./artifacts/llama2-7b-int8.yaml
```

## Usage

### 1. Python API

```python
from soloai_llm_opt import Optimizer, DeploymentConfig

config = DeploymentConfig(
    model="meta-llama/Llama-2-7b-hf",
    device="cuda",
    quantization="int8",
    batch_size=8,
    max_input_length=2048,
    max_output_length=512,
)

optimizer = Optimizer(config)
report = optimizer.run()

print("Latency:", report.latency_ms)
print("Throughput:", report.tokens_per_second)
print("Peak Memory:", report.peak_memory_gb)
```

---

### 2. Benchmark Different Configurations

```bash
soloai-llm-opt benchmark \
  --model mistralai/Mistral-7B-v0.1 \
  --device cuda \
  --batch-sizes 1,2,4,8,16 \
  --quantizations fp16,int8,int4 \
  --output results/benchmark-report.json
```

---

### 3. Deploy with a YAML Profile

Create a deployment profile:

```yaml
# deploy.yaml
model: meta-llama/Llama-2-7b-hf
device: cuda
quantization: int4
batch_size: 16
max_input_length: 4096
max_output_length: 1024
enable_dynamic_batching: true
target_latency_ms: 100
monitoring:
  enabled: true
  export_prometheus: true
```

Run deployment optimization:

```bash
soloai-llm-opt deploy --config deploy.yaml
```

---

### 4. Observability Example

```python
from soloai_llm_opt import Monitor, Optimizer, DeploymentConfig

config = DeploymentConfig(
    model="Qwen/Qwen2-7B-Instruct",
    device="cuda",
    quantization="int4",
    batch_size=4,
)

optimizer = Optimizer(config)
report = optimizer.run()

monitor = Monitor()
monitor.log_metric("latency_ms", report.latency_ms)
monitor.log_metric("tokens_per_second", report.tokens_per_second)
monitor.log_metric("peak_memory_gb", report.peak_memory_gb)
```
