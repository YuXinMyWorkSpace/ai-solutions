## Overview

**SoloAI LLM 部署优化** 提供一套用于优化大模型部署流程的工程化方案，帮助开发者和企业快速完成从模型加载、量化加速、推理服务到监控运维的落地。  
项目重点关注 **低延迟推理、高吞吐服务、GPU/CPU 资源优化、容器化部署、可观测性和生产级稳定性**。

## Features

- **多种推理优化策略**：支持量化、KV Cache 优化、批处理调度与并发推理配置
- **灵活部署方式**：支持本地部署、Docker 容器化部署及云端服务化集成
- **高性能推理接口**：提供标准化 API 接口，便于接入业务系统和 AI 应用
- **资源利用率优化**：针对 GPU 显存、CPU 内存和吞吐能力进行调优
- **生产级可观测性**：支持日志、指标监控和性能分析，便于定位瓶颈
- **可扩展架构设计**：适合企业私有化部署、多模型管理和横向扩容场景

## Installation

### Requirements

- Python 3.10+
- CUDA 11.8+（如使用 GPU）
- Docker 24+（可选）
- NVIDIA Driver / NVIDIA Container Toolkit（GPU 容器部署可选）

### Option 1: Clone the repository

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
python -m venv .venv
source .venv/bin/activate   # Linux / macOS
# .venv\Scripts\activate    # Windows

pip install -U pip
pip install -r requirements.txt
```

### Option 2: Install with Docker

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
docker build -t soloai/llm-opt:latest .
docker run --gpus all -p 8000:8000 soloai/llm-opt:latest
```

## Quick Start

以下示例展示如何启动一个基础的 LLM 推理服务：

```bash
python serve.py \
  --model /models/your-llm \
  --host 0.0.0.0 \
  --port 8000 \
  --device cuda \
  --max-batch-size 16 \
  --dtype float16
```

服务启动后，可通过 HTTP API 进行推理请求：

```bash
curl -X POST "http://127.0.0.1:8000/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "your-llm",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user", "content": "请介绍一下 LLM 部署优化的关键点。"}
    ],
    "temperature": 0.7,
    "max_tokens": 256
  }'
```

## Usage Examples

### 1. Quantized Deployment

使用 4-bit / 8-bit 量化降低显存占用并提升部署效率：

```bash
python serve.py \
  --model /models/your-llm \
  --quantization int4 \
  --device cuda \
  --max-batch-size 32
```

### 2. High-Throughput Inference

针对高并发请求进行批处理和并行优化：

```bash
python serve.py \
  --model /models/your-llm \
  --device cuda \
  --max-batch-size 64 \
  --num-workers 4 \
  --enable-dynamic-batching
```

### 3. CPU Deployment

适用于测试环境、轻量级服务或边缘场景：

```bash
python serve.py \
  --model /models/your-llm \
  --device cpu \
  --threads 8
```

### 4. Python Client Example

```python
import requests

url = "http://127.0.0.1:8000/v1/chat/completions"
payload = {
    "model": "your-llm",
    "messages": [
        {"role": "user", "content": "如何降低 LLM 推理延迟？"}
    ],
    "temperature": 0.2,
    "max_tokens": 128
}

response = requests.post(url, json=payload, timeout=60)
print(response.json())
```

### 5. Docker Compose Example

```yaml
version: "3.9"

services:
  llm-server:
    image: soloai/llm-opt:latest
    ports:
      - "8000:8000"
    environment:
      MODEL_PATH: /models/your-llm
      DEVICE: cuda
      MAX_BATCH_SIZE: 16
    volumes:
      - ./models:/models
    deploy:
      resources:
        reservations:
          devices:
            - capabilities: [gpu]
```

启动：

```bash
docker compose up -d
```

## Best Practices for LLM Deployment Optimization

为获得最佳生产效果，建议优先关注以下方向：

- 根据模型规模选择合适的 **量化策略**（FP16 / INT8 / INT4）
- 使用 **动态批处理** 提升吞吐，同时控制 P95 / P99 延迟
- 开启 **KV Cache** 与高效 attention 实现，减少重复计算
- 对 GPU 显存使用、CPU 线程数和 I/O 路径进行压测与调优
- 通过容器化与监控系统实现可回滚、可观测、可扩容的部署架构
