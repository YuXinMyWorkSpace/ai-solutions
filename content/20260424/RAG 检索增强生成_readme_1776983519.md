# SoloAI RAG 检索增强生成

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/rag-retrieval-augmented-generation/ci.yml?branch=main&style=flat-square)](https://github.com/SoloAI/rag-retrieval-augmented-generation/actions)
[![License](https://img.shields.io/github/license/SoloAI/rag-retrieval-augmented-generation?style=flat-square)](./LICENSE)

A production-ready open-source framework for **RAG（Retrieval-Augmented Generation，检索增强生成）** workflows, designed to help developers build accurate, grounded, and scalable AI applications.  
It combines document ingestion, embedding, vector retrieval, re-ranking, and LLM generation into a modular pipeline optimized for enterprise knowledge systems and intelligent assistants.

---

## Overview

**SoloAI RAG 检索增强生成** is built for teams and developers who need a reliable way to connect large language models with private or domain-specific knowledge bases. It supports end-to-end retrieval pipelines, making it easier to reduce hallucinations, improve answer relevance, and deliver explainable AI outputs.

### Why RAG?

RAG enables LLMs to answer questions based on retrieved context instead of relying only on model memory. This is especially useful for:

- Enterprise knowledge bases
- Customer support automation
- Internal search assistants
- Legal, medical, and financial document QA
- Multi-source semantic search applications

---

## Features

- **Modular RAG Pipeline**: Document parsing, chunking, embedding, indexing, retrieval, and generation
- **Vector Database Integration**: Works with common vector stores such as FAISS, Milvus, Chroma, and Pinecone
- **Hybrid Retrieval Support**: Combine dense vector search with keyword or metadata filtering for improved relevance
- **LLM Provider Flexibility**: Compatible with OpenAI, Azure OpenAI, local LLMs, and other API-based models
- **Re-ranking and Context Optimization**: Improve retrieval quality with re-rankers and prompt-aware context selection
- **Production-Oriented Design**: Configurable APIs, logging, evaluation hooks, and scalable deployment patterns

---

## Architecture

```text
Documents -> Parsing -> Chunking -> Embedding -> Vector Store
                                                   |
User Query -> Query Embedding -> Retriever -> Re-ranker -> Prompt Builder -> LLM -> Answer
                                                   |
                                             Source Citations
```

## Quick Start

Below is a minimal example showing how to ingest documents and run a retrieval-augmented query.

```python
from soloai_rag import RAGPipeline

pipeline = RAGPipeline(
    embedding_model="text-embedding-3-large",
    llm_model="gpt-4o-mini",
    vector_store="faiss"
)

pipeline.ingest(
    source_path="./data/docs",
    chunk_size=512,
    chunk_overlap=64
)

response = pipeline.query(
    "什么是 RAG 检索增强生成？它如何减少大模型幻觉？",
    top_k=5,
    return_sources=True
)

print(response.answer)
print(response.sources)
```

Run it:

```bash
python examples/quick_start.py
```

## Usage Examples

### 1. Index local documents

```python
from soloai_rag import DocumentIndexer

indexer = DocumentIndexer(
    embedding_model="text-embedding-3-large",
    vector_store="chroma"
)

indexer.add_documents("./knowledge_base/")
indexer.persist("./storage/index")
```

### 2. Query with source citations

```python
from soloai_rag import RAGPipeline

pipeline = RAGPipeline.load("./storage/index")

result = pipeline.query(
    "Summarize the product requirements in the uploaded documentation.",
    top_k=3,
    return_sources=True
)

print("Answer:", result.answer)
for source in result.sources:
    print(source["path"], source["score"])
```

### 3. Hybrid retrieval with metadata filters

```python
result = pipeline.query(
    "查找 2024 年发布的 API 升级说明",
    top_k=5,
    filters={"year": 2024, "type": "release_note"},
    retrieval_mode="hybrid"
)
```

### 4. API server mode

```bash
uvicorn api.main:app --host 0.0.0.0 --port 8000
```

Example request:

```bash
curl -X POST "http://localhost:8000/query" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "Explain the onboarding workflow from the HR handbook.",
    "top_k": 5,
    "return_sources": true
  }'
```

## Configuration

Example configuration file:

```yaml
embedding:
  provider: openai
  model: text-embedding-3-large

llm:
  provider: openai
  model: gpt-4o-mini
  temperature: 0.2

retrieval:
  top_k: 5
  mode: hybrid
  reranker: true

chunking:
  chunk_size: 512
  chunk_overlap: 64

vector_store:
  provider: faiss
  persist_path: ./storage/index
```

## Best Practices for RAG Systems

- Use domain-specific document preprocessing to improve chunk quality
- Tune `chunk_size`, `chunk_overlap`, and `top_k` based on your corpus
- Add metadata filters for better retrieval precision
- Use re-ranking to improve the final context window
- Log retrieval outputs and evaluate answer grounding regularly
- Return source citations for transparency and trust
