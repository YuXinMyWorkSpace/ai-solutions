---

## 项目简介

**LLM 部署优化** 旨在为开发者、MLOps 工程师和 AI 基础设施团队提供一套可复用的优化方案，用于改进大模型在部署阶段的性能表现，包括推理加速、资源利用优化、批处理调度、量化部署与服务监控等核心能力。

该项目适用于 GPU/CPU 混合环境、云原生推理架构、边缘部署场景以及需要兼顾成本与 SLA 的 LLM 在线服务。

---

## 核心特性

- **高性能推理优化**：支持批处理、并发调度、KV Cache 优化与低延迟推理策略。
- **多种部署方案**：兼容本地部署、容器化部署、Kubernetes、私有云与混合云环境。
- **量化与压缩支持**：支持 INT8 / 4-bit 等常见量化方案，降低显存占用与推理成本。
- **可观测性与监控**：集成日志、指标采集与链路追踪，便于定位瓶颈和进行容量规划。
- **弹性扩缩容**：支持基于流量和资源指标的自动扩缩容，提高集群利用率。
- **面向生产环境设计**：强调稳定性、可维护性、安全隔离与版本化发布流程。

## 安装说明

### 环境要求

- Python 3.10+
- Docker 24+
- NVIDIA Driver / CUDA（如使用 GPU）
- Kubernetes 1.26+（可选）
- `git`, `make`

### 1. 克隆项目

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
```

### 2. 创建虚拟环境并安装依赖

```bash
python -m venv .venv
source .venv/bin/activate   # Linux / macOS
# .venv\Scripts\activate    # Windows

pip install -U pip
pip install -r requirements.txt
```

### 3. 可选：安装开发依赖

```bash
pip install -r requirements-dev.txt
pre-commit install
```

### 4. 使用 Docker 启动

```bash
docker build -t soloai/llm-opt:latest .
docker run --gpus all -p 8080:8080 soloai/llm-opt:latest
```

## 快速开始

下面示例展示如何启动一个基础的 LLM 推理服务，并启用常见优化参数。

```bash
python serve.py \
  --model ./models/your-llm \
  --host 0.0.0.0 \
  --port 8080 \
  --max-batch-size 16 \
  --dtype float16 \
  --kv-cache \
  --quantization int8
```

服务启动后，可通过 HTTP API 发起推理请求：

```bash
curl -X POST "http://localhost:8080/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "your-llm",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user", "content": "请总结一下 LLM 部署优化的关键点。"}
    ],
    "temperature": 0.2,
    "max_tokens": 256
  }'
```

## 使用示例

### 示例 1：启用量化部署

适用于显存资源受限、希望降低推理成本的场景。

```bash
python serve.py \
  --model ./models/your-llm \
  --quantization 4bit \
  --dtype float16 \
  --max-batch-size 8
```

### 示例 2：高吞吐批处理推理

适用于内部工具、AI Agent 平台、客服问答等高并发业务。

```bash
python serve.py \
  --model ./models/your-llm \
  --max-batch-size 64 \
  --scheduler dynamic \
  --tensor-parallel-size 2
```

### 示例 3：Kubernetes 部署

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: llm-serve
spec:
  replicas: 2
  selector:
    matchLabels:
      app: llm-serve
  template:
    metadata:
      labels:
        app: llm-serve
    spec:
      containers:
        - name: llm-serve
          image: soloai/llm-opt:latest
          ports:
            - containerPort: 8080
          env:
            - name: MODEL_PATH
              value: /models/your-llm
            - name: MAX_BATCH_SIZE
              value: "32"
          resources:
            limits:
              nvidia.com/gpu: 1
```

应用部署：

```bash
kubectl apply -f deploy/k8s/llm-serve.yaml
```

### 示例 4：Python SDK 调用

```python
import requests

payload = {
    "model": "your-llm",
    "messages": [
        {"role": "user", "content": "Explain the benefits of quantized LLM inference."}
    ],
    "temperature": 0.1,
    "max_tokens": 128,
}

resp = requests.post("http://localhost:8080/v1/chat/completions", json=payload, timeout=30)
print(resp.json())
```
