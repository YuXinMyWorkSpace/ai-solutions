## Overview

**LLM 部署优化** is a practical framework and toolkit focused on optimizing the deployment lifecycle of large language models. It helps engineering teams standardize model serving, reduce latency, improve throughput, control infrastructure cost, and simplify scalable production rollout.

This project is suitable for teams working on:

- LLM inference serving
- GPU/CPU deployment optimization
- Quantization and runtime tuning

- Production AI platform engineering

## Installation

### Prerequisites

- Python 3.10+
- `pip` or `poetry`
- Docker (optional, recommended)
- NVIDIA GPU + CUDA runtime (optional, for accelerated inference)

### Option 1: Install from source

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
python -m venv .venv
source .venv/bin/activate  # Linux/macOS
# .venv\Scripts\activate   # Windows

pip install --upgrade pip
pip install -r requirements.txt
```

### Option 2: Install in editable mode

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
pip install -e .
```

### Option 3: Docker

```bash
docker build -t soloai/llm-deployment-optimization:latest .
docker run --rm -p 8080:8080 soloai/llm-deployment-optimization:latest
```

## Quick Start

Start a local optimized inference service:

```bash
python -m app.server \
  --model ./models/your-llm \
  --host 0.0.0.0 \
  --port 8080 \
  --device cuda \
  --max-batch-size 8 \
  --dtype float16
```

Test the API:

```bash
curl -X POST "http://localhost:8080/v1/generate" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Explain KV cache optimization for transformer inference.",
    "max_tokens": 128,
    "temperature": 0.2
  }'
```

Example response:

```json
{
  "id": "resp_001",
  "model": "your-llm",
  "output": "KV cache optimization reduces repeated computation during autoregressive decoding...",
  "latency_ms": 142
}
```

## Usage

### 1. Run with a configuration file

```bash
python -m app.server --config configs/production.yaml
```

Example configuration:

```yaml
server:
  host: 0.0.0.0
  port: 8080

model:
  path: ./models/your-llm
  device: cuda
  dtype: float16

optimization:
  max_batch_size: 16
  tensor_parallel: 2
  enable_kv_cache: true
  quantization: int8

observability:
  metrics: true
  healthcheck: true
  structured_logging: true
```

### 2. Python API example

```python
from app.engine import InferenceEngine

engine = InferenceEngine(
    model_path="./models/your-llm",
    device="cuda",
    dtype="float16",
    max_batch_size=8,
    quantization="int8"
)

result = engine.generate(
    prompt="Summarize best practices for low-latency LLM serving.",
    max_tokens=100,
    temperature=0.3
)

print(result)
```

### 3. Health check and metrics

```bash
curl http://localhost:8080/health
curl http://localhost:8080/metrics
```

### 4. Example optimization workflow

```bash
python scripts/profile_inference.py --model ./models/your-llm --device cuda
python scripts/benchmark.py --config configs/benchmark.yaml
python scripts/export_runtime.py --model ./models/your-llm --quantization int8
```
