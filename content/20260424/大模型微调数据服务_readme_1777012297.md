## 快速开始

下面示例展示了一个典型的微调数据处理流程：读取原始对话数据、清洗文本、格式化为训练样本，并输出为 JSONL。

### 示例输入

```json
[
  {
    "id": "001",
    "instruction": "解释什么是检索增强生成（RAG）",
    "input": "",
    "output": "RAG 是一种将信息检索与大语言模型生成结合的方法..."
  }
]
```

### Python 快速示例

```python
import json

raw_data = [
    {
        "id": "001",
        "instruction": "解释什么是检索增强生成（RAG）",
        "input": "",
        "output": "RAG 是一种将信息检索与大语言模型生成结合的方法..."
    }
]

def normalize_record(record):
    return {
        "id": record["id"],
        "messages": [
            {"role": "user", "content": record["instruction"]},
            {"role": "assistant", "content": record["output"]}
        ]
    }

processed = [normalize_record(item) for item in raw_data]

with open("train.jsonl", "w", encoding="utf-8") as f:
    for item in processed:
        f.write(json.dumps(item, ensure_ascii=False) + "\n")

print("训练数据已生成: train.jsonl")
```

### 输出示例

```json
{"id":"001","messages":[{"role":"user","content":"解释什么是检索增强生成（RAG）"},{"role":"assistant","content":"RAG 是一种将信息检索与大语言模型生成结合的方法..."}]}
```

## 使用示例

### 1. 构建 SFT 指令微调数据

适用于通用问答、行业助手、客服机器人、知识助手等任务。

```json
{
  "instruction": "请总结以下合同条款的主要风险点",
  "input": "合同内容：......",
  "output": "主要风险点包括：1. 付款条件不明确；2. 违约责任界定不足；3. 争议解决条款存在歧义。"
}
```

---

### 2. 构建多轮对话数据

适用于对话机器人、业务助手、Copilot、Agent 等场景。

```json
{
  "id": "dialog_1001",
  "messages": [
    {"role": "system", "content": "你是一名专业的金融客服助手。"},
    {"role": "user", "content": "信用卡逾期会有什么影响？"},
    {"role": "assistant", "content": "信用卡逾期可能会对征信、利息和违约金产生影响。"},
    {"role": "user", "content": "逾期三天也会上征信吗？"},
    {"role": "assistant", "content": "是否上征信取决于发卡机构政策及宽限期规则，建议以银行公告为准。"}
  ]
}
```

---

### 3. 构建 DPO / 偏好优化数据

```json
  "chosen": "尊敬的王总，您好！感谢您昨日的沟通，我们整理了项目建议书...",

}
```

### 4. 数据清洗与脱敏示例

适用于隐私合规、企业知识入库、客服数据处理等任务。

```python
import re

def mask_sensitive(text: str) -> str:
    text = re.sub(r'1[3-9]\d{9}', '[PHONE]', text)
    text = re.sub(r'[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}', '[EMAIL]', text)
    return text

sample = "请联系张三，手机号13800138000，邮箱test@example.com"
print(mask_sensitive(sample))
# 输出: 请联系张三，手机号[PHONE]，邮箱[EMAIL]
```

## 最佳实践

- 明确训练目标：先定义 SFT、DPO、评测或 RAG 数据目标，再设计采样策略
- 统一标注规范：保证跨标注员、跨批次数据的一致性
- 优先质量而非数量：高质量、小而精的数据集通常优于低质量大规模堆叠
- 强化脱敏合规：对私域、客服、医疗、金融等数据进行严格脱敏处理
- 引入多层质检：建议结合规则、模型与人工审核三层质检机制
- 设计验收标准：提前定义准确率、一致性、覆盖率、格式合规率等指标
