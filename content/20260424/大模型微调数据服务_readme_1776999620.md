## 项目简介

**SoloAI 大模型微调数据服务**旨在为企业、研究团队和 AI 产品团队提供可复用、可扩展、可审计的数据处理能力。  
围绕大模型训练与微调全链路，项目覆盖数据采集、去重清洗、质量评估、指令构造、对话格式化、训练集导出等关键环节。

适用场景包括但不限于：

- 行业垂直大模型数据构建
- 企业知识库问答数据集生成
- 客服 / 销售 / 医疗 / 法务 / 教育场景对话数据整理
- 指令微调（Instruction Tuning / SFT）
- RLHF / DPO 偏好数据预处理
- RAG 评测集与训练样本准备

---

## 核心特性

- **多源数据接入**：支持文本、FAQ、知识库、网页、表格、工单、客服记录等多种数据来源。
- **标准化清洗流程**：内置去重、脱敏、格式修正、噪声过滤、字段归一化等步骤。
- **面向微调的数据结构化**：支持生成 instruction / input / output、多轮对话、偏好对、问答对等格式。
- **质量控制与可审计**：支持规则校验、抽样检查、质量标签、处理日志与版本追踪。
- **灵活导出**：支持 JSON / JSONL / CSV 等训练常用格式，便于接入主流 LLM 微调框架。
- **适配行业场景**：可针对金融、教育、电商、医疗、政务、制造等业务领域进行定制化处理。

## 安装说明

### 环境要求

- Python 3.10+
- pip 23+
- 推荐使用虚拟环境

### 1. 克隆项目

```bash
git clone https://github.com/SoloAI/llm-finetune-data-service.git
cd llm-finetune-data-service
```

### 2. 创建虚拟环境并安装依赖

```bash
python -m venv .venv
source .venv/bin/activate
```

Windows:

```bash
.venv\Scripts\activate
```

安装依赖：

```bash
pip install -U pip
pip install -r requirements.txt
```

### 3. 配置环境变量

复制示例配置文件：

```bash
cp .env.example .env
```

示例：

```env
INPUT_DIR=./data/raw
OUTPUT_DIR=./data/processed
LOG_LEVEL=INFO
EXPORT_FORMAT=jsonl
```

## 快速开始

下面示例展示如何将原始问答数据清洗并导出为适合大模型监督微调（SFT）的 JSONL 格式。

### 1. 准备输入数据

`data/raw/sample.json`：

```json
[
  {
    "question": "如何申请退款？",
    "answer": "您可以在订单详情页提交退款申请，审核通过后系统将原路退回。"
  },
  {
    "question": "发票如何开具？",
    "answer": "请在下单时填写发票信息，订单完成后可申请电子发票。"
  }
]
```

### 2. 运行处理命令

```bash
python -m app.cli process \
  --input ./data/raw/sample.json \
  --task sft \
  --output ./data/processed/sft_dataset.jsonl
```

### 3. 输出示例

```json
{"instruction":"请回答以下问题","input":"如何申请退款？","output":"您可以在订单详情页提交退款申请，审核通过后系统将原路退回。"}
{"instruction":"请回答以下问题","input":"发票如何开具？","output":"请在下单时填写发票信息，订单完成后可申请电子发票。"}
```

## 使用示例

### 示例 1：将 FAQ 转换为 SFT 数据

```bash
python -m app.cli process \
  --input ./data/raw/faq.csv \
  --task sft \
  --mapping question=question,answer=answer \
  --output ./data/processed/faq_sft.jsonl
```

### 示例 2：构建多轮对话数据

```bash
python -m app.cli process \
  --input ./data/raw/dialogues.json \
  --task conversation \
  --output ./data/processed/multi_turn.jsonl
```

输出示例：

```json
{
  "messages": [
    {"role": "system", "content": "你是一名专业客服助手。"},
    {"role": "user", "content": "订单多久发货？"},
    {"role": "assistant", "content": "通常在付款后 24 小时内发货。"}
  ]
}
```

### 示例 3：数据清洗与脱敏

```bash
python -m app.cli clean \
  --input ./data/raw/customer_service.xlsx \
  --remove-duplicates \
  --mask-phone \
  --mask-email \
  --output ./data/processed/customer_service_clean.json
```

### 示例 4：导出偏好对数据

适用于 DPO / RLHF 数据准备：

```bash
python -m app.cli process \
  --input ./data/raw/preferences.json \
  --task preference \
  --output ./data/processed/preference_pairs.jsonl
```

输出示例：

```json
{
  "prompt": "用户询问如何重置密码",
  "chosen": "您可以在登录页点击“忘记密码”，通过短信或邮箱验证后完成重置。",
  "rejected": "请联系管理员。"
}
```

### 示例 5：通过 Python API 调用

```python
from app.pipeline import DataPipeline

pipeline = DataPipeline(
    input_path="./data/raw/sample.json",
    output_path="./data/processed/output.jsonl",
    task="sft",
    export_format="jsonl"
)

pipeline.run()
print("Dataset exported successfully.")
```
