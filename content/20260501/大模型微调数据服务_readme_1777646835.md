# SoloAI 大模型微调数据服务

[![Build Status](https://img.shields.io/badge/build-passing-brightgreen)](./)
[![License](https://img.shields.io/badge/license-MIT-blue.svg)](./LICENSE)

**SoloAI 大模型微调数据服务** is an open-source project focused on delivering scalable, high-quality data pipelines for large language model fine-tuning. It helps teams standardize data collection, cleaning, transformation, validation, and export workflows for LLM training, instruction tuning, and domain adaptation.

---

## Overview

Modern LLM fine-tuning depends heavily on clean, structured, and domain-relevant datasets. SoloAI provides a practical framework for building **fine-tuning data services for large models**, enabling organizations to prepare supervised fine-tuning (SFT), preference, conversation, and task-specific datasets efficiently and reliably.

## Features

- **Multi-source data ingestion**  
  Import data from JSON, JSONL, CSV, XLSX, APIs, internal knowledge bases, and annotation platforms.

- **Dataset cleaning and normalization**  
  Standardize formats, remove duplicates, repair malformed samples, and normalize multilingual content.

- **Fine-tuning schema conversion**  
  Convert raw data into training-ready formats for instruction tuning, chat tuning, Q&A, classification, and custom tasks.

- **Quality validation pipeline**  
  Apply rule-based checks, schema validation, content filtering, and consistency scoring before export.

- **Scalable export workflow**  
  Export processed datasets to JSONL and other common formats compatible with mainstream LLM training frameworks.

- **Extensible service architecture**  
  Modular design allows teams to integrate custom preprocessors, validators, and domain-specific adapters.

## Quick Start

Prepare a source dataset and run a basic processing pipeline:

```bash
python -m soloai.pipeline \
  --input ./data/raw/customer_support.jsonl \
  --output ./data/processed/customer_support_sft.jsonl \
  --task sft \
  --deduplicate \
  --validate \
  --normalize
```

Example output:

```text
[INFO] Loaded 12,480 records
[INFO] Removed 532 duplicate samples
[INFO] Normalized 11,948 records
[INFO] Validation passed: 11,721 records
[INFO] Exported dataset to ./data/processed/customer_support_sft.jsonl
```

## Usage

### 1. Python API Example

```python
from soloai.pipeline import DataPipeline

pipeline = DataPipeline(
    task="sft",
    deduplicate=True,
    normalize=True,
    validate=True
)

pipeline.run(
    input_path="./data/raw/domain_data.jsonl",
    output_path="./data/processed/domain_sft.jsonl"
)
```

### 2. Convert Raw Records to Chat Fine-Tuning Format

Input:

```json
{
  "instruction": "解释什么是向量数据库",
  "input": "",
  "output": "向量数据库是一种专门用于存储、索引和检索高维向量数据的数据库系统。"
}
```

Converted output:

```json
{
  "messages": [
    {
      "role": "system",
      "content": "You are a helpful AI assistant."
    },
    {
      "role": "user",
      "content": "解释什么是向量数据库"
    },
    {
      "role": "assistant",
      "content": "向量数据库是一种专门用于存储、索引和检索高维向量数据的数据库系统。"
    }
  ]
}
```

### 3. CLI Example for Validation Only

```bash
python -m soloai.validate \
  --input ./data/processed/domain_sft.jsonl \
  --schema chat \
  --report ./reports/validation_report.json
```

### 4. Example Configuration

```yaml
project:
  name: medical-qa-finetune
  task: sft

input:
  path: ./data/raw/medical_qa.jsonl
  format: jsonl

processing:
  deduplicate: true
  normalize_unicode: true
  remove_empty_fields: true
  language_filter: zh

validation:
  schema: chat
  min_turns: 2
  max_length: 4096
  profanity_filter: true

output:
  path: ./data/processed/medical_qa_chat.jsonl
  format: jsonl
```

Run with config:

```bash
python -m soloai.pipeline --config ./configs/medical_qa.yaml
```
