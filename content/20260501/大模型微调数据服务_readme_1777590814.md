# 🤖 SoloAI LLM Fine-Tuning Data Service
![Build Status](https://github.com/SoloAI/llm-finetune-data-service/actions/workflows/ci.yml/badge.svg) ![License](https://img.shields.io/badge/License-Apache%202.0-blue.svg) ![Python](https://img.shields.io/badge/Python-3.9%2B-3776AB)

SoloAI LLM Fine-Tuning Data Service (大模型微调数据服务) is an enterprise-grade pipeline for generating, curating, and formatting high-quality instruction-tuning datasets for large language models. It automates the end-to-end SFT data workflow, enabling researchers and ML engineers to accelerate model alignment with reproducible, production-ready datasets.

---

## ✨ Key Features
- **🧠 LLM-Driven Data Synthesis**: Automatically generate domain-specific instruction-response pairs using configurable prompt templates and multi-agent validation.
- **🔍 Semantic & Lexical Deduplication**: Hybrid filtering pipeline combining MinHash, SimHash, and embedding-based cosine similarity to eliminate redundancy while preserving diversity.
- **📦 Multi-Format Export**: Native support for `JSONL`, `Parquet`, `Alpaca`, `ShareGPT`, and Hugging Face `Dataset` formats with schema validation.
- **⚡ Distributed Processing**: Built on Ray/Dask for horizontal scaling across CPU/GPU clusters with intelligent caching and fault tolerance.
- **🛡️ Quality Guardrails**: Automated toxicity detection, PII redaction, hallucination scoring, and length/complexity balancing for RLHF/DPO readiness.
- **🔌 Framework Agnostic**: Seamless drop-in integration with `transformers`, `trl`, `vLLM`, and `Unsloth` training pipelines.

## 🚀 Quick Start

```python
from soloai.data import FineTunePipeline

# Initialize pipeline with target domain and output schema
pipeline = FineTunePipeline(
    domain="medical",
    base_model="Qwen/Qwen2.5-7B-Instruct",
    output_format="sharegpt",
    enable_semantic_dedup=True
)

# Generate & curate dataset
dataset = pipeline.generate(
    seed_corpus="raw_medical_qa.json",
    target_samples=5000,
    quality_threshold=0.85
)

# Export to training-ready format
dataset.save("output/medical_sft_v1.jsonl")
print(f"✅ Curated {len(dataset)} samples. Avg length: {dataset.avg_tokens} tokens.")
```

## 🛠️ Usage Examples

### 1. Custom Template Engine & Multi-Source Ingestion
```python
from soloai.data import TemplateRegistry, DataIngestor

# Register domain-specific prompt structure
TemplateRegistry.register("finance_risk", {
    "system": "You are a senior risk analyst. Provide structured, compliant advice.",
    "user": "Context: {context}\nQuestion: {query}",
    "assistant": "{response}"
})

# Ingest mixed sources (CSV, JSON, HF Hub)
ingestor = DataIngestor(sources=["./reports.csv", "huggingface:finance/qa-bank"])
raw_data = ingestor.load()

# Apply template & generate
pipeline = FineTunePipeline(template="finance_risk")
curated = pipeline.transform(raw_data, batch_size=32)
```

### 2. Hugging Face Dataset Export & Validation
```python
from datasets import Dataset as HFDataset

# Convert to HF format with automatic schema mapping
hf_ds = curated.to_huggingface()

# Push to Hub with metadata & tags
hf_ds.push_to_hub(
    "SoloAI/finance-sft-v1",
    token="your_hf_token",
    tags=["sft", "finance", "soloai", "llm-data"]
)
```

### 3. Integration with TRL / SFT Trainer
```python
from trl import SFTTrainer
from soloai.data import load_dataset

train_ds = load_dataset("output/medical_sft_v1.jsonl", split="train")

trainer = SFTTrainer(
    model="meta-llama/Llama-3.1-8B",
    train_dataset=train_ds,
    dataset_text_field="text",
    max_seq_length=4096
)
trainer.train()
```
