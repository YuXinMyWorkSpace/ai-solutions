## 📖 Overview
SoloAI's **LLM Fine-Tuning Data Service** is an open-source, production-ready pipeline for generating, curating, and validating high-quality instruction datasets for large language models. Designed for SFT, DPO, and RLHF workflows, it automates data augmentation, semantic deduplication, and format standardization to accelerate model alignment across enterprise domains.

## ✨ Key Features
- **Automated Dataset Generation & Augmentation**: LLM-driven instruction synthesis with controllable diversity, difficulty scaling, and domain-specific prompt templates.
- **Multi-Format Export & Integration**: Native support for JSONL, Parquet, Arrow, and Hugging Face `datasets` format with zero-config compatibility for `transformers`, `trl`, and `axolotl`.
- **Intelligent Quality Filtering**: Built-in semantic deduplication, toxicity screening, length normalization, and LLM-as-a-judge scoring pipelines.
- **Full Fine-Tuning Paradigm Support**: Unified data schema for Supervised Fine-Tuning (SFT), Direct Preference Optimization (DPO), and Reward Modeling (RM).
- **Distributed & Scalable Processing**: Ray-backed parallel execution with memory-efficient chunking, enabling billion-token corpus processing on standard cloud instances.
- **Reproducible Data Versioning**: Integrated dataset fingerprinting, seed locking, and audit trails for compliance and experiment tracking.

## 🚀 Quick Start
Generate a ready-to-train SFT dataset in under 5 minutes:
```python
from soloai.data import FineTunePipeline

# Initialize pipeline with target model & quality constraints
pipeline = FineTunePipeline(
    base_model="meta-llama/Llama-3-8B",
    output_format="jsonl",
    min_quality_score=0.85
)

# Generate domain-specific instruction data
dataset = pipeline.generate(
    domain="healthcare_clinical",
    num_samples=5000,
    seed=42,
    api_endpoint="http://localhost:8000/v1"  # Optional local LLM endpoint
)

# Export to training directory
dataset.save("output/clinical_sft_dataset.jsonl")
print(f"✅ Saved {len(dataset)} validated instruction pairs.")
```

## 📖 Usage Examples

### 1. Advanced Data Cleaning & DPO Pair Generation
```python
from soloai.data import DataProcessor, QualityFilter, DPOFormatter

# Load raw conversational logs
raw_data = DataProcessor.load_parquet("raw_chat_logs.parquet")

# Apply semantic deduplication & safety filters
cleaned = QualityFilter(
    strategy="sentence_transformers",
    similarity_threshold=0.82,
    block_toxicity=True,
    remove_pii=True
).apply(raw_data)

# Convert to preference pairs (chosen/rejected)
dpo_data = DPOFormatter(
    scoring_model="gpt-4o-mini",
    preference_strategy="length_controlled_quality"
).convert(cleaned)

dpo_data.export("output/dpo_pairs.jsonl")
```

### 2. CLI Workflow for Batch Processing
```bash
# Generate, filter, and export in one command
soloai generate \
  --config configs/finance_sft.yaml \
  --input data/raw_faq.csv \
  --output datasets/finance_sft_v1.jsonl \
  --workers 8 \
  --validate
```
