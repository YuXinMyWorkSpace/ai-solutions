**SoloAI LLM Deployment Optimizer** is a production-grade inference acceleration and orchestration framework engineered to reduce latency, maximize throughput, and minimize hardware costs for large language model deployments. It delivers up to 3× faster token generation through dynamic quantization, kernel fusion, and intelligent multi-device scheduling.

## 📦 Installation

**Prerequisites:** Python 3.9+, CUDA 12.1+ (for GPU acceleration), `pip` or `conda`

```bash
# Install via PyPI
pip install soloai-llm-opt

# Install with GPU extensions
pip install soloai-llm-opt[gpu]

# Docker deployment (recommended for production)
docker pull soloai/llm-deploy-opt:latest
```

> 💡 For ROCm or Apple Silicon builds, refer to the `docs/platform-support.md` guide.

## ⚡ Quick Start

```python
from soloai_opt import LLMEngine, QuantizationConfig, SchedulerConfig

# Initialize optimized engine
engine = LLMEngine(
    model_path="meta-llama/Llama-3-8B-Instruct",
    quantization=QuantizationConfig(bits=8, method="awq"),
    scheduler=SchedulerConfig(tensor_parallel=2, max_batch_size=64)
)

# Synchronous inference
output = engine.generate(
    prompt="Explain transformer attention mechanisms in 3 bullet points:",
    max_tokens=256,
    temperature=0.7
)
print(output)
```

## 🛠️ Usage Examples

### 1. Configuration-Driven Deployment
```yaml
# config.yaml
model:
  path: mistralai/Mixtral-8x7B-Instruct-v0.1
  dtype: bfloat16
optimization:
  quantization: int4_gptq
  kernel: flash_attn_2
  memory_pool: paged_attention
serving:
  port: 8080
  max_concurrent: 128
  enable_streaming: true
```
```bash
soloai-opt serve --config config.yaml --workers 4
```

### 2. Streaming & Async API
```python
import asyncio
from soloai_opt import AsyncLLMEngine

async def stream_response():
    engine = AsyncLLMEngine("Qwen/Qwen2.5-7B-Instruct")
    async for token in engine.generate_stream("Write a Python function to sort a list:"):
        print(token, end="", flush=True)

asyncio.run(stream_response())
```

### 3. Prometheus Metrics Integration
```bash
# Expose metrics at /metrics
soloai-opt serve --config config.yaml --enable-metrics
curl http://localhost:8080/metrics
# Output: soloai_tokens_generated_total, soloai_kv_cache_usage_bytes, etc.
```
