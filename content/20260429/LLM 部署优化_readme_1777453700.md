## 📖 Overview
SoloAI's **LLM Deployment Optimization Framework** delivers production-grade, low-latency inference for large language models across heterogeneous compute environments. By integrating dynamic request routing, memory-aware KV-cache management, and hardware-accelerated kernel fusion, it maximizes throughput while minimizing GPU memory footprint and operational overhead.

## ✨ Key Features
- 🔹 **Dynamic Batching & Speculative Decoding** – Adaptive request grouping with draft-verify pipelines to reduce per-token latency by up to 3.2×
- 🔹 **Quantization-Aware Serving** – Native INT4/INT8/FP4 support with calibration-free weight-only quantization and seamless fallback
- 🔹 **Multi-GPU & Distributed Tensor Parallelism** – Zero-copy NCCL communication, pipeline parallelism, and automatic topology discovery
- 🔹 **PagedAttention & KV-Cache Optimization** – Fragmentation-free memory management enabling 40–60% higher concurrent context windows
- 🔹 **Hardware-Agnostic Runtime** – Unified backend supporting NVIDIA CUDA, AMD ROCm, Intel CPU (OpenVINO), and Apple Silicon
- 🔹 **Production Observability** – Built-in Prometheus metrics, OpenTelemetry tracing, and auto-scaling hooks for Kubernetes/Slurm

## 📦 Installation
Requires Python 3.10+ and a compatible CUDA/ROCm driver.

```bash
# 1. Create isolated environment
python -m venv .venv && source .venv/bin/activate

# 2. Install core package + CUDA 12.x wheels
pip install --upgrade pip
pip install soloai-llm-deploy-opt[cuda12]

# Optional: Install ROCm or CPU-only variants
# pip install soloai-llm-deploy-opt[rocm]
# pip install soloai-llm-deploy-opt[cpu]
```

> 💡 For enterprise deployments, we recommend using the provided `Dockerfile` and Helm charts available in `/deploy/`.

## ⚡ Quick Start
Launch an optimized inference server with a single command:

```python
from soloai.deploy import OptimizedLLMServer, QuantizationConfig, ServingConfig

# Configure hardware-aware optimizations
config = ServingConfig(
    model="meta-llama/Meta-Llama-3.1-8B-Instruct",
    quantization=QuantizationConfig(dtype="int4", method="awq"),
    max_batch_size=128,
    kv_cache_dtype="fp8",
    tensor_parallel_size=2,
    port=8000
)

server = OptimizedLLMServer(config)
server.start()
```

Verify the endpoint:
```bash
curl -X POST http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model": "llama-3.1-8b", "messages": [{"role": "user", "content": "Explain PagedAttention."}], "max_tokens": 256}'
```

## 🛠️ Usage Examples

### 📊 YAML-Driven Production Deployment
```yaml
# deploy/production.yaml
runtime:
  backend: vllm_compatible
  device_map: auto
  enable_paged_attention: true

serving:
  max_concurrent_requests: 512
  timeout: 30s
  rate_limit:
    enabled: true
    tokens_per_second: 10000

observability:
  metrics_exporter: prometheus
  log_level: INFO
  trace_sampling_rate: 0.1
```

```python
from soloai.deploy import load_from_yaml

server = load_from_yaml("deploy/production.yaml")
server.serve()
```

### 🔄 Async Python Client with Streaming
```python
import asyncio
from soloai.client import AsyncLLMClient

async def stream_response():
    async with AsyncLLMClient("http://localhost:8000") as client:
        async for chunk in client.stream_chat(
            model="llama-3.1-8b",
            messages=[{"role": "user", "content": "Optimize this inference pipeline."}],
            temperature=0.7,
            max_tokens=512
        ):
            print(chunk.choices[0].delta.content, end="", flush=True)

asyncio.run(stream_response())
```
