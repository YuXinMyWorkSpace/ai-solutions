## 📖 Overview
A production-grade data pipeline and toolkit for generating, cleaning, validating, and formatting high-quality instruction-tuning datasets for Large Language Models. Streamline your AI training workflow from raw unstructured text to framework-ready fine-tuning data.

## ✨ Key Features
- 🧹 **Automated Data Cleaning & Deduplication**: Semantic hashing, regex filtering, and PII redaction to remove noise and near-duplicates at scale.
- 📐 **Multi-Format Conversion**: Native support for JSON, CSV, Parquet, and raw text with one-click export to Hugging Face `datasets`, Alpaca, and ChatML formats.
- ✅ **Quality Scoring & Validation**: Hybrid heuristic + LLM-assisted validators to score instruction clarity, response alignment, and schema compliance.
- ⚡ **Distributed Processing Engine**: Built on Ray/Dask for horizontal scaling across CPU/GPU clusters, enabling billion-token pipelines.
- 🔒 **Privacy-First Architecture**: Configurable on-prem execution, token-level anonymization, and audit logging for enterprise compliance (GDPR/HIPAA-ready).

## 🚀 Quick Start
Initialize a fine-tuning pipeline, load raw data, apply cleaning & validation, and export:

```python
from soloai.data import FineTunePipeline

# Configure pipeline for instruction-tuning format
pipeline = FineTunePipeline(
    format="chatml",
    validation_threshold=0.85,
    max_workers=8
)

# Load, clean, and validate
raw_ds = pipeline.load("raw_conversations.jsonl")
cleaned_ds = pipeline.clean(raw_ds)
validated_ds = pipeline.validate(cleaned_ds)

# Export to HuggingFace-ready format
validated_ds.to_hf("ft_dataset_v1")
print("✅ Dataset ready for SFT/LoRA training.")
```

## 🛠️ Usage Examples

### 1. Custom Validation Rules
Define domain-specific quality filters using the rule engine:
```python
from soloai.validators import RuleValidator

validator = RuleValidator()
validator.add_rule("min_tokens", lambda x: len(x.split()) >= 50)
validator.add_rule("no_code_blocks", lambda x: "```" not in x)

filtered = validator.apply(validated_ds, columns=["instruction", "response"])
```

### 2. Distributed Processing with Ray
Scale to large corpora with minimal code changes:
```python
from soloai.engine import RayPipeline

# Automatically distributes across available cluster nodes
ray_pipeline = RayPipeline.from_config("config/distributed.yaml")
result = ray_pipeline.process("s3://bucket/raw_data/", output_dir="processed/")
```

### 3. Synthetic Data Augmentation
Generate high-quality instruction-response pairs from seed documents:
```python
from soloai.augment import SyntheticGenerator

gen = SyntheticGenerator(model="qwen2.5-7b-instruct", temperature=0.7)
augmented = gen.generate_from_corpus("domain_docs/", n_samples=5000)
```
