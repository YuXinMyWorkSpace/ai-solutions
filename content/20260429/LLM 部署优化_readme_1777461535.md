# 🚀 SoloAI LLM Deployment Optimizer
![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/llm-deploy-optimizer/ci.yml?label=Build&style=flat-square) ![License](https://img.shields.io/badge/License-Apache%202.0-blue.svg) ![PyPI](https://img.shields.io/pypi/v/llm-deploy-optimizer?style=flat-square)

A production-grade inference optimization framework engineered to accelerate Large Language Model deployment across heterogeneous hardware. SoloAI's engine reduces end-to-end latency by up to 65% and cuts GPU memory overhead by 50% through dynamic KV-cache management, adaptive quantization, and continuous request batching.

## ✨ Key Features
- **Adaptive Quantization Pipeline**: Seamless INT4/INT8/FP8 switching with per-layer calibration and zero-accuracy degradation guarantees.
- **Dynamic Memory Orchestration**: Intelligent KV-cache eviction, paged attention, and unified memory pooling for multi-model serving.
- **Continuous & Speculative Batching**: Asynchronous request scheduling with draft-model speculative decoding for throughput scaling.
- **Hardware-Agnostic Runtime**: Native support for CUDA, ROCm, Intel CPU, and domestic NPUs with automatic kernel dispatch.
- **Real-Time Telemetry & Auto-Scaling**: OpenTelemetry-native metrics, Prometheus/Grafana dashboards, and Kubernetes HPA integration.
- **Framework Interoperability**: Drop-in compatibility with vLLM, TensorRT-LLM, and Hugging Face `transformers` pipelines.

## 📦 Installation
Requires Python 3.9+ and CUDA 11.8+ (or ROCm 5.7+).

```bash
# Install via PyPI (recommended)
pip install llm-deploy-optimizer

# Or install from source for custom kernel compilation
git clone https://github.com/SoloAI/llm-deploy-optimizer.git
cd llm-deploy-optimizer
pip install -e ".[cuda]"  # or ".[rocm]" / ".[cpu]"
```

> 💡 **Tip**: For enterprise deployments, we recommend using our pre-built Docker images: `docker pull soloai/llm-optimizer:latest`

## ⚡ Quick Start
Initialize an optimized inference server with minimal configuration:

```python
from soloai_llm import OptimizedEngine, InferenceConfig

config = InferenceConfig(
    model_id="meta-llama/Llama-3-8B-Instruct",
    quantization="int8",
    max_batch_size=128,
    enable_kv_cache_paging=True,
    speculative_decoding=True
)

engine = OptimizedEngine(config)
engine.start_serving(host="0.0.0.0", port=8000)
```

## 🛠 Usage Examples

### 1. REST API Consumption
```bash
curl -X POST http://localhost:8000/v1/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "meta-llama/Llama-3-8B-Instruct",
    "prompt": "Explain the attention mechanism in transformers.",
    "max_tokens": 256,
    "temperature": 0.7
  }'
```

### 2. Advanced Python Client with Streaming & Metrics
```python
import asyncio
from soloai_llm import AsyncClient, MetricsCollector

async def run_inference():
    client = AsyncClient("http://localhost:8000")
    metrics = MetricsCollector()

    async for chunk in client.stream_generate(
        prompt="Optimize this deployment pipeline for production.",
        max_new_tokens=512,
        stream=True
    ):
        print(chunk.text, end="", flush=True)
        
    print(f"\n📊 Metrics: {metrics.get_summary()}")

asyncio.run(run_inference())
```

### 3. Kubernetes Deployment (Helm)
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: llm-optimizer
spec:
  template:
    spec:
      containers:
      - name: inference
        image: soloai/llm-optimizer:latest
        env:
        - name: OPTIMIZER_QUANTIZATION
          value: "fp8"
        - name: OPTIMIZER_MAX_BATCH_SIZE
          value: "256"
        resources:
          limits:
            nvidia.com/gpu: 2
```
