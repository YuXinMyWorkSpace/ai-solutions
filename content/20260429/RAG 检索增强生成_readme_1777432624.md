SoloRAG is a production-ready Retrieval-Augmented Generation (检索增强生成) framework engineered by SoloAI to bridge enterprise knowledge bases with large language models. It delivers low-latency, context-accurate responses through hybrid retrieval, adaptive chunking, and built-in evaluation pipelines optimized for scalable AI deployments.

---

## ✨ Key Features
- 🔍 **Hybrid Semantic & Lexical Retrieval**: Combines dense vector embeddings with BM25 sparse search to maximize recall and precision across diverse corpora.
- 📐 **Adaptive Document Chunking**: Context-aware splitting with configurable overlap, metadata preservation, and hierarchical indexing for complex documents.
- 🔄 **Vector Database Agnostic**: Unified API supporting Milvus, Weaviate, Qdrant, Chroma, Pinecone, and FAISS with seamless backend switching.
- 📊 **Built-in RAG Evaluation & Observability**: Integrated RAGAS metrics, hallucination scoring, latency tracking, and prompt tracing out-of-the-box.
- ⚡ **Semantic Caching & Query Routing**: Multi-tier caching, intent classification, and fallback routing to guarantee sub-500ms response times in production.
- 🌐 **Multi-Format & Multi-Language Support**: Native parsers for PDF, DOCX, Markdown, HTML, and images with cross-lingual embedding compatibility.

## 🚀 Quick Start
Get a functional RAG pipeline running in under 10 lines of code.

```python
from solorag import RAGPipeline

# Initialize with default hybrid retriever & OpenAI-compatible LLM
rag = RAGPipeline(
    retriever_type="hybrid",
    llm_model="gpt-4o-mini",
    embedder="text-embedding-3-large",
    top_k=3
)

# Ingest local documents (supports PDF, TXT, MD, DOCX)
rag.ingest(["./docs/architecture.pdf", "./docs/api_reference.md"])

# Query the knowledge base
response = rag.query("How do I configure OAuth2 authentication for the API gateway?")
print("✅ Answer:", response.answer)
print("📚 Sources:", [src.metadata for src in response.sources])
```

## 🛠️ Usage Examples

### 1. Custom Chunking & Reranking Strategy
Fine-tune retrieval for domain-specific accuracy using cross-encoders and hierarchical splitting.

```python
from solorag import RAGConfig, RAGPipeline

config = RAGConfig(
    chunk_size=512,
    chunk_overlap=64,
    reranker_model="cross-encoder/ms-marco-MiniLM-L-6-v2",
    top_k=5,
    enable_hierarchical_indexing=True
)

rag = RAGPipeline(config=config)
rag.ingest("./data/legal_contracts/")
result = rag.query("What are the termination clauses for enterprise SLAs?")
```

### 2. Production Evaluation & Benchmarking
Measure factual consistency, context relevance, and answer faithfulness before deployment.

```python
from solorag import Evaluator

evaluator = Evaluator(pipeline=rag)
report = evaluator.run(
    dataset="benchmarks/qa_golden_set.json",
    metrics=["faithfulness", "context_precision", "answer_relevance"]
)
print(report.to_dataframe())
report.export_html("evaluation_report.html")
```

### 3. Deploy as REST API
Expose your RAG pipeline via FastAPI with built-in streaming and telemetry.

```python
from solorag.serve import RAGServer

server = RAGServer(pipeline=rag, port=8000, enable_streaming=True)
server.run()
# POST /query {"prompt": "Explain the deployment architecture"}
```
