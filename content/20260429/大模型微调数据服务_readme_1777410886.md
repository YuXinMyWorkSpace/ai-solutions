An enterprise-grade, open-source data pipeline for large language model (LLM) fine-tuning. SoloAI FineTuneData automates dataset generation, semantic cleaning, instruction formatting, and quality benchmarking to accelerate model training with high-fidelity, domain-specific data.

---

## ✨ Key Features

- **📐 Multi-Format Instruction Support**: Native export to ShareGPT, Alpaca, JSONL, and Parquet with schema validation.
- **🧹 Automated Cleaning & Deduplication**: MinHash/SimHash LSH deduplication, semantic similarity filtering, and length/regex-based sanitization.
- **📊 Quality Scoring & LLM-as-a-Judge**: Built-in metrics for instruction following, factual consistency, toxicity, and format compliance.
- **🌍 Domain-Adaptive Synthetic Generation**: RAG-enhanced prompt templating and iterative self-refinement loops for vertical-specific datasets.
- **⚡ Distributed & GPU-Accelerated Pipeline**: Ray-backed parallel processing with optional vLLM/Transformers backend integration.
- **🔌 Framework Agnostic**: Seamless interoperability with Hugging Face `datasets`, Axolotl, Unsloth, and vLLM fine-tuning workflows.

## 🚀 Quick Start

```python
from solofinetune import DataPipeline, QualityEvaluator

# Initialize pipeline with source and target schema
pipeline = DataPipeline(
    source="raw_corpus.jsonl",
    target_format="sharegpt",
    quality_threshold=0.85,
    dedup_strategy="minhash_lsh"
)

# Run cleaning, formatting, and scoring
dataset = pipeline.run()

# Export training-ready dataset
dataset.save("output/fine_tune_ready.jsonl")
print(f"✅ Processed {len(dataset)} high-quality instruction pairs.")
```

## 📖 Usage Examples

### 1. YAML Configuration for Production Pipelines
```yaml
# config.yaml
pipeline:
  cleaning:
    min_length: 50
    max_length: 4096
    dedup_method: "semantic_simhash"
    language_filter: ["en", "zh"]
  generation:
    strategy: "rag_synthetic"
    llm_backend: "vllm"
    model_name: "Qwen2.5-7B-Instruct"
    prompt_template: "domain_expert_v3"
  evaluation:
    metrics: ["instruction_following", "factual_consistency", "format_compliance"]
    judge_model: "gpt-4o-mini"
    parallel_workers: 8
```

### 2. CLI Execution
```bash
solofinetune run \
  --config config.yaml \
  --input-dir ./raw_data/ \
  --output-dir ./datasets/ \
  --log-level INFO
```

### 3. Programmatic Quality Filtering
```python
from solofinetune import QualityEvaluator

evaluator = QualityEvaluator(
    judge_backend="openai",
    scoring_schema="rubric_v2"
)

scored_dataset = evaluator.score(dataset)
filtered_dataset = scored_dataset.filter(lambda row: row["quality_score"] >= 0.8)
```
