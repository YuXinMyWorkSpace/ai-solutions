# 🤖 SoloAI LLM Fine-Tuning Data Service

![Build Status](https://img.shields.io/badge/Build-Passing-brightgreen) ![License](https://img.shields.io/badge/License-MIT-blue.svg) ![Python](https://img.shields.io/badge/Python-3.9%2B-blue) ![Version](https://img.shields.io/badge/Version-1.2.0-orange)

A high-performance, enterprise-grade data pipeline and service platform engineered specifically for Large Language Model (LLM) fine-tuning. SoloAI automates raw corpus ingestion, semantic deduplication, quality scoring, and format standardization, delivering production-ready instruction-tuning datasets optimized for modern transformer architectures (LoRA, QLoRA, SFT, DPO).

---

## ✨ Key Features

| Feature | Description |
|---------|-------------|
| 🔍 **Intelligent Deduplication** | MinHash + SimHash + semantic embedding clustering to remove near-duplicates while preserving domain diversity |
| 📊 **LLM-as-a-Judge Quality Scoring** | Automated heuristic & model-based evaluation for instruction clarity, factual accuracy, and toxicity filtering |
| 🔄 **Multi-Format Standardization** | Seamless conversion between `Alpaca`, `ShareGPT`, `OpenAI ChatML`, `JSONL`, and `Parquet` schemas |
| ⚡ **Distributed Processing Engine** | Built on Ray & Dask for horizontal scaling across multi-GPU/cluster environments |
| 🌐 **Hugging Face & vLLM Ready** | Native `datasets` library integration with push-to-hub, streaming, and tokenization-aware chunking |
| 🛡️ **Data Provenance & Compliance** | Audit trails, PII redaction, and license-aware filtering for enterprise-grade governance |

## 📦 Installation

Requires Python 3.9+ and `pip`.

```bash

cd llm-finetune-data-service

# Install core package + optional distributed backend
pip install -e ".[core]"
# For Ray/Dask distributed processing:
pip install -e ".[distributed]"

# Verify installation

```

## 🚀 Quick Start

Initialize a minimal data pipeline, process raw JSONL, and export to Hugging Face Hub:

```python
from soloai.ft_data import DataPipeline

# 1. Configure pipeline
pipeline = DataPipeline(
    input_path="data/raw_corpus.jsonl",
    output_format="sharegpt",
    max_seq_len=4096,
    dedup_strategy="minhash_semantic"
)

# 2. Execute processing (clean → deduplicate → format)
dataset = pipeline.run()

# 3. Push to HF Hub or save locally
dataset.save_to_hub("soloai/domain-finetune-v1")
dataset.to_jsonl("data/processed/train.jsonl")
```

## 📖 Usage Examples

### Advanced Pipeline with Custom Config
```yaml
# pipeline_config.yaml
processing:
  chunk_size: 10000
  tokenizer: "meta-llama/Llama-3-8B"
  max_tokens_per_sample: 2048
  drop_incomplete: true

filters:
  min_length: 50
  max_length: 4096
  toxicity_threshold: 0.15
  language: "en,zh"

augmentation:
  enabled: true
  strategy: "instruction_paraphrase"
  llm_judge_model: "gpt-4o-mini"
```

```python
from soloai.ft_data import ConfigurableProcessor

proc = ConfigurableProcessor("pipeline_config.yaml")
raw_ds = proc.load("mixed_sources.parquet")
clean_ds = proc.filter(raw_ds)
augmented_ds = proc.augment(clean_ds)
augmented_ds.export("final_sft_dataset.jsonl", split_ratio=0.9)
```

### Domain-Specific Data Generation
```python
from soloai.ft_data import SyntheticDataGenerator

generator = SyntheticDataGenerator(
    base_model="Qwen/Qwen2.5-7B-Instruct",
    domain="financial_compliance",
    prompt_templates="templates/finance_sft.yaml"
)

synthetic_ds = generator.generate(n_samples=5000, temperature=0.7)
synthetic_ds.validate_schema("sharegpt")
synthetic_ds.save("synthetic_finance_train.jsonl")
```
