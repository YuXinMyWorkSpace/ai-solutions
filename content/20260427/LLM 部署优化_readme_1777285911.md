## Installation

### Prerequisites

- Python 3.10+
- pip or poetry
- NVIDIA GPU recommended for high-throughput inference
- CUDA drivers installed if using GPU acceleration
- Docker optional for containerized deployment

### Install from source

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
python -m venv .venv
source .venv/bin/activate   # Linux / macOS
# .venv\Scripts\activate    # Windows

pip install --upgrade pip
pip install -r requirements.txt
pip install -e .
```

### Optional extras

```bash
# Development dependencies
pip install -r requirements-dev.txt

# GPU / acceleration stack if defined by your environment
pip install torch transformers accelerate
```

### Docker

```bash
docker build -t soloai/llm-deployment-optimization:latest .
docker run --gpus all -p 8000:8000 soloai/llm-deployment-optimization:latest
```

## Quick Start

Start a local optimized inference server:

```bash
python -m soloai_llm_opt.serve \
  --model meta-llama/Llama-3-8B-Instruct \
  --host 0.0.0.0 \
  --port 8000 \
  --backend vllm \
  --max-batch-size 16 \
  --dtype float16
```

Send a test request:

```bash
curl -X POST "http://localhost:8000/generate" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Explain the benefits of continuous batching for LLM serving.",
    "max_tokens": 128,
    "temperature": 0.7
  }'
```

Example response:

```json
{
  "text": "Continuous batching improves GPU utilization by combining incoming requests dynamically, reducing idle time and increasing throughput while maintaining acceptable latency."
}
```

## Usage Examples

### 1. Run benchmarking

Benchmark latency and throughput under different concurrency settings:

```bash
python -m soloai_llm_opt.benchmark \
  --endpoint http://localhost:8000/generate \
  --concurrency 32 \
  --requests 500 \
  --input-tokens 256 \
  --output-tokens 128
```

### 2. Enable quantized deployment

Reduce memory footprint for lower-cost deployment:

```bash
python -m soloai_llm_opt.serve \
  --model mistralai/Mistral-7B-Instruct-v0.2 \
  --backend transformers \
  --quantization 4bit \
  --max-batch-size 8
```

### 3. Use configuration file

`config.yaml`

```yaml
model: meta-llama/Llama-3-8B-Instruct
backend: vllm
host: 0.0.0.0
port: 8000
dtype: float16
max_batch_size: 16
tensor_parallel_size: 2
metrics: true
log_level: info
```

Run with config:

```bash
python -m soloai_llm_opt.serve --config config.yaml
```

### 4. Python API example

```python
from soloai_llm_opt import InferenceServerConfig, start_server

config = InferenceServerConfig(
    model="meta-llama/Llama-3-8B-Instruct",
    backend="vllm",
    host="0.0.0.0",
    port=8000,
    dtype="float16",
    max_batch_size=16,
)

start_server(config)
```

### 5. Metrics and observability

Expose metrics for Prometheus scraping:

```bash
python -m soloai_llm_opt.serve \
  --model meta-llama/Llama-3-8B-Instruct \
  --metrics \
  --metrics-port 9090
```

Example metrics to track:

- request latency p50 / p95 / p99
- tokens per second
- GPU memory usage
- queue depth
- batch efficiency
- error rate
