# SoloAI 大模型微调数据服务

[![Build Status](https://img.shields.io/badge/build-passing-brightgreen.svg)](#)
[![License](https://img.shields.io/badge/license-MIT-blue.svg)](./LICENSE)

**SoloAI 大模型微调数据服务** is an open-source project focused on delivering scalable, high-quality data services for large language model fine-tuning. It helps teams standardize data collection, cleaning, annotation, validation, and delivery workflows for enterprise-grade LLM training and alignment.

## Overview

Large model performance depends heavily on data quality. SoloAI provides a practical framework and workflow toolkit for **fine-tuning dataset preparation**, helping AI teams build reliable, compliant, and domain-specific datasets for instruction tuning, supervised fine-tuning (SFT), preference data, and evaluation.

## Features

- **Fine-tuning data pipeline**: Support for collection, deduplication, normalization, filtering, and structured export.
- **Multi-format dataset support**: Work with JSON, JSONL, CSV, Markdown, QA pairs, conversations, and instruction-following formats.
- **Annotation and quality assurance**: Human-in-the-loop labeling, schema validation, sampling review, and consistency checks.
- **Domain adaptation services**: Build vertical datasets for finance, healthcare, legal, e-commerce, customer service, and more.
- **Data compliance and governance**: Support privacy-aware processing, sensitive data filtering, and traceable delivery standards.
- **Training-ready outputs**: Export datasets compatible with common LLM fine-tuning frameworks and evaluation pipelines.

## Quick Start

Prepare a sample input dataset:

```json
[
  {
    "instruction": "总结以下文本的核心观点",
    "input": "大模型微调的效果高度依赖训练数据质量。",
    "output": "高质量训练数据是提升大模型微调效果的关键。"
  }
]
```

Run a basic preprocessing workflow:

```bash
python scripts/preprocess.py \
  --input data/raw/sample.json \
  --output data/processed/sample.jsonl \
  --format sft
```

Validate the processed dataset:

```bash
python scripts/validate.py \
  --input data/processed/sample.jsonl \
  --schema configs/sft.schema.json
```

## Usage Examples

### 1. Convert raw business data into SFT format

```bash
python scripts/convert.py \
  --input ./data/raw/customer_support.csv \
  --output ./data/processed/customer_support.jsonl \
  --task instruction_tuning
```

Example output:

```json
{"instruction":"用户想申请退款，如何处理？","input":"","output":"请先确认订单状态、支付方式和退款政策，再引导用户提交退款申请。"}
{"instruction":"解释会员续费规则","input":"","output":"会员续费通常按当前套餐周期自动扣费，用户可在到期前取消自动续费。"}
```

### 2. Deduplicate and clean dataset records

```bash
python scripts/clean.py \
  --input ./data/raw/domain_data.jsonl \
  --output ./data/processed/domain_data.cleaned.jsonl \
  --remove-duplicates \
  --trim-whitespace \
  --drop-empty
```

### 3. Run quality checks

```bash
python scripts/qa_check.py \
  --input ./data/processed/domain_data.cleaned.jsonl \
  --report ./reports/qa_report.json
```

Sample QA report:

```json
{
  "total_records": 10000,
  "valid_records": 9735,
  "invalid_records": 265,
  "duplicate_ratio": 0.018,
  "empty_output_ratio": 0.006
}
```

### 4. Export training-ready data

```bash
python scripts/export.py \
  --input ./data/processed/domain_data.cleaned.jsonl \
  --output ./exports/train.jsonl \
  --framework transformers
```
