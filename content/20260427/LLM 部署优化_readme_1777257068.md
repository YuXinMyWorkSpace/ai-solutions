## Installation

### Requirements

- Python 3.10+
- `pip` or `poetry`
- CUDA-enabled GPU recommended for production inference
- Docker optional for containerized deployment

### Install from source

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
python -m venv .venv
source .venv/bin/activate   # Linux / macOS
# .venv\Scripts\activate    # Windows

pip install --upgrade pip
pip install -r requirements.txt
```

### Optional: install with development dependencies

```bash
pip install -r requirements-dev.txt
```

### Docker setup

```bash
docker build -t soloai-llm-opt:latest .
docker run --gpus all -p 8000:8000 soloai-llm-opt:latest
```

## Quick Start

Below is a minimal example for starting an optimized LLM inference service.

```bash
python -m src.server \
  --model /models/your-llm \
  --host 0.0.0.0 \
  --port 8000 \
  --precision fp16 \
  --max-batch-size 8
```

Test the endpoint:

```bash
curl -X POST "http://localhost:8000/v1/generate" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Explain how LLM deployment optimization reduces inference cost.",
    "max_tokens": 128,
    "temperature": 0.7
  }'
```

Example response:

```json
{
  "id": "gen_001",
  "output": "LLM deployment optimization reduces inference cost by improving hardware utilization, enabling batching, applying quantization, and minimizing idle compute time.",
  "latency_ms": 142
}
```

## Usage

### 1. Run with optimized precision

Use lower precision to improve throughput and reduce memory usage.

```bash
python -m src.server \
  --model /models/your-llm \
  --precision int8 \
  --tensor-parallel 2 \
  --max-batch-size 16
```

### 2. Enable metrics and observability

```bash
python -m src.server \
  --model /models/your-llm \
  --enable-metrics \
  --metrics-port 9090
```

Example metrics endpoint:

```bash
curl http://localhost:9090/metrics
```

### 3. Python client example

```python
import requests

url = "http://localhost:8000/v1/generate"
payload = {
    "prompt": "Summarize best practices for deploying an LLM in production.",
    "max_tokens": 100,
    "temperature": 0.3
}

response = requests.post(url, json=payload, timeout=30)
response.raise_for_status()

print(response.json())
```

### 4. Example configuration file

```yaml
server:
  host: 0.0.0.0
  port: 8000

model:
  path: /models/your-llm
  precision: fp16
  quantization: int8

inference:
  max_batch_size: 8
  max_input_tokens: 4096
  max_output_tokens: 512

observability:
  metrics: true
  metrics_port: 9090
```

Run with config:

```bash
python -m src.server --config configs/production.yaml
```
