# SoloAI LLM 部署优化

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/llm-deployment-optimization/ci.yml?branch=main)](https://github.com/SoloAI/llm-deployment-optimization/actions)
[![License](https://img.shields.io/github/license/SoloAI/llm-deployment-optimization)](./LICENSE)

A production-focused open-source toolkit for **LLM 部署优化** (LLM deployment optimization), built to improve inference efficiency, reduce serving cost, and simplify scalable deployment across cloud, on-prem, and edge environments.  
SoloAI helps engineering teams optimize large language model serving with practical workflows for benchmarking, quantization, acceleration, observability, and reproducible deployment.

---

## Overview

Modern LLM applications require more than model quality—they demand **low latency**, **high throughput**, **predictable cost**, and **operational reliability**. This project provides a technical foundation for teams deploying large language models in real-world environments, with a focus on:

- inference performance optimization
- GPU/CPU resource efficiency
- model quantization and runtime acceleration
- scalable API serving
- deployment reproducibility and observability

This repository is designed for AI engineers, MLOps teams, platform engineers, and enterprises building production-grade LLM systems.

---

## Features

- **High-performance inference optimization**  
  Tune serving pipelines for lower latency and higher throughput across common LLM runtimes.

- **Quantization-ready workflows**  
  Support optimization strategies such as 4-bit / 8-bit quantization to reduce memory footprint and infrastructure cost.

- **Scalable deployment architecture**  
  Deploy LLM services with containerized workflows suitable for Kubernetes, Docker, and hybrid infrastructure.

- **Benchmarking and profiling tools**  
  Measure latency, token throughput, memory usage, and concurrency performance for informed deployment decisions.

- **Observability and operational visibility**  
  Integrate logs, metrics, and health checks to monitor model serving in production.

- **Configurable API serving**  
  Expose optimized LLM inference endpoints for internal platforms, SaaS products, or enterprise AI applications.

## Installation

### Prerequisites

- Python 3.10+
- CUDA-compatible GPU recommended
- Docker optional for containerized deployment
- Linux environment recommended for production workloads

### Clone the repository

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
```

### Install with pip

```bash
python -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
```

### Optional: install development dependencies

```bash
pip install -r requirements-dev.txt
```

### Optional: run with Docker

```bash
docker build -t soloai/llm-deployment-optimization .
docker run --gpus all -p 8000:8000 soloai/llm-deployment-optimization
```

## Quick Start

Start a local optimized inference server:

```bash
python -m app.serve \
  --model meta-llama/Llama-3-8B-Instruct \
  --host 0.0.0.0 \
  --port 8000 \
  --quantization 4bit \
  --max-batch-size 16
```

Send a test request:

```bash
curl -X POST "http://localhost:8000/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "meta-llama/Llama-3-8B-Instruct",
    "messages": [
      {"role": "user", "content": "Explain KV cache optimization in LLM inference."}
    ],
    "temperature": 0.2,
    "max_tokens": 256
  }'
```

## Usage

### 1. Benchmark model serving performance

Use the benchmarking utility to compare deployment settings:

```bash
python -m tools.benchmark \
  --endpoint http://localhost:8000/v1/chat/completions \
  --concurrency 32 \
  --requests 200 \
  --input-tokens 512 \
  --output-tokens 128
```

Example output metrics:

- P50 latency
- P95 latency
- tokens/sec
- GPU memory usage
- request success rate

---

### 2. Run with quantization enabled

Reduce memory consumption while preserving acceptable quality:

```bash
python -m app.serve \
  --model mistralai/Mistral-7B-Instruct-v0.2 \
  --quantization 8bit \
  --gpu-memory-utilization 0.9
```

---

### 3. Deploy with Docker Compose

```yaml
version: "3.9"

services:
  llm-server:
    image: soloai/llm-deployment-optimization:latest
    ports:
      - "8000:8000"
    environment:
      MODEL_NAME: meta-llama/Llama-3-8B-Instruct
      QUANTIZATION: 4bit
      MAX_BATCH_SIZE: 16
    deploy:
      resources:
        reservations:
          devices:
            - capabilities: [gpu]
```

Start services:

```bash
docker compose up -d
```

---

### 4. Python client example

```python
import requests

url = "http://localhost:8000/v1/chat/completions"
payload = {
    "model": "meta-llama/Llama-3-8B-Instruct",
    "messages": [
        {"role": "system", "content": "You are a technical assistant."},
        {"role": "user", "content": "How can I reduce LLM inference latency in production?"}
    ],
    "temperature": 0.3,
    "max_tokens": 200
}

response = requests.post(url, json=payload, timeout=60)
response.raise_for_status()

print(response.json())
```

## Configuration

Example configuration file:

```yaml
model:
  name: meta-llama/Llama-3-8B-Instruct
  quantization: 4bit
  max_input_tokens: 4096
  max_output_tokens: 512

server:
  host: 0.0.0.0
  port: 8000
  workers: 1
  max_batch_size: 16

performance:
  gpu_memory_utilization: 0.9
  enable_kv_cache: true
  enable_continuous_batching: true
```

Run with config:

```bash
python -m app.serve --config configs/production.yaml
```
