# SoloAI-RAG 🤖📚
[![Build Status](https://github.com/SoloAI/solo-rag/actions/workflows/ci.yml/badge.svg)](https://github.com/SoloAI/solo-rag/actions)
[![License](https://img.shields.io/badge/License-Apache%202.0-blue.svg)](LICENSE)

**SoloAI-RAG** is a high-performance, modular Retrieval-Augmented Generation (RAG) framework engineered to bridge enterprise knowledge bases with state-of-the-art large language models. It delivers contextually precise, hallucination-resistant outputs through optimized vector indexing, hybrid retrieval pipelines, and deterministic prompt orchestration.

## ✨ Key Features
- 🔍 **Hybrid Retrieval Engine**: Combines dense embeddings, sparse BM25, and cross-encoder reranking for maximum recall and precision.
- 🧩 **Intelligent Chunking Pipeline**: Semantic, recursive, and layout-aware document splitting with automatic metadata preservation.
- ⚡ **Low-Latency Streaming & Caching**: Built-in response streaming, KV-cache optimization, and LRU query caching for production-grade throughput.
- 📊 **Native Evaluation & Observability**: Integrated RAGAS metrics, trace logging, and retrieval hit-rate dashboards for continuous system tuning.
- 🔌 **Pluggable Architecture**: Seamless interoperability with LangChain, LlamaIndex, ChromaDB, Weaviate, Pinecone, and any OpenAI-compatible LLM API.

## 📦 Installation
Requires **Python 3.9+**.

```bash

pip install solo-rag

# Full installation with vector DBs, rerankers, and LLM adapters
pip install "solo-rag[all]"

# Development setup
git clone https://github.com/SoloAI/solo-rag.git
cd solo-rag
pip install -e ".[dev]"
```

## 🚀 Quick Start
Initialize a production-ready RAG pipeline in under 10 lines:

```python
from solo_rag import RAGPipeline, VectorStore, LLMProvider

# 1. Configure components
rag = RAGPipeline(
    vector_store=VectorStore.from_config("chromadb", path="./data/vectors"),
    llm=LLMProvider("openai", model="gpt-4o-mini", temperature=0.1),
    chunk_size=512,
    top_k=4
)

# 2. Ingest documents
rag.ingest(["docs/technical_manual.pdf", "docs/api_reference.md"])

# 3. Query with grounded context
response = rag.query("What are the recommended timeout thresholds for batch processing?")
print(f"Answer: {response.answer}\nSources: {response.citations}")
```

## 🛠️ Advanced Usage Examples

### Hybrid Search with Custom Reranking
```python
retriever = rag.get_retriever(
    strategy="hybrid",
    bm25_weight=0.3,
    dense_weight=0.7,
    reranker="cross-encoder/ms-marco-MiniLM-L-6-v2",
    metadata_filter={"department": "engineering", "version": ">=2.0"}
)

for chunk in rag.stream_query("Explain the tokenization fallback mechanism", retriever=retriever):
    print(chunk.delta, end="", flush=True)
```

### Automated RAG Evaluation
```python
from solo_rag.eval import evaluate_rag

# Run benchmark against golden dataset
results = evaluate_rag(
    pipeline=rag,
    dataset="tests/fixtures/benchmark_qa.json",
    metrics=["faithfulness", "answer_relevancy", "context_precision"]
)

print(results.summary())
# Export to JSON/CSV for CI/CD gating
results.to_json("eval_report.json")
```
