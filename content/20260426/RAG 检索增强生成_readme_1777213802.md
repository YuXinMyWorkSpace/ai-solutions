# SoloAI RAG 检索增强生成
[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/rag-retrieval-augmented-generation/ci.yml?branch=main)](https://github.com/SoloAI/rag-retrieval-augmented-generation/actions)

面向生产环境的 **RAG（Retrieval-Augmented Generation，检索增强生成）** 开源项目，帮助开发者构建高质量、可追溯、可扩展的 AI 问答与知识库系统。  
本项目聚焦中文与多语言场景，覆盖文档解析、向量检索、重排序、上下文增强生成与服务化部署。

---

## 简介

**SoloAI RAG 检索增强生成** 是一个用于构建企业级知识检索与大模型问答系统的开源框架。  
它提供从数据摄取、文本切分、Embedding、向量数据库检索，到 LLM 生成、引用溯源与 API 集成的一体化能力，适用于客服、内部知识库、文档问答、行业智能助手等场景。

---

## 核心特性

- **多源知识接入**：支持 Markdown、PDF、TXT、HTML、Word 等常见文档格式的导入与解析
- **高性能语义检索**：集成向量检索、关键词检索与混合检索策略，提升召回效果
- **Rerank 重排序能力**：通过重排序模型优化 Top-K 结果，增强答案相关性与准确性
- **可追溯答案生成**：生成结果可附带来源片段与引用，满足企业知识合规与可信需求
- **模块化架构设计**：便于替换 Embedding 模型、向量数据库、LLM 服务与检索组件
- **面向生产部署**：支持 API 服务化、配置化运行、容器化部署与二次开发

---

## 技术架构

典型 RAG 工作流如下：

1. **文档导入**：加载企业文档、FAQ、产品手册或结构化数据
2. **文档预处理**：清洗内容、分块切片、提取元数据
3. **向量化索引**：调用 Embedding 模型生成向量并写入向量数据库
4. **检索召回**：根据用户问题执行向量检索、混合检索或过滤检索
5. **上下文增强**：对召回内容进行重排序、压缩与上下文组装
6. **大模型生成**：将高相关上下文注入 Prompt，输出可信答案
7. **来源引用**：返回答案及对应文档来源，便于审计与验证

## 安装

### 环境要求

- Python 3.10+
- pip 或 poetry
- 可选：Docker / Docker Compose
- 可选：OpenAI-compatible API、Qwen、GLM、DeepSeek 或本地推理服务
- 可选：FAISS / Milvus / Weaviate / Chroma / Elasticsearch

### 使用 pip 安装

```bash
git clone https://github.com/SoloAI/rag-retrieval-augmented-generation.git
cd rag-retrieval-augmented-generation

python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

pip install -U pip
pip install -r requirements.txt
```

### 环境变量配置

创建 `.env` 文件：

```bash
cp .env.example .env
```

示例配置：

```env
OPENAI_API_KEY=your_api_key
OPENAI_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4o-mini

EMBEDDING_MODEL=text-embedding-3-large
VECTOR_STORE=faiss

CHUNK_SIZE=800
CHUNK_OVERLAP=100
TOP_K=5
```

### Docker 安装

```bash
docker build -t soloai-rag .
docker run -p 8000:8000 --env-file .env soloai-rag
```

## 快速开始

以下示例演示如何完成文档索引并发起一次 RAG 问答。

### 1）构建索引

```bash
python scripts/index_documents.py \
  --input ./data/knowledge_base \
  --output ./storage/index \
  --embedding-model text-embedding-3-large
```

### 2）启动 API 服务

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 3）发送查询请求

```bash
curl -X POST "http://127.0.0.1:8000/api/v1/chat" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "RAG 检索增强生成在企业知识库中的核心优势是什么？",
    "top_k": 5,
    "return_sources": true
  }'
```

## 使用示例

### Python SDK 示例

```python
from soloai_rag import RAGPipeline

pipeline = RAGPipeline.from_config("configs/default.yaml")

pipeline.index_documents("./data/knowledge_base")

result = pipeline.query(
    question="如何降低大模型幻觉并提升问答可信度？",
    top_k=5,
    return_sources=True
)

print("Answer:")
print(result["answer"])
print("\nSources:")
for item in result["sources"]:
    print(f'- {item["document"]} ({item["score"]})')
```

### 自定义检索参数

```python
result = pipeline.query(
    question="什么是混合检索？",
    top_k=8,
    search_type="hybrid",
    rerank=True,
    filters={"department": "product"},
    return_sources=True
)
```

### FastAPI 接口示例

```python
from fastapi import FastAPI
from soloai_rag import RAGPipeline

app = FastAPI(title="SoloAI RAG API")
pipeline = RAGPipeline.from_config("configs/default.yaml")

@app.post("/ask")
def ask(payload: dict):
    return pipeline.query(
        question=payload["query"],
        top_k=payload.get("top_k", 5),
        return_sources=payload.get("return_sources", True)
    )
```

## 最佳实践

- 针对长文档场景合理设置 `CHUNK_SIZE` 与 `CHUNK_OVERLAP`
- 优先使用 **混合检索 + Rerank** 提升复杂问答的准确率
- 为文档增加元数据标签，如业务线、部门、时间戳、权限等级
- 对高价值场景开启 **来源返回** 与 **回答审计**
- 定期重建索引，保证知识库内容与业务文档同步更新
