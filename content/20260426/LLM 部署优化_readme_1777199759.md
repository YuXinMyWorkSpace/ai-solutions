## Installation

### Prerequisites

- Python 3.10+
- pip or poetry
- Docker (optional, recommended for production)
- NVIDIA GPU + CUDA drivers (optional, for GPU inference)
- Linux environment recommended for high-performance serving

### Install from source

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
python -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
```

### Optional: install development dependencies

```bash
pip install -r requirements-dev.txt
```

### Optional: Docker setup

```bash
docker build -t soloai/llm-deployment-optimization:latest .
docker run --rm -it -p 8000:8000 soloai/llm-deployment-optimization:latest
```

## Quick Start

Below is a minimal example for launching an optimized LLM serving endpoint.

### 1. Configure deployment

Create a config file:

```yaml
model:
  name: "your-llm-model"
  path: "/models/your-llm-model"
  dtype: "float16"
  quantization: "int8"

serving:
  host: "0.0.0.0"
  port: 8000
  max_batch_size: 16
  max_concurrency: 64
  request_timeout: 60

optimization:
  enable_dynamic_batching: true
  enable_kv_cache: true
  tensor_parallel_size: 1
  gpu_memory_utilization: 0.9

observability:
  enable_metrics: true
  log_level: "INFO"
```

### 2. Start the service

```bash
python -m soloai_llm_deploy serve --config configs/production.yaml
```

### 3. Send a test request

```bash
curl -X POST "http://localhost:8000/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "your-llm-model",
    "messages": [
      {"role": "user", "content": "Explain how deployment optimization reduces LLM inference cost."}
    ],
    "temperature": 0.2
  }'
```

## Usage Examples

## 1. Run a local optimized inference server

```bash
python -m soloai_llm_deploy serve \
  --model /models/llama \
  --port 8000 \
  --max-batch-size 32 \
  --max-concurrency 128 \
  --enable-dynamic-batching
```

## 2. Enable quantized deployment

```bash
python -m soloai_llm_deploy serve \
  --model /models/llama \
  --quantization int4 \
  --dtype float16
```

## 3. Export deployment metrics

```bash
python -m soloai_llm_deploy serve \
  --config configs/production.yaml \
  --enable-metrics \
  --metrics-port 9090
```

Example Prometheus metrics endpoint:

```bash
curl http://localhost:9090/metrics
```

## 4. Python API example

```python
from soloai_llm_deploy import DeploymentEngine, DeploymentConfig

config = DeploymentConfig(
    model_name="your-llm-model",
    model_path="/models/your-llm-model",
    dtype="float16",
    quantization="int8",
    max_batch_size=16,
    max_concurrency=64,
    enable_dynamic_batching=True,
)

engine = DeploymentEngine(config)
engine.start()

response = engine.generate(
    prompt="Summarize the benefits of dynamic batching for LLM serving.",
    max_tokens=128,
    temperature=0.3,
)

print(response.text)
```

## 5. Kubernetes deployment example

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: soloai-llm-serving
spec:
  replicas: 2
  selector:
    matchLabels:
      app: soloai-llm-serving
  template:
    metadata:
      labels:
        app: soloai-llm-serving
    spec:
      containers:
        - name: llm-server
          image: soloai/llm-deployment-optimization:latest
          ports:
            - containerPort: 8000
          env:
            - name: MODEL_PATH
              value: /models/your-llm-model
            - name: MAX_BATCH_SIZE
              value: "16"
            - name: MAX_CONCURRENCY
              value: "64"
```
