## 项目简介

**SoloAI 大模型微调数据服务** 是一个面向开源生态与企业场景的数据服务项目，提供从原始数据处理到训练集交付的一站式能力。  
项目聚焦于**数据清洗、指令构造、偏好对构建、质量评估、格式转换与交付标准化**，适用于通用大模型、垂直行业模型及多轮对话模型的数据准备流程。

## 核心特性

- **多类型微调数据处理**：支持 SFT、Instruction Tuning、DPO/Preference、RLHF、分类、抽取、问答、对话等数据格式
- **高质量数据清洗与去重**：支持规则清洗、PII 脱敏、重复样本检测、低质量样本过滤
- **标准化数据交付**：支持 JSONL、CSV、Parquet 等常见训练格式，便于接入主流训练框架
- **可扩展的数据质检流程**：支持字段校验、标签一致性检查、人工抽检与自动评分
- **行业级数据构造能力**：适配金融、医疗、教育、客服、电商、法律、制造等垂直领域
- **面向模型训练优化**：可根据目标模型、上下文长度、任务类型与交付标准进行定制化生产

## 快速开始

以下示例展示如何将原始问答数据转换为适合大模型监督微调（SFT）的 `jsonl` 数据格式。

### 输入示例

```json
[
  {
    "question": "什么是向量数据库？",
    "answer": "向量数据库是一种用于存储和检索向量表示的数据系统。"
  }
]
```

### Python 示例

```python
import json

raw_data = [
    {
        "question": "什么是向量数据库？",
        "answer": "向量数据库是一种用于存储和检索向量表示的数据系统。"
    }
]

output_path = "train_sft.jsonl"

with open(output_path, "w", encoding="utf-8") as f:
    for item in raw_data:
        sample = {
            "messages": [
                {"role": "user", "content": item["question"]},
                {"role": "assistant", "content": item["answer"]}
            ]
        }
        f.write(json.dumps(sample, ensure_ascii=False) + "\n")

print(f"SFT 数据已生成: {output_path}")
```

### 输出示例

```json
{"messages":[{"role":"user","content":"什么是向量数据库？"},{"role":"assistant","content":"向量数据库是一种用于存储和检索向量表示的数据系统。"}]}
```

## 使用示例

## 1. 数据清洗

用于过滤空值、去除异常样本、统一字段格式。

```python
def clean_record(record: dict) -> dict | None:
    question = str(record.get("question", "")).strip()
    answer = str(record.get("answer", "")).strip()

    if not question or not answer:
        return None

    if len(answer) < 5:
        return None

    return {
        "question": question,
        "answer": answer
    }

records = [
    {"question": "什么是微调？", "answer": "微调是指在预训练模型基础上继续训练。"},
    {"question": "", "answer": "无效数据"}
]

cleaned = [r for r in (clean_record(x) for x in records) if r]
print(cleaned)
```

---

## 2. 构造指令微调数据

将业务文本转换为 instruction-following 格式，适配通用大模型训练。

```python
import json

dataset = [
    {
        "instruction": "根据给定内容生成摘要",
        "input": "大模型微调通常依赖高质量指令数据、偏好数据和评测数据。",
        "output": "大模型微调依赖高质量的训练与评测数据。"
    }
]

with open("instruction_data.jsonl", "w", encoding="utf-8") as f:
    for item in dataset:
        f.write(json.dumps(item, ensure_ascii=False) + "\n")
```

---

## 3. 构造 DPO / 偏好数据

适用于偏好优化、对齐训练和响应排序任务。

```python
import json

preference_data = [
    {
        "prompt": "请解释什么是监督微调。",
        "chosen": "监督微调是利用高质量标注数据对预训练模型进行进一步训练，以适配特定任务。",
        "rejected": "监督微调就是随便调整模型参数。"
    }
]

with open("dpo_data.jsonl", "w", encoding="utf-8") as f:
    for item in preference_data:
        f.write(json.dumps(item, ensure_ascii=False) + "\n")
```

## 最佳实践

1. **先定义训练目标**：明确是 SFT、DPO、分类还是 RAG 评测任务  
2. **统一数据 Schema**：在清洗前定义字段规范与校验规则  
3. **控制样本质量**：优先保证正确性、一致性、覆盖度与可解释性  
4. **引入人工抽检**：关键行业场景建议结合专家审核  
5. **按模型特性优化**：根据上下文长度、语言风格、任务边界设计数据模板
