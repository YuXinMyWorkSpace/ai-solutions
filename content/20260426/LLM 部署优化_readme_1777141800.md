## Installation

### Prerequisites

- Python 3.10+
- `pip` or `uv`
- CUDA-compatible GPU recommended for high-throughput deployment
- Docker optional for containerized serving

### Clone the repository

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
```

### Install dependencies

```bash
pip install -r requirements.txt
```

### Optional: Create a virtual environment

```bash
python -m venv .venv
source .venv/bin/activate   # Linux / macOS
# .venv\Scripts\activate    # Windows
pip install -r requirements.txt
```

### Optional: Docker setup

```bash
docker build -t soloai/llm-deployment-optimization .
docker run --rm -p 8000:8000 soloai/llm-deployment-optimization
```

## Quick Start

Below is a minimal example for launching an optimized LLM inference service.

```bash
python serve.py \
  --model ./models/your-llm \
  --host 0.0.0.0 \
  --port 8000 \
  --precision fp16 \
  --max-batch-size 8 \
  --num-workers 2
```

Test the API:

```bash
curl -X POST http://127.0.0.1:8000/generate \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Explain how quantization improves LLM deployment efficiency.",
    "max_tokens": 128,
    "temperature": 0.7
  }'
```

Example response:

```json
{
  "output": "Quantization reduces the numerical precision of model weights and activations, which lowers memory usage and can improve inference speed while maintaining acceptable output quality.",
  "latency_ms": 142,
  "model": "your-llm"
}
```

## Usage

### 1. Run with quantized weights

Use lower-precision model artifacts to reduce VRAM usage and improve deployment density.

```bash
python serve.py \
  --model ./models/your-llm-awq \
  --quantization awq \
  --precision int4 \
  --max-batch-size 16
```

### 2. Tune for low-latency serving

For interactive applications such as chat systems, optimize for response time.

```bash
python serve.py \
  --model ./models/your-llm \
  --precision fp16 \
  --max-batch-size 4 \
  --prefill-optimization true \
  --kv-cache true
```

### 3. Tune for high throughput

For internal tools, batch generation, or multi-user workloads, increase concurrency.

```bash
python serve.py \
  --model ./models/your-llm \
  --precision bf16 \
  --max-batch-size 32 \
  --num-workers 4 \
  --queue-size 256
```

### 4. Python SDK style invocation

```python
from client import LLMClient

client = LLMClient(base_url="http://127.0.0.1:8000")

response = client.generate(
    prompt="Give me three deployment strategies for reducing LLM inference cost.",
    max_tokens=120,
    temperature=0.3,
)

print(response["output"])
```

### 5. Example configuration file

```yaml
server:
  host: 0.0.0.0
  port: 8000
  workers: 2

model:
  path: ./models/your-llm
  precision: fp16
  quantization: none

optimization:
  max_batch_size: 8
  kv_cache: true
  prefill_optimization: true

observability:
  metrics: true
  structured_logging: true
```

Run with config:

```bash
python serve.py --config config.yaml
```
