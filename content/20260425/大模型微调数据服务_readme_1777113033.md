# SoloAI 大模型微调数据服务

[![Build Status](https://img.shields.io/badge/build-passing-brightgreen)](#)
[![License](https://img.shields.io/badge/license-MIT-blue.svg)](./LICENSE)

**SoloAI 大模型微调数据服务** 是一个面向企业与研究团队的开源数据服务项目，专注于为大语言模型（LLM）微调、指令优化、偏好对齐与评测提供高质量、可扩展、可治理的数据处理能力。  
本项目致力于帮助团队高效完成从原始数据采集、清洗、标注、转换到训练集交付的全流程数据工程建设。

---

## 项目简介

在大模型落地过程中，**高质量数据**往往比模型结构本身更决定最终效果。SoloAI 提供一套围绕“大模型微调数据服务”的工程化方案，支持构建适用于 SFT、DPO、RAG、分类、抽取、对话、Agent 等多种场景的数据生产与交付流程。

---

## 核心特性

- **多源数据接入**：支持文本、问答、对话、文档、表格及业务系统导出数据的统一接入与处理
- **数据清洗与标准化**：支持去重、脱敏、格式统一、低质样本过滤、指令模板转换
- **微调数据集构建**：面向 SFT、偏好对齐、分类、信息抽取、RAG 问答等任务生成结构化训练数据
- **质量控制机制**：支持规则校验、采样复核、标签一致性检查与数据集评估
- **可扩展交付格式**：兼容 JSON、JSONL、CSV 及主流训练框架数据格式
- **企业级服务支持**：适配行业定制数据服务、批量交付、周期性更新与私有化流程集成

---

## 适用场景

- 行业大模型微调数据集建设
- 企业客服/知识库问答训练数据生成
- 指令跟随（Instruction Tuning）数据构造
- 偏好数据与对齐数据生产
- 垂直领域抽取、分类、审核与总结任务数据整理
- RAG 知识问答数据准备与评测集构建

## 快速开始

以下示例展示如何将原始问答数据转换为适用于大模型监督微调（SFT）的 JSONL 数据集。

### 输入示例

```json
[
  {
    "question": "什么是增值税专用发票？",
    "answer": "增值税专用发票是用于增值税抵扣的发票。"
  },
  {
    "question": "如何申请企业营业执照？",
    "answer": "可以通过当地市场监督管理部门线上或线下办理。"
  }
]
```

### Python 快速处理示例

```python
from soloai_data_service.cleaning import normalize_text
from soloai_data_service.exporters import export_jsonl

raw_data = [
    {
        "question": "什么是增值税专用发票？",
        "answer": "增值税专用发票是用于增值税抵扣的发票。"
    },
    {
        "question": "如何申请企业营业执照？",
        "answer": "可以通过当地市场监督管理部门线上或线下办理。"
    }
]

dataset = []
for item in raw_data:
    dataset.append({
        "messages": [
            {"role": "system", "content": "你是一名专业的企业知识助手。"},
            {"role": "user", "content": normalize_text(item["question"])},
            {"role": "assistant", "content": normalize_text(item["answer"])}
        ]
    })

export_jsonl(dataset, "data/output/sft_dataset.jsonl")
print("SFT 数据集已生成：data/output/sft_dataset.jsonl")
```

### 输出示例

```json
{"messages":[{"role":"system","content":"你是一名专业的企业知识助手。"},{"role":"user","content":"什么是增值税专用发票？"},{"role":"assistant","content":"增值税专用发票是用于增值税抵扣的发票。"}]}
{"messages":[{"role":"system","content":"你是一名专业的企业知识助手。"},{"role":"user","content":"如何申请企业营业执照？"},{"role":"assistant","content":"可以通过当地市场监督管理部门线上或线下办理。"}]}
```

## 使用示例

### 1. 清洗原始文本数据

```python
from soloai_data_service.cleaning import normalize_text, remove_duplicates

records = [
    "  这是第一条样本。 ",
    "这是第一条样本。",
    "第二条样本。"
]

normalized = [normalize_text(x) for x in records]
unique_records = remove_duplicates(normalized)

print(unique_records)
```

### 2. 构建指令微调数据

```python
from soloai_data_service.labeling import build_instruction_sample

sample = build_instruction_sample(
    instruction="请总结以下文本的核心观点",
    input_text="大模型在企业场景中需要结合业务知识、流程与真实数据进行优化。",
    output_text="企业级大模型优化依赖业务知识、流程和真实数据的结合。"
)

print(sample)
```

示例输出：

```json
{
  "instruction": "请总结以下文本的核心观点",
  "input": "大模型在企业场景中需要结合业务知识、流程与真实数据进行优化。",
  "output": "企业级大模型优化依赖业务知识、流程和真实数据的结合。"
}
```

### 3. 导出为 JSONL

```python
from soloai_data_service.exporters import export_jsonl

samples = [
    {
        "instruction": "提取公司名称",
        "input": "SoloAI 为客户提供大模型数据服务。",
        "output": "SoloAI"
    }
]

export_jsonl(samples, "data/output/instruction_dataset.jsonl")
```

### 4. 数据质量校验

```python
from soloai_data_service.validation import validate_record

record = {
    "instruction": "判断文本情感",
    "input": "这个产品交付效率很高。",
    "output": "正向"
}

result = validate_record(record)
print(result)
```

## 数据格式建议

### SFT 对话格式

```json
{
  "messages": [
    {"role": "system", "content": "你是一个专业助手"},
    {"role": "user", "content": "请解释什么是知识蒸馏"},
    {"role": "assistant", "content": "知识蒸馏是将大模型能力迁移到小模型的训练方法。"}
  ]
}
```

### Instruction 格式

```json
{
  "instruction": "提取合同中的签约主体",
  "input": "甲方为某科技有限公司，乙方为某智能系统有限公司。",
  "output": "甲方：某科技有限公司；乙方：某智能系统有限公司。"
}
```

### Preference/DPO 格式

```json
{
  "prompt": "请写一个简洁的项目总结",
  "chosen": "本项目已完成核心模块开发，并通过测试验证。",
  "rejected": "这个项目大概做了一些东西，应该还行吧。"
}
```
