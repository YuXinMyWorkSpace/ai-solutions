# SoloAI LLM 部署优化

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/llm-deployment-optimization/ci.yml?branch=main)](https://github.com/SoloAI/llm-deployment-optimization/actions)
[![License](https://img.shields.io/github/license/SoloAI/llm-deployment-optimization)](./LICENSE)

面向生产环境的 **LLM 部署优化（Large Language Model Deployment Optimization）** 开源项目，专注于提升大语言模型在推理阶段的 **吞吐、时延、资源利用率与部署稳定性**。  
该项目适用于 AI 平台、企业私有化部署、GPU 推理服务、边缘推理与多模型服务编排等场景。

---

## Overview

**SoloAI LLM 部署优化** 提供一套可扩展的工程化能力，用于优化大语言模型的部署流程，包括模型加载、推理加速、显存控制、量化支持、并发调度和服务化封装。  
目标是帮助团队更快构建 **高性能、低成本、可观测、易扩展** 的 LLM 推理系统。

---

## Features

- **高性能推理优化**：支持降低首 token 延迟（TTFT）与整体响应时延，提升吞吐能力。
- **量化与资源优化**：支持 4-bit / 8-bit 等量化部署策略，降低显存占用与基础设施成本。
- **多后端兼容**：可适配 Transformers、vLLM、TensorRT-LLM、ONNX Runtime 等常见推理后端。
- **服务化部署能力**：提供标准 API 服务封装，便于接入 Web、Agent、RAG 与企业应用。
- **并发与批处理调度**：支持动态 batching、请求队列管理与 GPU 资源调优。
- **可观测性与生产可用性**：集成日志、指标监控、健康检查与部署配置管理。

## Installation

### Requirements

- Python 3.10+
- CUDA 11.8+（如使用 GPU）
- PyTorch 2.1+
- 推荐 Linux 环境（Ubuntu 20.04+）

### Install from source

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization

python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

pip install -U pip
pip install -r requirements.txt
pip install -e .
```

### Optional dependencies

安装量化、推理后端或服务能力所需组件：

```bash
# vLLM backend
pip install vllm

# bitsandbytes quantization
pip install bitsandbytes

# API service
pip install fastapi uvicorn

# monitoring
pip install prometheus-client
```

## Quick Start

以下示例展示如何以最小配置启动一个本地 LLM 推理服务。

```python
from soloai_llm_deploy import LLMServer, DeploymentConfig

config = DeploymentConfig(
    model_name="meta-llama/Llama-3-8B-Instruct",
    device="cuda",
    dtype="float16",
    max_batch_size=8,
    quantization="4bit",
    backend="vllm",
)

server = LLMServer(config)
server.load_model()
response = server.generate("请简要说明 LLM 部署优化的核心目标。")

print(response)
```

运行：

```bash
python examples/quick_start.py
```

## Usage

### 1. Start an API server

```bash
python -m soloai_llm_deploy.serve \
  --model meta-llama/Llama-3-8B-Instruct \
  --backend vllm \
  --device cuda \
  --dtype float16 \
  --port 8000
```

### 2. Request inference

```bash
curl -X POST "http://127.0.0.1:8000/generate" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "什么是低延迟 LLM 推理架构？",
    "max_tokens": 256,
    "temperature": 0.7
  }'
```

### 3. Python client example

```python
from soloai_llm_deploy import Client

client = Client(base_url="http://127.0.0.1:8000")

result = client.generate(
    prompt="请列出三种常见的 LLM 推理优化方法。",
    max_tokens=128,
    temperature=0.3,
)

print(result.text)
```

### 4. Quantized deployment example

```python
from soloai_llm_deploy import LLMServer, DeploymentConfig

config = DeploymentConfig(
    model_name="mistralai/Mistral-7B-Instruct-v0.2",
    backend="transformers",
    device="cuda",
    quantization="8bit",
    max_new_tokens=256,
)

server = LLMServer(config)
server.load_model()

print(server.generate("解释量化对显存与吞吐的影响。"))
```

### 5. Production deployment notes

```yaml
server:
  host: 0.0.0.0
  port: 8000
  workers: 1

model:
  name: meta-llama/Llama-3-8B-Instruct
  backend: vllm
  dtype: float16
  quantization: 4bit

performance:
  max_batch_size: 16
  max_concurrency: 128
  gpu_memory_utilization: 0.9

observability:
  metrics: true
  healthcheck: true
  structured_logging: true
```
