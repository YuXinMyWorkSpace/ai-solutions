# SoloAI RAG 检索增强生成

![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/rag-retrieval-augmented-generation/ci.yml?branch=main&style=flat-square)
![License](https://img.shields.io/github/license/SoloAI/rag-retrieval-augmented-generation?style=flat-square)

A production-ready open-source framework by **SoloAI** for **RAG（Retrieval-Augmented Generation，检索增强生成）** workflows, enabling LLM applications to retrieve, rank, and ground responses with enterprise knowledge. Built for scalable document ingestion, semantic retrieval, and accurate answer generation across knowledge bases, private corpora, and business data.

---

## Overview

**RAG 检索增强生成** combines information retrieval with large language models to generate context-aware, source-grounded responses. This project helps developers build AI assistants, enterprise search systems, internal knowledge bots, and domain-specific Q&A applications with better factuality, transparency, and relevance.

## Features

- **Document ingestion pipeline** for PDF, Markdown, TXT, HTML, and structured data
- **Vector search + hybrid retrieval** with support for embeddings and keyword-based recall
- **Chunking, indexing, and metadata filtering** for precise context selection
- **LLM answer generation with citations** to improve trust and traceability
- **Configurable RAG workflow** for local development and production deployment
- **API-first architecture** for integration with chatbots, enterprise apps, and AI platforms

## Installation

### Prerequisites

- Python 3.10+
- `pip` or `poetry`
- Optional:
  - OpenAI / compatible LLM API key
  - Vector database such as FAISS, Chroma, Milvus, or PostgreSQL + pgvector

### Install from source

```bash
git clone https://github.com/SoloAI/rag-retrieval-augmented-generation.git
cd rag-retrieval-augmented-generation
pip install -r requirements.txt
```

### Optional environment setup

Create a `.env` file:

```env
LLM_PROVIDER=openai
LLM_API_KEY=your_api_key_here
EMBEDDING_MODEL=text-embedding-3-large
CHAT_MODEL=gpt-4o-mini
VECTOR_DB=faiss
DATA_DIR=./data
INDEX_DIR=./index
```

### Run tests

```bash
pytest
```

## Quick Start

Below is a minimal example of indexing documents and querying the RAG pipeline.

```python
from soloai_rag import RAGPipeline

rag = RAGPipeline(
    embedding_model="text-embedding-3-large",
    chat_model="gpt-4o-mini",
    vector_store="faiss"
)

# Index documents
rag.ingest("./data")

# Ask a question
response = rag.query("什么是 RAG 检索增强生成？它适合哪些企业场景？")

print(response.answer)
print(response.sources)
```

## Usage Examples

### 1. Ingest local files

```python
from soloai_rag import DocumentIngestor

ingestor = DocumentIngestor(
    chunk_size=800,
    chunk_overlap=120,
    supported_formats=["pdf", "md", "txt", "html"]
)

documents = ingestor.load("./knowledge_base")
print(f"Loaded {len(documents)} documents")
```

### 2. Build a vector index

```python
from soloai_rag import VectorIndexer

indexer = VectorIndexer(
    provider="faiss",
    embedding_model="text-embedding-3-large"
)

indexer.create_index(documents)
indexer.persist("./index")
```

### 3. Query with metadata filters

```python
from soloai_rag import RAGPipeline

rag = RAGPipeline.load("./index")

response = rag.query(
    question="如何降低企业知识库问答中的幻觉？",
    filters={"department": "engineering", "lang": "zh"}
)

print(response.answer)
for source in response.sources:
    print(source)
```

### 4. Start an API service

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Example request:

```bash
curl -X POST "http://localhost:8000/query" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "RAG 系统如何实现可追溯回答？",
    "top_k": 5,
    "filters": {
      "domain": "enterprise-ai"
    }
  }'
```

Example response:

```json
{
  "answer": "通过检索相关知识片段并附带来源引用，RAG 系统可以生成可追溯、可验证的回答。",
  "sources": [
    {
      "id": "doc_102",
      "title": "RAG Architecture Guide",
      "score": 0.93
    }
  ]
}
```
