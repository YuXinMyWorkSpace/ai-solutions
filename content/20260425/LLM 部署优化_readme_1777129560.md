# SoloAI LLM Deployment Optimization

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/llm-deployment-optimization/ci.yml?branch=main&style=flat-square)](https://github.com/SoloAI/llm-deployment-optimization/actions)

A production-focused open-source toolkit by **SoloAI** for **LLM 部署优化** (LLM deployment optimization), designed to improve inference performance, reduce serving cost, and simplify scalable deployment across cloud, edge, and on-prem environments.  
It helps engineering teams optimize model serving pipelines with practical tools for quantization, batching, resource scheduling, and observability.

---

## Overview

**SoloAI LLM Deployment Optimization** is built for developers and platform teams who need faster, more efficient, and more reliable large language model deployment. The project focuses on real-world inference optimization strategies, including throughput tuning, latency reduction, memory efficiency, and production operations.

### Keywords
LLM deployment optimization, LLM serving, inference acceleration, quantization, batching, GPU utilization, model serving infrastructure, production AI systems, scalable LLM deployment

## Features

- **High-performance inference optimization** for reduced latency and improved throughput
- **Quantization and memory reduction workflows** to lower GPU/CPU resource usage

- **Multi-environment deployment support** across local, cloud, Kubernetes, and edge setups

## Installation

### Requirements

- Python 3.10+
- CUDA 11.8+ recommended for GPU serving
- Docker 24+ optional
- Kubernetes 1.26+ optional for cluster deployment

### Install from source

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
python -m venv .venv
source .venv/bin/activate
pip install -U pip setuptools wheel
pip install -r requirements.txt
pip install -e .
```

### Install with Docker

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
docker build -t soloai/llm-deployment-optimization:latest .
docker run --gpus all -p 8000:8000 soloai/llm-deployment-optimization:latest
```

## Quick Start

Start an optimized inference server:

```bash
python -m soloai_llm_opt.serve \
  --model mistralai/Mistral-7B-Instruct-v0.2 \
  --host 0.0.0.0 \
  --port 8000 \
  --dtype float16 \
  --batch-size 8 \
  --max-input-tokens 2048 \
  --max-total-tokens 4096
```

Send a request:

```bash
curl -X POST "http://localhost:8000/v1/generate" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Explain how dynamic batching improves LLM inference efficiency.",
    "max_tokens": 128,
    "temperature": 0.2
  }'
```

Example response:

```json
{
  "text": "Dynamic batching combines multiple inference requests into a shared execution step, improving hardware utilization and increasing throughput while reducing per-request overhead."
}
```

## Usage Examples

### 1. Run Benchmark

Measure latency, throughput, and token generation speed:

```bash
python -m soloai_llm_opt.benchmark \
  --endpoint http://localhost:8000/v1/generate \
  --concurrency 16 \
  --requests 200 \
  --input-tokens 512 \
  --output-tokens 128
```

### 2. Enable Quantization

Use 4-bit quantization to reduce memory usage:

```bash
python -m soloai_llm_opt.serve \
  --model mistralai/Mistral-7B-Instruct-v0.2 \
  --quantization int4 \
  --batch-size 16
```

### 3. Python API Example

```python
from soloai_llm_opt import OptimizedLLMServer, GenerationConfig

server = OptimizedLLMServer(
    model_name="mistralai/Mistral-7B-Instruct-v0.2",
    dtype="float16",
    quantization="int4",
    batch_size=8,
)

server.start(host="0.0.0.0", port=8000)

config = GenerationConfig(
    max_tokens=128,
    temperature=0.3,
)

result = server.generate(
    prompt="Summarize the benefits of LLM inference optimization.",
    config=config,
)

print(result.text)
```

### 4. Kubernetes Deployment

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: soloai-llm-opt
spec:
  replicas: 1
  selector:
    matchLabels:
      app: soloai-llm-opt
  template:
    metadata:
      labels:
        app: soloai-llm-opt
    spec:
      containers:
        - name: server
          image: soloai/llm-deployment-optimization:latest
          ports:
            - containerPort: 8000
          args:
            - "--model"
            - "mistralai/Mistral-7B-Instruct-v0.2"
            - "--quantization"
            - "int4"
            - "--batch-size"
            - "16"
```

## Configuration

Example configuration file:

```yaml
model:
  name: mistralai/Mistral-7B-Instruct-v0.2
  dtype: float16
  quantization: int4

serving:
  host: 0.0.0.0
  port: 8000
  batch_size: 16
  max_input_tokens: 2048
  max_total_tokens: 4096

optimization:
  dynamic_batching: true
  kv_cache_optimization: true
  tensor_parallelism: 1

monitoring:
  prometheus: true
  structured_logging: true
```

Run with config:

```bash
python -m soloai_llm_opt.serve --config configs/production.yaml
```
