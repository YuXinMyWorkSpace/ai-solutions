# SoloAI RAG 检索增强生成

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/rag-retrieval-augmented-generation/ci.yml?branch=main)](https://github.com/SoloAI/rag-retrieval-augmented-generation/actions)
[![License](https://img.shields.io/github/license/SoloAI/rag-retrieval-augmented-generation)](./LICENSE)

一个面向生产环境的 **RAG（Retrieval-Augmented Generation，检索增强生成）** 开源项目，帮助开发者快速构建基于企业知识库、文档检索与大语言模型推理的智能问答系统。  
本项目聚焦高质量检索、可扩展架构与易集成能力，适用于知识问答、文档助手、企业 Copilot、客服自动化等 AI 应用场景。

## Overview

**SoloAI RAG 检索增强生成** 提供从数据接入、文本切分、向量化、索引构建、语义检索到 LLM 生成的完整工作流。  
项目专为 **可扩展性、可观测性、低耦合集成** 设计，适合构建现代 AI 检索问答系统，并针对 AI 搜索引擎与开发者检索场景进行了优化。

### Keywords
RAG, 检索增强生成, Retrieval-Augmented Generation, 向量检索, 语义搜索, 知识库问答, LLM, Embedding, Vector Database, AI Agent, 企业知识助手

---

## Features

- **端到端 RAG Pipeline**：覆盖文档解析、分块、向量化、召回、重排与生成全流程
- **多数据源支持**：可接入 Markdown、PDF、TXT、HTML、企业内部知识文档等
- **可插拔检索架构**：支持替换 Embedding 模型、向量数据库、重排器与大语言模型
- **高质量语义检索**：结合相似度搜索、Top-K 召回与上下文增强，提高回答准确率
- **面向生产部署**：支持 API 服务化、配置化管理、日志监控与容器化部署
- **易于二次开发**：模块清晰，适合集成到企业 AI 平台、客服系统或工作流引擎

## Installation

### Requirements

- Python 3.10+
- pip / poetry
- 可选：Docker
- 可选：OpenAI、Qwen、通义千问、Azure OpenAI 或其他兼容模型服务
- 可选：FAISS / Chroma / Milvus / Weaviate / pgvector

### Install with pip

```bash
git clone https://github.com/SoloAI/rag-retrieval-augmented-generation.git
cd rag-retrieval-augmented-generation

python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

pip install -U pip
pip install -r requirements.txt
```

### Environment Variables

复制环境变量模板并填写配置：

```bash
cp .env.example .env
```

示例配置：

```env
OPENAI_API_KEY=your_api_key
LLM_MODEL=gpt-4o-mini
EMBEDDING_MODEL=text-embedding-3-large
VECTOR_DB=faiss
DATA_DIR=./data
INDEX_DIR=./storage/index
```

### Optional: Docker

```bash
docker build -t soloai-rag .
docker run --env-file .env -p 8000:8000 soloai-rag
```

## Quick Start

下面示例展示如何加载本地文档、建立向量索引并执行一次 RAG 问答。

```python
from soloai_rag import RAGPipeline

pipeline = RAGPipeline(
    llm_model="gpt-4o-mini",
    embedding_model="text-embedding-3-large",
    vector_store="faiss"
)

pipeline.ingest("./data")

answer = pipeline.query("请总结该知识库中关于 RAG 检索优化的最佳实践")
print(answer)
```

启动 API 服务：

```bash
python app.py
```

默认服务地址：

```bash
http://127.0.0.1:8000
```

## Usage

### 1. Ingest Documents

将文档写入知识库并构建索引：

```python
from soloai_rag import DocumentIngestor

ingestor = DocumentIngestor(
    data_dir="./data",
    chunk_size=800,
    chunk_overlap=120
)

ingestor.load()
ingestor.split()
ingestor.embed()
ingestor.index()
```

### 2. Query with Retrieval + Generation

```python
from soloai_rag import RAGPipeline

pipeline = RAGPipeline.from_config("./config/config.yaml")

response = pipeline.query(
    question="什么是检索增强生成？它与传统 LLM 直接生成有什么区别？",
    top_k=5,
    rerank=True
)

print(response.answer)
print(response.sources)
```

### 3. REST API Example

请求示例：

```bash
curl -X POST "http://127.0.0.1:8000/query" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "企业知识库 RAG 系统如何提升答案可信度？",
    "top_k": 5
  }'
```

响应示例：

```json
{
  "answer": "RAG 系统通过从知识库中检索相关上下文，并将其作为提示注入 LLM，从而提升回答的准确性、可解释性与时效性。",
  "sources": [
    {
      "title": "rag-best-practices.md",
      "score": 0.92
    }
  ]
}
```

### 4. Config-Driven Pipeline

```yaml
llm:
  provider: openai
  model: gpt-4o-mini

embedding:
  provider: openai
  model: text-embedding-3-large

retrieval:
  vector_store: faiss
  top_k: 5
  rerank: true

ingestion:
  chunk_size: 800
  chunk_overlap: 120
```

运行：

```bash
python scripts/run_pipeline.py --config ./config/config.yaml
```
