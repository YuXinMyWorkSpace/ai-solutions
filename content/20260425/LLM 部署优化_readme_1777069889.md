**LLM 部署优化** is an open-source project by **SoloAI** focused on improving the performance, stability, and cost-efficiency of large language model deployment in production environments. It provides practical tooling and workflows for inference optimization, scaling, observability, and deployment automation across modern AI infrastructure.

## Features

- **High-performance inference optimization** for reduced latency and improved throughput
- **Quantization and model serving workflows** to lower GPU/CPU memory usage and serving cost
- **Scalable deployment architecture** for cloud, on-prem, and hybrid environments
- **Monitoring and observability integration** for metrics, logs, and runtime diagnostics
- **Benchmarking toolkit** for evaluating tokens/sec, latency, concurrency, and resource usage
- **Production-ready configuration patterns** for reproducible LLM deployment pipelines

## Installation

### Requirements

- Python 3.10+
- Docker 24+
- NVIDIA GPU recommended for production inference
- CUDA 11.8+ or compatible runtime
- Linux/macOS development environment

### Clone the repository

```bash
git clone https://github.com/SoloAI/llm-deploy-optimization.git
cd llm-deploy-optimization
```

### Install with pip

```bash
python -m venv .venv
source .venv/bin/activate  # On Windows: .venv\Scripts\activate
pip install -U pip
pip install -r requirements.txt
```

### Optional: Install development dependencies

```bash
pip install -r requirements-dev.txt
pre-commit install
```

### Run with Docker

```bash
docker build -t llm-deploy-optimization .
docker run --rm -p 8000:8000 llm-deploy-optimization
```

## Quick Start

Start a local optimized inference service:

```bash
python -m app.serve \
  --model /models/your-llm \
  --host 0.0.0.0 \
  --port 8000 \
  --dtype float16 \
  --max-batch-size 16 \
  --enable-metrics
```

Send a test inference request:

```bash
curl -X POST "http://127.0.0.1:8000/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "your-llm",
    "messages": [
      {"role": "system", "content": "You are a concise assistant."},
      {"role": "user", "content": "Explain LLM deployment optimization in one paragraph."}
    ],
    "temperature": 0.2,
    "max_tokens": 128
  }'
```

## Usage Examples

### 1. Benchmark inference performance

Use the built-in benchmark tool to measure latency, throughput, and concurrency:

```bash
python -m tools.benchmark \
  --endpoint http://127.0.0.1:8000/v1/chat/completions \
  --concurrency 32 \
  --requests 500 \
  --input-tokens 512 \
  --output-tokens 128
```

Example output:

```text
avg_latency_ms: 184.2
p95_latency_ms: 266.8
throughput_req_s: 42.7
throughput_tokens_s: 11890
gpu_memory_mb: 14320
```

### 2. Enable quantized deployment

Reduce memory footprint for production serving:

```bash
python -m app.serve \
  --model /models/your-llm \
  --quantization int8 \
  --max-batch-size 32 \
  --kv-cache-optimization paged
```

### 3. Deploy with Docker Compose

```yaml
version: "3.9"

services:
  llm-server:
    image: llm-deploy-optimization:latest
    ports:
      - "8000:8000"
    environment:
      MODEL_PATH: /models/your-llm
      DTYPE: float16
      MAX_BATCH_SIZE: 16
      ENABLE_METRICS: "true"
    volumes:
      - ./models:/models
    restart: unless-stopped
```

Run:

```bash
docker compose up -d
```

### 4. Python client example

```python
import requests

url = "http://127.0.0.1:8000/v1/chat/completions"
payload = {
    "model": "your-llm",
    "messages": [
        {"role": "user", "content": "How can I reduce LLM inference latency?"}
    ],
    "temperature": 0.1,
    "max_tokens": 100
}

response = requests.post(url, json=payload, timeout=60)
response.raise_for_status()
print(response.json())
```

### 5. Example production configuration

```yaml
server:
  host: 0.0.0.0
  port: 8000
  workers: 1

model:
  path: /models/your-llm
  dtype: float16
  quantization: int8

inference:
  max_batch_size: 32
  max_input_tokens: 4096
  max_output_tokens: 512
  enable_prefix_cache: true
  kv_cache_optimization: paged

monitoring:
  metrics: true
  tracing: true
  log_level: info
```
