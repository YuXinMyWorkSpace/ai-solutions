# LLM 部署优化

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/llm-deployment-optimization/ci.yml?branch=main)](https://github.com/SoloAI/llm-deployment-optimization/actions)
[![License](https://img.shields.io/github/license/SoloAI/llm-deployment-optimization)](./LICENSE)

面向生产环境的 **LLM 部署优化** 开源项目，专注于提升大语言模型在推理阶段的性能、稳定性与成本效率。  
由 **SoloAI** 维护，适用于 AI 应用落地、私有化部署、推理加速、资源调度与服务化场景。

---

## 项目简介

**LLM 部署优化** 提供一套面向工程实践的工具链与最佳实践，用于优化大语言模型的部署流程，包括模型加载、推理吞吐、显存管理、量化配置、并发服务和可观测性。  
本项目适合希望在本地、私有云、公有云或边缘环境中实现高性能 LLM 推理部署的开发者、平台团队和企业技术团队。

---

## 核心特性

- **高性能推理优化**：支持批处理、并发调度、KV Cache 优化与低延迟推理策略
- **多种部署模式**：适配本地 GPU、Kubernetes、Docker、私有化集群与云端推理服务
- **量化与显存优化**：支持 4-bit / 8-bit 量化、张量并行、混合精度与显存占用控制
- **可观测性与监控**：集成日志、指标采集、延迟追踪与吞吐分析，便于定位瓶颈
- **服务化接口**：提供 REST API / OpenAI-compatible API 方案，方便接入业务系统
- **工程化易扩展**：模块化设计，便于接入不同模型、推理引擎与部署平台

## 安装说明

### 环境要求

- Python 3.10+
- CUDA 11.8+（如使用 NVIDIA GPU）
- Linux/macOS（生产环境推荐 Linux）
- 可选：Docker 24+、Kubernetes 1.26+

### 方式一：从源码安装

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
python -m venv .venv
source .venv/bin/activate  # Windows 请使用 .venv\Scripts\activate
pip install -U pip
pip install -r requirements.txt
pip install -e .
```

### 方式二：使用 Docker

```bash
docker build -t soloai/llm-deployment-optimization:latest .
docker run --gpus all -p 8080:8080 soloai/llm-deployment-optimization:latest
```

## 快速开始

以下示例展示如何启动一个基础的 LLM 推理服务：

```bash
python -m app.server \
  --model /models/llama-3-8b-instruct \
  --host 0.0.0.0 \
  --port 8080 \
  --dtype float16 \
  --max-batch-size 16 \
  --tensor-parallel-size 1
```

服务启动后，可通过 HTTP 请求进行调用：

```bash
curl http://127.0.0.1:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "llama-3-8b-instruct",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user", "content": "请简要介绍一下 LLM 部署优化的核心思路。"}
    ],
    "temperature": 0.3,
    "max_tokens": 256
  }'
```

## 使用示例

### 1. 使用 YAML 配置启动服务

```yaml
server:
  host: 0.0.0.0
  port: 8080

model:
  name: llama-3-8b-instruct
  path: /models/llama-3-8b-instruct
  dtype: float16
  quantization: 4bit

runtime:
  max_batch_size: 32
  max_input_tokens: 4096
  max_output_tokens: 1024
  tensor_parallel_size: 2

observability:
  metrics: true
  tracing: true
  log_level: info
```

启动命令：

```bash
python -m app.server --config config.yaml
```

---

### 2. Python SDK 调用示例

```python
import requests

url = "http://127.0.0.1:8080/v1/chat/completions"
payload = {
    "model": "llama-3-8b-instruct",
    "messages": [
        {"role": "user", "content": "如何降低 LLM 推理延迟？"}
    ],
    "temperature": 0.2,
    "max_tokens": 128
}

resp = requests.post(url, json=payload, timeout=60)
print(resp.json())
```

---

### 3. 启用量化优化

```bash
python -m app.server \
  --model /models/llama-3-8b-instruct \
  --quantization 4bit \
  --dtype float16 \
  --max-batch-size 32
```

适用场景：

- GPU 显存有限的部署环境
- 多模型共存的推理服务
- 对吞吐与成本有严格要求的业务场景

---

### 4. Kubernetes 部署示例

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: llm-inference
spec:
  replicas: 1
  selector:
    matchLabels:
      app: llm-inference
  template:
    metadata:
      labels:
        app: llm-inference
    spec:
      containers:
        - name: llm-server
          image: soloai/llm-deployment-optimization:latest
          ports:
            - containerPort: 8080
          resources:
            limits:
              nvidia.com/gpu: 1
```
