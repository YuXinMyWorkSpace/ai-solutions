## 概述

**SoloAI RAG 检索增强生成** 是一个专注于文档检索、语义召回、上下文增强与大模型生成的开源框架。它支持从数据接入、向量化、索引构建到检索、重排、答案生成的完整 RAG pipeline，帮助团队更高效地落地 LLM + Knowledge Base 应用。

## 核心特性

- **模块化 RAG Pipeline**：覆盖文档加载、切分、Embedding、向量检索、重排、生成与结果追踪。
- **多数据源支持**：支持 Markdown、PDF、TXT、HTML、数据库及企业内部知识文档接入。
- **高质量语义检索**：支持向量检索、关键词检索、混合检索与可选 reranker 提升召回精度。
- **可扩展架构**：兼容主流 Embedding 模型、向量数据库与大语言模型接口。
- **生产级可观测性**：支持日志、调试、检索链路分析与回答来源追踪。
- **面向企业落地**：适合知识库问答、FAQ 自动化、行业文档分析、私有化 RAG 部署。

## 安装

### 环境要求

- Python 3.10+
- pip 或 poetry
- 可选：Docker / Docker Compose
- 可选：OpenAI 兼容模型接口、向量数据库服务

### 使用 pip 安装

```bash
git clone https://github.com/SoloAI/rag-retrieval-augmented-generation.git
cd rag-retrieval-augmented-generation

python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

pip install -U pip
pip install -r requirements.txt
```

### 配置环境变量

复制示例配置文件：

```bash
cp .env.example .env
```

示例 `.env`：

```env
OPENAI_API_KEY=your_api_key
OPENAI_BASE_URL=https://api.openai.com/v1
EMBEDDING_MODEL=text-embedding-3-large
CHAT_MODEL=gpt-4o-mini

VECTOR_DB=faiss
TOP_K=5
CHUNK_SIZE=500
CHUNK_OVERLAP=100
```

### 使用 Docker

```bash
docker build -t soloai-rag .
docker run --rm -p 8000:8000 --env-file .env soloai-rag
```

## 快速开始

以下示例展示如何使用 SoloAI RAG 对本地文档建立索引，并基于检索结果生成答案。

```python
from soloai_rag import RAGPipeline

pipeline = RAGPipeline(
    embedding_model="text-embedding-3-large",
    chat_model="gpt-4o-mini",
    vector_store="faiss"
)

pipeline.ingest("./data")

response = pipeline.query(
    question="RAG 检索增强生成的核心流程是什么？",
    top_k=5
)

print(response.answer)
print(response.sources)
```

## 使用示例

### 1. 构建文档索引

```python
from soloai_rag import DocumentLoader, IndexBuilder

loader = DocumentLoader(path="./docs", recursive=True)
documents = loader.load()

index_builder = IndexBuilder(
    embedding_model="text-embedding-3-large",
    vector_store="faiss"
)
index_builder.build(documents)
index_builder.save("./storage/index")
```

### 2. 加载索引并执行问答

```python
from soloai_rag import RAGPipeline

pipeline = RAGPipeline.load("./storage/index")

result = pipeline.query(
    question="如何优化企业知识库的召回率？",
    top_k=8,
    use_reranker=True
)

print("Answer:", result.answer)
for source in result.sources:
    print("-", source)
```

### 3. 使用 API 服务模式

```bash
python -m app.server
```

示例请求：

```bash
curl -X POST http://localhost:8000/query \
  -H "Content-Type: application/json" \
  -d '{
    "question": "请总结该知识库中关于混合检索的最佳实践",
    "top_k": 5
  }'
```

示例响应：

```json
{
  "answer": "混合检索结合了关键词检索与向量检索，可以在保证语义召回的同时提高精确匹配能力。",
  "sources": [
    "docs/rag_overview.md",
    "docs/hybrid_search.md"
  ],
  "latency_ms": 842
}
```

### 4. 混合检索配置示例

```python
pipeline = RAGPipeline(
    embedding_model="text-embedding-3-large",
    chat_model="gpt-4o-mini",
    vector_store="faiss",
    retrieval_mode="hybrid"
)

result = pipeline.query("如何降低大模型幻觉并提升可追溯性？")
print(result.answer)
```
