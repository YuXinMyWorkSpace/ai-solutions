# SoloAI RAG 检索增强生成

> SoloAI RAG 提供从文档解析、向量检索、重排序到大模型生成的完整技术链路，适用于企业级 AI 知识增强场景。

## 简介

**SoloAI RAG 检索增强生成** 是一个专注于中文与多语言知识库问答的开源框架，旨在提升大模型在私域数据、行业文档和企业知识场景中的回答准确率、可解释性与可控性。  
项目支持灵活接入主流向量数据库、Embedding 模型与大语言模型，适合搭建 AI 搜索、智能客服、企业 Copilot、文档问答等系统。

---

## 核心特性

- **完整 RAG 流水线**：覆盖文档加载、切分、向量化、索引构建、召回、重排序、答案生成全流程
- **多数据源接入**：支持 Markdown、PDF、TXT、HTML、网页内容及企业内部知识文档
- **可扩展检索架构**：兼容主流向量数据库与 Embedding/LLM Provider，便于按需替换
- **面向中文优化**：针对中文语义检索、长文档问答与知识增强生成进行优化
- **生产级可观测性**：便于接入日志、监控、检索调试与效果评估
- **适配多业务场景**：可用于企业知识库、智能问答、客服机器人、行业助手与 AI 搜索

## 安装

### 环境要求

- Python 3.10+
- pip 或 poetry
- 可选：Docker / Docker Compose
- 可选：向量数据库（如 FAISS、Milvus、Qdrant、Chroma 等）

### 使用 pip 安装

```bash
git clone https://github.com/SoloAI/rag.git
cd rag
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -U pip
pip install -r requirements.txt
```

### 使用 poetry 安装

```bash
git clone https://github.com/SoloAI/rag.git
cd rag
poetry install
poetry shell
```

### 环境变量配置

创建 `.env` 文件：

```env
OPENAI_API_KEY=your_api_key
EMBEDDING_MODEL=text-embedding-3-large
LLM_MODEL=gpt-4o-mini
VECTOR_DB=faiss
```

## 快速开始

下面示例展示如何将本地知识文档导入索引，并基于 RAG 执行问答。

```python
from rag.pipeline import RAGPipeline

pipeline = RAGPipeline(
    embedding_model="text-embedding-3-large",
    llm_model="gpt-4o-mini",
    vector_store="faiss"
)

# 1. 构建知识库索引
pipeline.index(
    source_path="./data/knowledge",
    chunk_size=500,
    chunk_overlap=50
)

# 2. 执行检索增强生成
response = pipeline.query(
    question="RAG 检索增强生成适合哪些企业知识库场景？",
    top_k=5
)

print(response.answer)
print(response.sources)
```

运行：

```bash
python examples/quickstart.py
```

## 使用示例

### 1. 导入 PDF 与 Markdown 文档

```python
from rag.pipeline import RAGPipeline

pipeline = RAGPipeline(
    embedding_model="bge-large-zh",
    llm_model="gpt-4o-mini",
    vector_store="qdrant"
)

pipeline.index(source_path="./docs")
```

支持目录中的：

- `.pdf`
- `.md`
- `.txt`
- `.html`

---

### 2. 构建企业知识库问答

```python
answer = pipeline.query(
    question="我们的售后服务流程是什么？",
    top_k=3,
    return_sources=True
)

print("Answer:", answer.answer)
for source in answer.sources:
    print(source)
```

适用于：

- 企业内部知识库
- SOP/流程文档检索
- 员工培训助手
- 客服知识问答

---

### 3. 开启重排序提升召回质量

```python
pipeline = RAGPipeline(
    embedding_model="bge-large-zh",
    reranker_model="bge-reranker-base",
    llm_model="gpt-4o-mini",
    vector_store="milvus"
)

result = pipeline.query(
    question="如何进行合同审批？",
    top_k=10,
    rerank_top_n=5
)

print(result.answer)
```

---

### 4. 以 API 服务方式启动

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

示例请求：

```bash
curl -X POST "http://localhost:8000/query" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "RAG 如何降低大模型幻觉？",
    "top_k": 5
  }'
```

示例响应：

```json
{
  "answer": "RAG 通过先检索外部知识，再基于检索结果生成答案，从而降低模型幻觉并提升可解释性。",
  "sources": [
    {
      "file": "rag-intro.md",
      "score": 0.92
    }
  ]
}
```

## 配置说明

可通过 `configs/config.yaml` 管理核心参数：

```yaml
embedding:
  provider: openai
  model: text-embedding-3-large

llm:
  provider: openai
  model: gpt-4o-mini
  temperature: 0.2

retrieval:
  top_k: 5
  chunk_size: 500
  chunk_overlap: 50

rerank:
  enabled: true
  top_n: 5

vector_store:
  type: faiss
```
