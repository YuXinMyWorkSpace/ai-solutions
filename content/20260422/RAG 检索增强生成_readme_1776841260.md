# SoloAI RAG 检索增强生成

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/rag-retrieval-augmented-generation/ci.yml?branch=main)](https://github.com/SoloAI/rag-retrieval-augmented-generation/actions)
[![License](https://img.shields.io/github/license/SoloAI/rag-retrieval-augmented-generation)](./LICENSE)

一个面向生产环境的 **RAG（Retrieval-Augmented Generation，检索增强生成）** 开源框架，用于构建高质量、可扩展、可观测的知识问答与 AI 应用。  
由 **SoloAI** 打造，帮助开发者快速集成文档检索、向量索引、上下文增强和大语言模型生成能力。

---

## 概述

**SoloAI RAG 检索增强生成** 提供从文档摄取、向量化、检索召回到上下文增强生成的完整工作流，适用于企业知识库、智能客服、搜索问答、内部 Copilot 和垂直领域 AI 助手等场景。

该项目专注于以下核心能力：

- 文档分块与嵌入向量生成
- 多种向量数据库/检索后端集成
- 基于语义检索的上下文召回
- 大模型生成与答案可追溯
- 面向生产环境的模块化架构与 API 设计

---

## 功能特性

- **模块化 RAG Pipeline**：支持文档加载、切分、嵌入、索引、检索、重排与生成的标准化流程
- **多后端检索支持**：可接入 FAISS、Chroma、Milvus、Elasticsearch 等向量检索引擎
- **高质量上下文增强**：通过 Top-K 检索、相似度过滤和可选 reranking 提升回答准确率
- **可观测与可追溯**：返回引用片段、来源文档和检索元数据，便于审计与调试
- **易于集成**：支持 Python SDK、CLI 和 REST API，适配现有 AI/LLM 应用栈
- **面向生产部署**：支持容器化部署、配置化管理和可扩展服务架构

## 快速开始

下面的示例展示如何使用 SoloAI RAG 构建一个最小可用的检索增强生成流程。

```python
from soloai_rag import RAGPipeline
from soloai_rag.document_loaders import DirectoryLoader
from soloai_rag.vectorstores import FAISSStore

# 1. 加载文档
loader = DirectoryLoader("./docs", glob="**/*.md")
documents = loader.load()

# 2. 初始化 RAG Pipeline
rag = RAGPipeline(
    vector_store=FAISSStore(index_path="./data/index"),
    embedding_model="text-embedding-3-large",
    llm_model="gpt-4o-mini",
    chunk_size=800,
    chunk_overlap=100,
)

# 3. 构建索引
rag.ingest(documents)

# 4. 发起查询
response = rag.query(
    "RAG 检索增强生成的核心流程是什么？",
    top_k=5,
    return_sources=True,
)

print(response.answer)
for source in response.sources:
    print(source.metadata)
```

## 使用示例

### 1. 构建本地知识库

将 PDF、Markdown、TXT 等文档导入并建立向量索引：

```python
from soloai_rag import RAGPipeline
from soloai_rag.document_loaders import PDFLoader, TextLoader

documents = []
documents.extend(PDFLoader("./knowledge/whitepaper.pdf").load())
documents.extend(TextLoader("./knowledge/faq.txt").load())

rag = RAGPipeline.from_config({
    "embedding_model": "text-embedding-3-large",
    "llm_model": "gpt-4o-mini",
    "vector_store": {
        "provider": "chroma",
        "persist_directory": "./chroma_db"
    }
})

rag.ingest(documents)
```

### 2. 语义检索 + 问答生成

```python
response = rag.query(
    query="如何提升检索召回率与生成答案的准确性？",
    top_k=8,
    score_threshold=0.75,
    return_sources=True
)

print("Answer:", response.answer)
print("Sources:")
for item in response.sources:
    print("-", item.metadata.get("source"), item.score)
```

### 3. 使用 CLI 建立索引

```bash
soloai-rag ingest \
  --input ./docs \
  --pattern "**/*.md" \
  --vector-store faiss \
  --output ./data/index
```

### 4. 启动 API 服务

```bash
soloai-rag serve --host 0.0.0.0 --port 8080
```

请求示例：

```bash
curl -X POST http://localhost:8080/query \
  -H "Content-Type: application/json" \
  -d '{
    "query": "什么是检索增强生成？",
    "top_k": 5,
    "return_sources": true
  }'
```

示例响应：

```json
{
  "answer": "检索增强生成（RAG）是一种结合外部知识检索与大语言模型生成的架构，用于提升回答的准确性、时效性与可解释性。",
  "sources": [
    {
      "content": "RAG 通过先检索相关文档片段，再将其作为上下文输入给 LLM。",
      "metadata": {
        "source": "./docs/intro.md"
      },
      "score": 0.92
    }
  ]
}
```
