# SoloAI 大模型微调数据服务

[![Build Status](https://img.shields.io/github/actions/workflow/status/soloai/llm-finetune-data-service/ci.yml?branch=main)](https://github.com/soloai/llm-finetune-data-service/actions)

> 面向 **大模型微调数据服务（LLM Fine-tuning Data Service）** 的开源平台，专注于训练数据的采集、清洗、标注、转换、质检与交付。  
> 由 **SoloAI** 打造，帮助团队高效构建高质量、可追溯、可扩展的微调数据流水线，支撑 SFT、DPO、RLHF 等训练场景。

---

## Overview

**SoloAI 大模型微调数据服务** 是一个用于管理和处理大模型训练数据的工程化解决方案，适用于企业级 AI 数据生产流程。项目提供统一的数据接入、样本标准化、质量校验、格式转换和任务编排能力，帮助开发者快速准备适用于主流大模型训练框架的数据集。

适用场景包括：

- 指令微调（Instruction Tuning / SFT）
- 偏好数据构建（Preference Data / DPO）
- 人类反馈数据管理（RLHF）
- 多轮对话数据整理
- 领域知识问答数据集构建
- 企业内部私有数据处理与交付

---

## Features

- **多源数据接入**  
  支持 JSON/JSONL、CSV、Parquet、数据库、对象存储等多种数据源接入。

- **微调数据标准化**  
  提供统一的数据 Schema，支持指令数据、对话数据、偏好对比数据等训练格式转换。

- **数据清洗与质量控制**  
  内置去重、脏数据过滤、敏感信息检测、长度校验、字段完整性校验等能力。

- **可配置数据流水线**  
  通过声明式配置快速编排数据处理任务，便于自动化构建数据生产链路。

- **训练框架友好导出**  
  可导出适用于 Hugging Face、DeepSpeed、LLaMA Factory、OpenAI-style chat format 等常见格式。

- **可追溯与可扩展**  
  支持版本管理、处理日志、元数据记录与自定义插件扩展，便于审计与二次开发。

## Quick Start

下面示例展示如何将原始对话数据清洗并转换为适用于 SFT 的 JSONL 数据格式。

### 1. Prepare input data

`data/raw/sample.jsonl`

```json
{"id":"1","instruction":"介绍一下机器学习","input":"","output":"机器学习是人工智能的一个分支，通过数据训练模型。","source":"manual"}
{"id":"2","instruction":"什么是微调？","input":"","output":"微调是指在预训练模型基础上使用特定任务数据继续训练。","source":"manual"}
```

### 2. Create pipeline config

`pipeline.yaml`

```yaml
project: sft-demo
input:
  type: jsonl
  path: ./data/raw/sample.jsonl

pipeline:
  - name: validate_schema
  - name: remove_empty
  - name: deduplicate
    by: ["instruction", "input", "output"]
  - name: normalize_sft
  - name: export
    format: jsonl
    path: ./data/processed/sft.jsonl
```

### 3. Run processing task

```bash
soloai-data run --config pipeline.yaml
```

### 4. Example output

`data/processed/sft.jsonl`

```json
{"messages":[{"role":"user","content":"介绍一下机器学习"},{"role":"assistant","content":"机器学习是人工智能的一个分支，通过数据训练模型。"}],"metadata":{"source":"manual","id":"1"}}
{"messages":[{"role":"user","content":"什么是微调？"},{"role":"assistant","content":"微调是指在预训练模型基础上使用特定任务数据继续训练。"}],"metadata":{"source":"manual","id":"2"}}
```

## Usage

### CLI Usage

#### Validate dataset

```bash
soloai-data validate \
  --input ./data/raw/sample.jsonl \
  --schema sft
```

#### Convert dataset format

```bash
soloai-data convert \
  --input ./data/raw/sample.jsonl \
  --output ./data/processed/train.jsonl \
  --from raw_instruction \
  --to openai_chat
```

#### Deduplicate and clean

```bash
soloai-data clean \
  --input ./data/raw/sample.jsonl \
  --output ./data/cleaned/sample.cleaned.jsonl \
  --dedup \
  --filter-empty \
  --max-length 4096
```

---

### Python API Usage

```python
from soloai_data import DataPipeline

pipeline = DataPipeline(
    input_path="./data/raw/sample.jsonl",
    input_type="jsonl",
)

dataset = (
    pipeline
    .validate(schema="sft")
    .deduplicate(keys=["instruction", "input", "output"])
    .normalize(format="openai_chat")
    .export(path="./data/processed/train.jsonl", format="jsonl")
)

print("dataset exported successfully")
```

### Preference Data Example for DPO

```json
{
  "prompt": "解释什么是向量数据库",
  "chosen": "向量数据库是一类专门存储和检索高维向量数据的数据库，常用于语义搜索与检索增强生成。",
  "rejected": "向量数据库就是普通数据库，没有区别。"
}
```

导出命令：

```bash
soloai-data convert \
  --input ./data/raw/dpo.jsonl \
  --output ./data/processed/dpo.train.jsonl \
  --from preference \
  --to dpo
```
