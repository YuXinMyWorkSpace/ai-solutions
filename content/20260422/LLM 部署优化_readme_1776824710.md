# SoloAI LLM 部署优化

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/llm-deployment-optimization/ci.yml?branch=main)](https://github.com/SoloAI/llm-deployment-optimization/actions)
[![License](https://img.shields.io/github/license/SoloAI/llm-deployment-optimization)](./LICENSE)

面向生产环境的 **LLM Deployment Optimization** 开源项目，专注于大语言模型（LLM）在推理部署阶段的性能调优、资源利用优化与可观测性增强。  
本项目帮助开发者和平台工程团队更高效地完成 **模型服务化、低延迟推理、吞吐提升、显存优化与弹性扩缩容**。

---

## 概述

**SoloAI LLM 部署优化** 提供一套可扩展的工具链与最佳实践，用于优化大模型在线推理服务，适用于 AI Infra、MLOps、AIGC 平台和企业级私有化部署场景。  
项目重点覆盖以下方向：

- LLM inference optimization
- GPU / CPU resource scheduling
- Quantization-aware deployment
- Batch inference and request scheduling
- Observability and performance benchmarking
- Production-ready serving architecture

---

## 核心特性

- **高性能推理优化**  
  支持面向吞吐（throughput）与延迟（latency）的部署策略优化，包括动态 batch、prefill/decode 分离调优等。

- **显存与资源利用率提升**  
  通过量化、KV Cache 管理、模型分片与并行策略降低显存占用，提高 GPU 利用率。

- **多部署后端兼容**  
  可集成常见推理引擎与服务框架，如 vLLM、TensorRT-LLM、Transformers、FastAPI 等。

- **性能基准测试与分析**  
  提供 benchmark 工具，帮助评估 QPS、TTFT、tokens/s、P95/P99 延迟等关键指标。

- **可观测性与运维友好**  
  支持日志、metrics、tracing 接入，便于定位瓶颈与构建生产级监控体系。

- **模块化架构设计**  
  便于根据实际业务需求扩展调度器、缓存策略、部署配置与服务接口。

## 安装

### 环境要求

- Python 3.10+
- CUDA 11.8+（如使用 GPU）
- Linux x86_64
- 推荐：NVIDIA GPU with 16GB+ VRAM

### 方式一：使用 pip 安装

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization

python -m venv .venv
source .venv/bin/activate

pip install -U pip
pip install -r requirements.txt
pip install -e .
```

### 方式二：开发模式安装

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization

pip install -r requirements-dev.txt
pre-commit install
pip install -e ".[dev]"
```

### 可选依赖

根据不同后端启用对应依赖：

```bash
# vLLM support
pip install -e ".[vllm]"

# TensorRT-LLM support
pip install -e ".[trtllm]"

# Observability support
pip install -e ".[monitoring]"
```

## 快速开始

以下示例展示如何使用 SoloAI 启动一个经过优化的 LLM 推理服务。

```python
from soloai_llm_opt.runtime import OptimizedLLMServer
from soloai_llm_opt.scheduler import DynamicBatchScheduler

server = OptimizedLLMServer(
    model="Qwen/Qwen2-7B-Instruct",
    device="cuda",
    max_batch_size=16,
    max_input_length=4096,
    quantization="int4"
)

scheduler = DynamicBatchScheduler(
    max_batch_size=16,
    max_wait_ms=20
)

server.attach_scheduler(scheduler)
server.start(host="0.0.0.0", port=8000)
```

启动服务：

```bash
python examples/run_server.py
```

调用接口：

```bash
curl -X POST http://127.0.0.1:8000/generate \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "请解释一下 LLM 推理优化中的 KV Cache 是什么？",
    "max_tokens": 128,
    "temperature": 0.7
  }'
```

## 使用示例

### 1. 启用量化部署

通过低比特量化降低显存占用并提升部署效率：

```python
from soloai_llm_opt.runtime import OptimizedLLMServer

server = OptimizedLLMServer(
    model="meta-llama/Llama-3-8B-Instruct",
    device="cuda",
    quantization="int4",
    gpu_memory_utilization=0.92
)

server.start()
```

---

### 2. 启用动态批处理

适用于高并发在线推理场景：

```python
from soloai_llm_opt.scheduler import DynamicBatchScheduler

scheduler = DynamicBatchScheduler(
    max_batch_size=32,
    max_wait_ms=15,
    target="throughput"
)
```

---

### 3. 运行性能基准测试

评估模型部署后的吞吐与延迟表现：

```bash
python benchmarks/run_benchmark.py \
  --endpoint http://127.0.0.1:8000/generate \
  --concurrency 32 \
  --requests 500 \
  --input-tokens 512 \
  --output-tokens 128
```

示例输出：

```text
Benchmark Summary
-----------------
QPS: 18.7
Avg Latency: 842 ms
P95 Latency: 1284 ms
P99 Latency: 1639 ms
TTFT: 221 ms
Throughput: 1240 tokens/s
GPU Utilization: 91%
```

---

### 4. 接入监控指标

将关键推理指标暴露给 Prometheus：

```python
from soloai_llm_opt.observability import MetricsExporter

metrics = MetricsExporter(port=9090)
metrics.enable_default_metrics()

# Exposed metrics:
# - llm_requests_total
# - llm_request_latency_ms
# - llm_ttft_ms
# - llm_tokens_generated_total
# - gpu_memory_usage_bytes
```

## 配置说明

你可以通过 YAML 文件管理部署参数：

```yaml
model:
  name: Qwen/Qwen2-7B-Instruct
  quantization: int4
  max_input_length: 4096

runtime:
  device: cuda
  tensor_parallel_size: 2
  gpu_memory_utilization: 0.9

scheduler:
  type: dynamic_batch
  max_batch_size: 16
  max_wait_ms: 20

observability:
  metrics: true
  tracing: false
  log_level: info
```

加载配置：

```bash
python examples/run_server.py --config configs/production.yaml
```
