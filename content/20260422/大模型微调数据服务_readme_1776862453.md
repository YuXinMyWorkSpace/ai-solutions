# 🤖 SoloAI Fine-Tuning Data Service (大模型微调数据服务)

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/fine-tuning-data-service/ci.yml?branch=main&label=build&style=flat-square)](https://github.com/SoloAI/fine-tuning-data-service/actions)
[![License](https://img.shields.io/github/license/SoloAI/fine-tuning-data-service?style=flat-square)](https://github.com/SoloAI/fine-tuning-data-service/blob/main/LICENSE)

An enterprise-grade data pipeline and management framework engineered to streamline the preparation, validation, and distribution of high-quality datasets for Large Language Model (LLM) fine-tuning. SoloAI's service ensures data consistency, mitigates alignment risks, and accelerates instruction-tuning, RLHF, and domain-adaptation workflows.

## ✨ Key Features

- **Automated Data Curation & Cleaning**: Deduplication (MinHash/SimHash), PII redaction, whitespace normalization, and toxic content filtering.
- **Multi-Format Schema Validation**: Native support for JSONL, Parquet, CSV, and Hugging Face `datasets` with strict JSON Schema enforcement.
- **LLM-Assisted Quality Scoring**: Built-in perplexity filtering, instruction-response alignment checks, and configurable reward-model scoring.
- **Distributed & Cloud-Native Processing**: Scalable via Dask/Ray backends; supports AWS S3, GCS, and Azure Blob Storage out-of-the-box.
- **Framework-Agnostic Export**: One-click conversion to Alpaca, ShareGPT, ChatML, and Axolotl-compatible formats.
- **Dataset Versioning & Provenance**: Git-like dataset tracking, immutable snapshots, and full audit trails for compliance and reproducibility.

## 🚀 Quick Start

Initialize a data pipeline, apply transformations, and export a fine-tuning-ready dataset in under 10 lines:

```python
from soloai.data import Pipeline

# Load raw instruction data
pipeline = Pipeline(source="raw_conversations.jsonl")

# Chain transformations & validation
processed = (
    pipeline
    .clean(deduplicate=True, redact_pii=True)
    .filter(min_tokens=64, max_tokens=4096)
    .format(schema="chatml")
    .validate(strict=True)
)

# Export to disk
processed.export("output/fine_tune_ready.jsonl")
print(f"✅ Processed {len(processed)} high-quality samples.")
```

## 📖 Usage Examples

### 1. LLM-Based Quality Filtering
```python
from soloai.data import QualityScorer, DatasetHub

# Score samples using a reference model
scorer = QualityScorer(model="gpt-4o-mini", threshold=0.85, batch_size=32)
scored_dataset = scorer.evaluate(processed)

# Filter low-confidence entries
verified = scored_dataset.filter(lambda x: x["quality_score"] >= 0.85)
```

### 2. Push to Hugging Face Hub
```python
hub = DatasetHub(token="hf_your_token_here")

hub.push_dataset(
    repo_id="soloai/domain-instruct-v1",
    dataset=verified,
    split="train",
    tags=["instruction-tuning", "verified", "domain-adaptation"],
    description="Curated dataset for financial domain LLM alignment."
)
```

### 3. Distributed Processing (Ray Backend)
```python
from soloai.data import Pipeline
from soloai.backends import RayExecutor

# Enable distributed execution for >10M samples
executor = RayExecutor(num_workers=16, memory_per_worker="8G")
pipeline = Pipeline(source="s3://bucket/raw-data.parquet", executor=executor)

# Process at scale
large_dataset = pipeline.clean().format(schema="alpaca").validate()
large_dataset.export("s3://bucket/processed/")
```
