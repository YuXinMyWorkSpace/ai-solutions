# SoloAI 大模型微调数据服务

[![Build Status](https://img.shields.io/badge/build-passing-brightgreen.svg)](#)  
[![License](https://img.shields.io/badge/license-Apache%202.0-blue.svg)](./LICENSE)

面向 **大模型微调数据服务** 的开源平台，提供从数据采集、清洗、标注、格式转换到数据集管理的完整工作流，帮助团队高效构建高质量 LLM Fine-tuning 数据集。  
由 **SoloAI** 打造，适用于监督微调（SFT）、指令微调、偏好数据构建、对话数据处理与企业级数据服务场景。

---

## 项目简介

**SoloAI 大模型微调数据服务** 是一个专注于 LLM 训练与微调数据生命周期管理的开源项目。它支持多源数据接入、自动清洗、数据标准化、质量评估与导出，帮助研发团队快速构建可用于 GPT、LLaMA、Qwen、Baichuan 等模型微调的数据资产。

---

## 核心特性

- **多源数据接入**：支持 JSON、JSONL、CSV、Parquet、数据库及对象存储等多种数据源。
- **数据清洗与标准化**：内置去重、脱敏、分段、字段映射、模板化转换与异常样本过滤能力。
- **面向微调任务的数据格式支持**：兼容 SFT、Instruction Tuning、ChatML、Alpaca、ShareGPT 等主流格式。
- **数据质量评估**：支持规则校验、统计分析、标签完整性检查与样本质量评分。
- **任务化处理流水线**：通过可配置 Pipeline 对数据导入、处理、验证、导出进行自动化编排。
- **开放集成能力**：提供 REST API、CLI 与 Python SDK，便于集成至 MLOps / DataOps / LLMOps 工作流。

## 快速开始

以下示例展示如何将原始问答数据转换为适用于监督微调（SFT）的标准数据格式。

### 1. 准备原始数据

`data/raw.jsonl`

```json
{"question":"什么是大模型微调？","answer":"大模型微调是指在预训练模型基础上，使用特定领域数据继续训练，以适配特定任务。"}
{"question":"什么是SFT？","answer":"SFT 指监督微调，通过带标签样本对模型进行有监督训练。"}
```

### 2. 编写配置文件

`config/pipeline.yaml`

```yaml
source:
  type: jsonl
  path: data/raw.jsonl

transform:
  - type: rename_fields
    mapping:
      question: input
      answer: output

  - type: template
    template:
      instruction: "请回答以下问题"
      input: "{{ input }}"
      output: "{{ output }}"

validate:
  - type: required_fields
    fields: [instruction, output]

export:
  type: jsonl
  path: data/sft_dataset.jsonl
  format: alpaca
```

### 3. 执行数据处理任务

```bash
soloai-data run -c config/pipeline.yaml
```

### 4. 输出结果示例

```json
{"instruction":"请回答以下问题","input":"什么是大模型微调？","output":"大模型微调是指在预训练模型基础上，使用特定领域数据继续训练，以适配特定任务。"}
{"instruction":"请回答以下问题","input":"什么是SFT？","output":"SFT 指监督微调，通过带标签样本对模型进行有监督训练。"}
```

## 使用示例

## 1. CLI 使用

### 查看帮助

```bash
soloai-data --help
```

### 导入数据集

```bash
soloai-data import \
  --source ./datasets/customer_support.csv \
  --name customer-support-v1 \
  --format csv
```

### 执行清洗与校验

```bash
soloai-data process \
  --dataset customer-support-v1 \
  --steps dedup,desensitize,validate
```

### 导出为 ShareGPT 格式

```bash
soloai-data export \
  --dataset customer-support-v1 \
  --format sharegpt \
  --output ./exports/customer_support_sharegpt.json
```

---

## 2. Python SDK 使用

```python
from soloai_data import Pipeline

pipeline = Pipeline.from_dict({
    "source": {
        "type": "jsonl",
        "path": "data/raw.jsonl"
    },
    "transform": [
        {
            "type": "rename_fields",
            "mapping": {
                "question": "input",
                "answer": "output"
            }
        }
    ],
    "validate": [
        {
            "type": "required_fields",
            "fields": ["input", "output"]
        }
    ],
    "export": {
        "type": "jsonl",
        "path": "data/output.jsonl",
        "format": "sft"
    }
})

result = pipeline.run()
print(result.summary())
```

## 3. REST API 使用

### 启动服务

```bash
uvicorn soloai_data.api:app --host 0.0.0.0 --port 8080
```

### 创建数据处理任务

```bash
curl -X POST http://localhost:8080/api/v1/jobs \
  -H "Content-Type: application/json" \
  -d '{
    "name": "sft-data-job",
    "source": {
      "type": "jsonl",
      "path": "data/raw.jsonl"
    },
    "transform": [
      {
        "type": "rename_fields",
        "mapping": {
          "question": "input",
          "answer": "output"
        }
      }
    ],
    "export": {
      "type": "jsonl",
      "path": "data/output.jsonl",
      "format": "alpaca"
    }
  }'
```

### 查询任务状态

```bash
curl http://localhost:8080/api/v1/jobs/sft-data-job
```
