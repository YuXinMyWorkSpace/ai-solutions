# SoloAI LLM 部署优化

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/llm-deployment-optimizer/ci.yml?branch=main&style=flat-square)](https://github.com/SoloAI/llm-deployment-optimizer/actions)

高性能、可扩展的 **LLM 部署优化** 工具链，专注于提升大语言模型在生产环境中的 **推理吞吐、响应延迟、资源利用率与部署稳定性**。  
由 **SoloAI** 开源维护，适用于本地部署、GPU 集群、容器化平台及云原生推理服务。

---

## 概述

**SoloAI LLM 部署优化** 是一个面向生产环境的大模型推理优化项目，帮助开发者与平台工程团队快速实现模型部署、量化压缩、批处理调度、缓存优化与可观测性集成。  
该项目适用于希望优化 **LLM inference performance**, **GPU utilization**, **latency reduction**, **throughput scaling**, 与 **cost-efficient deployment** 的团队。

---

## 功能特性

- **多维推理性能优化**：支持批处理、并发调度、KV Cache 优化与请求路由策略
- **量化与模型压缩**：支持 INT8 / 4-bit 等常见量化部署策略，降低显存占用
- **多后端部署适配**：兼容本地 GPU、Docker、Kubernetes 与主流推理框架
- **可观测性支持**：集成 Prometheus 指标、日志追踪与性能分析
- **自动化配置调优**：基于硬件与模型规模推荐部署参数
- **生产级工程能力**：支持 CI/CD、配置管理、服务健康检查与滚动更新

## 安装

### 环境要求

- Python 3.10+
- CUDA 11.8+（如使用 NVIDIA GPU）
- Docker 24+（可选）
- Kubernetes 1.26+（可选，用于集群部署）

### 方式一：使用 pip 安装

```bash
git clone https://github.com/SoloAI/llm-deployment-optimizer.git
cd llm-deployment-optimizer
pip install -U pip
pip install -r requirements.txt
pip install -e .
```

### 方式二：使用 Docker

```bash
git clone https://github.com/SoloAI/llm-deployment-optimizer.git
cd llm-deployment-optimizer
docker build -t soloai/llm-deployment-optimizer:latest .
docker run --gpus all -p 8080:8080 soloai/llm-deployment-optimizer:latest
```

### 验证安装

```bash
soloai-llm-opt --help
```

## 快速开始

以下示例展示如何使用默认配置启动一个优化后的 LLM 推理服务：

```bash
soloai-llm-opt serve \
  --model meta-llama/Llama-2-7b-chat-hf \
  --host 0.0.0.0 \
  --port 8080 \
  --tensor-parallel-size 1 \
  --max-batch-size 16 \
  --kv-cache-policy dynamic
```

服务启动后，可通过 HTTP API 发起推理请求：

```bash
curl -X POST "http://localhost:8080/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "meta-llama/Llama-2-7b-chat-hf",
    "messages": [
      {"role": "user", "content": "请解释一下什么是 LLM 部署优化？"}
    ],
    "temperature": 0.7,
    "max_tokens": 256
  }'
```

## 使用示例

### 1. 自动推荐部署参数

根据模型大小与硬件资源生成建议配置：

```bash
soloai-llm-opt tune \
  --model meta-llama/Llama-2-13b-chat-hf \
  --gpu-memory 24 \
  --target latency \
  --output config.generated.yaml
```

示例输出配置：

```yaml
model: meta-llama/Llama-2-13b-chat-hf
precision: int8
max_batch_size: 8
tensor_parallel_size: 2
kv_cache_policy: dynamic
prefill_optimization: true
```

### 2. 使用配置文件启动服务

创建配置文件 `config.yaml`：

```yaml
server:
  host: 0.0.0.0
  port: 8080

model:
  name: meta-llama/Llama-2-7b-chat-hf
  precision: int8

optimization:
  max_batch_size: 16
  tensor_parallel_size: 1
  kv_cache_policy: dynamic
  enable_prefix_cache: true

observability:
  prometheus_enabled: true
  log_level: info
```

使用配置启动：

```bash
soloai-llm-opt serve --config config.yaml
```

### 3. 导出性能分析报告

```bash
soloai-llm-opt profile \
  --endpoint http://localhost:8080 \
  --duration 60 \
  --concurrency 32 \
  --report report.json
```

### 4. Python SDK 调用示例

```python
from soloai_llm_opt import Client

client = Client(base_url="http://localhost:8080")

response = client.chat.completions.create(
    model="meta-llama/Llama-2-7b-chat-hf",
    messages=[
        {"role": "user", "content": "给我总结一下提升 LLM 推理吞吐的关键方法"}
    ],
    temperature=0.5,
    max_tokens=200,
)

print(response.choices[0].message["content"])
```
