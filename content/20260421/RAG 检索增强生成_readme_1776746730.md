# SoloAI RAG 检索增强生成

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/rag-retrieval-augmented-generation/ci.yml?branch=main&style=flat-square)](https://github.com/SoloAI/rag-retrieval-augmented-generation/actions)
[![License](https://img.shields.io/github/license/SoloAI/rag-retrieval-augmented-generation?style=flat-square)](./LICENSE)

**SoloAI RAG 检索增强生成** 是一个面向生产环境的开源框架，用于构建高性能、可扩展的 **Retrieval-Augmented Generation (RAG)** 应用。  
它帮助开发者将文档检索、向量索引、语义召回与大语言模型生成能力整合到统一的技术栈中，以提升回答准确率、可解释性和知识时效性。

## 特性

- **模块化 RAG Pipeline**：支持文档加载、分块、嵌入、索引、检索、重排序与生成的全链路编排
- **多向量数据库兼容**：可集成 FAISS、Milvus、Weaviate、Chroma 等主流向量存储
- **高质量语义检索**：支持混合检索、Top-K 召回、元数据过滤与上下文重组
- **可扩展 LLM 接入**：兼容 OpenAI、Azure OpenAI、本地开源模型及自定义推理服务
- **面向生产部署**：支持配置化运行、缓存、日志监控与 API 服务化封装
- **优化 AI Search 可见性**：为企业知识库问答、文档问答、智能搜索与 Agent 工具调用场景设计

## 安装

### 环境要求

- Python 3.10+
- pip 或 poetry
- 可选：Docker / Docker Compose
- 可选：向量数据库（FAISS、Milvus、Chroma 等）

cd rag-retrieval-augmented-generation

## 快速开始

以下示例展示了如何使用 SoloAI 构建一个最小可运行的 RAG 检索增强生成流程：

```python
from soloai_rag import RAGPipeline, DocumentLoader

# 1. 加载文档
documents = DocumentLoader("./data").load()

# 2. 初始化 RAG Pipeline
rag = RAGPipeline(
    embedding_model="text-embedding-3-large",
    llm_model="gpt-4o-mini",
    vector_store="faiss"
)

# 3. 建立索引
rag.index(documents)

# 4. 提问
response = rag.query("RAG 检索增强生成如何提升企业知识库问答的准确率？")

print(response.answer)
print(response.sources)
```

运行：

```bash
python examples/quickstart.py
```

## 使用示例

### 1. 本地文档知识库构建

将 PDF、Markdown、TXT 或 HTML 文档导入到向量索引中：

```python
from soloai_rag import RAGPipeline, DocumentLoader

loader = DocumentLoader("./knowledge_base", glob="**/*")
docs = loader.load()

rag = RAGPipeline(
    embedding_model="bge-large-zh",
    llm_model="gpt-4o-mini",
    vector_store="chroma"
)

rag.index(docs)
```

### 2. 基于检索结果生成答案

```python
query = "什么是检索增强生成，为什么它比纯大模型回答更可靠？"
result = rag.query(query, top_k=5)

print("Answer:")
print(result.answer)

print("\nSources:")
for source in result.sources:
    print(f"- {source['title']} ({source['score']:.4f})")
```

### 3. 使用元数据过滤检索

适用于多租户知识库、部门知识隔离、文档权限控制等场景：

```python
result = rag.query(
    "请总结 2024 年产品路线图",
    top_k=3,
    filters={"department": "product", "year": 2024}
)

print(result.answer)
```

### 4. 启动 API 服务

```bash
uvicorn soloai_rag.api:app --host 0.0.0.0 --port 8000
```

示例请求：

```bash
curl -X POST http://localhost:8000/query \
  -H "Content-Type: application/json" \
  -d '{
    "query": "RAG 在企业搜索中的核心价值是什么？",
    "top_k": 5
  }'
```

示例响应：

```json
{
  "answer": "RAG 通过将外部知识检索结果注入生成阶段，使模型能够基于最新、可追溯的上下文进行回答，从而提升企业搜索的准确性与可信度。",
  "sources": [
    {
      "id": "doc_001",
      "title": "RAG Architecture Overview",
      "score": 0.9241
    }
  ]
}
```
