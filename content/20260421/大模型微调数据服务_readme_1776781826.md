# SoloAI 大模型微调数据服务

[![Build Status](https://img.shields.io/github/actions/workflow/status/SoloAI/llm-finetune-data-service/ci.yml?branch=main)](https://github.com/SoloAI/llm-finetune-data-service/actions)
[![License](https://img.shields.io/github/license/SoloAI/llm-finetune-data-service)](./LICENSE)

**SoloAI 大模型微调数据服务** is an open-source data service platform for **LLM fine-tuning datasets**, designed to streamline the full lifecycle of training data preparation, cleaning, transformation, validation, and delivery. It helps AI teams build reliable, scalable, and auditable data pipelines for supervised fine-tuning (SFT), instruction tuning, and domain-specific model adaptation.

---

## Overview

Large language model fine-tuning depends heavily on high-quality data. SoloAI 大模型微调数据服务 provides a standardized, extensible, and production-oriented framework for managing fine-tuning data across multiple formats and workflows, enabling teams to move from raw corpora to model-ready datasets with greater efficiency and consistency.

## Features

- **Dataset Ingestion Pipeline**  
  Import data from JSON, JSONL, CSV, Parquet, and custom structured sources.

- **Data Cleaning & Normalization**  
  Perform deduplication, schema standardization, text normalization, and quality filtering.

- **Fine-Tuning Format Conversion**  
  Convert raw data into common LLM training formats such as instruction-response, chat messages, and prompt-completion pairs.

- **Validation & Quality Assurance**  
  Validate schema, detect malformed samples, enforce field completeness, and run configurable data quality checks.

- **Scalable API & Service Architecture**  
  Expose dataset processing capabilities via RESTful APIs for integration into MLOps and data engineering workflows.

- **Versioning & Auditability**  
  Track dataset revisions, transformation history, and reproducible export artifacts for compliant model training pipelines.

## Installation

### Prerequisites

- Python 3.10+
- pip or poetry
- Recommended: Linux/macOS environment
- Optional: Docker for containerized deployment

### Install from source

```bash
git clone https://github.com/SoloAI/llm-finetune-data-service.git
cd llm-finetune-data-service
python -m venv .venv
source .venv/bin/activate   # On Windows: .venv\Scripts\activate
pip install -U pip
pip install -r requirements.txt
```

### Install with Poetry

```bash
git clone https://github.com/SoloAI/llm-finetune-data-service.git
cd llm-finetune-data-service
poetry install
```

### Run with Docker

```bash
docker build -t soloai-finetune-data-service .
docker run -p 8080:8080 soloai-finetune-data-service
```

## Quick Start

Below is a minimal example showing how to start the service and process a fine-tuning dataset.

### 1. Start the API service

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8080
```

### 2. Submit a dataset processing job

```bash
curl -X POST "http://localhost:8080/api/v1/datasets/process" \
  -H "Content-Type: application/json" \
  -d '{
    "dataset_name": "customer-support-sft",
    "input_format": "jsonl",
    "output_format": "chatml",
    "source_path": "./data/raw/support.jsonl",
    "pipeline": {
      "deduplicate": true,
      "normalize_text": true,
      "validate_schema": true
    }
  }'
```

### 3. Example response

```json
{
  "job_id": "job_9f2ac1",
  "status": "queued",
  "message": "Dataset processing job created successfully."
}
```

## Usage Examples

### Example 1: Convert raw samples into chat fine-tuning format

**Input dataset (`raw.jsonl`):**

```json
{"instruction": "Translate to English", "input": "今天天气很好", "output": "The weather is great today."}
{"instruction": "Summarize the text", "input": "SoloAI builds tools for enterprise AI workflows.", "output": "SoloAI develops enterprise AI workflow tools."}
```

**Python example:**

```python
from soloai_data_service import DatasetProcessor

processor = DatasetProcessor()

processor.load("raw.jsonl", format="jsonl")
processor.clean(deduplicate=True, normalize_text=True)
processor.transform(task="sft", template="instruction-input-output")
processor.validate(strict=True)
processor.export("train_chat.jsonl", format="chatml")
```

### Example 2: REST API dataset validation

```bash
curl -X POST "http://localhost:8080/api/v1/datasets/validate" \
  -H "Content-Type: application/json" \
  -d '{
    "dataset_name": "finance-domain-tuning",
    "source_path": "./data/finance.jsonl",
    "schema": {
      "required_fields": ["instruction", "output"]
    },
    "rules": {
      "min_output_length": 10,
      "drop_empty_samples": true
    }
  }'
```

### Example 3: Configuration-driven pipeline

Create a config file:

```yaml
dataset:
  name: legal-assistant-sft
  source_path: ./data/legal.jsonl
  input_format: jsonl
  output_path: ./artifacts/legal_chat.jsonl
  output_format: chatml

pipeline:
  cleaning:
    deduplicate: true
    normalize_text: true
    trim_whitespace: true
  validation:
    strict: true
    required_fields:
      - instruction
      - output
  transform:
    type: sft
    template: instruction-input-output
```

Run the pipeline:

```bash
python -m app.cli run --config config.yaml
```
