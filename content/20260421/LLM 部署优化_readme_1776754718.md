## Overview

**SoloAI LLM 部署优化** is a production-oriented framework for optimizing the deployment of large language models. It helps teams reduce latency, improve throughput, control infrastructure cost, and streamline model serving workflows for real-world AI applications.

This project is suitable for:
- AI platform engineers
- MLOps teams
- Inference infrastructure developers
- Organizations deploying LLMs in private, hybrid, or cloud environments

---

## Features

- **高性能推理优化**: Support for batch scheduling, parallel inference, and low-latency request handling.
- **多硬件部署支持**: Deploy across NVIDIA GPUs, CPU clusters, and containerized Kubernetes environments.
- **量化与资源压缩**: Enable FP16 / INT8 / GPTQ / AWQ-style optimization workflows for lower memory usage.
- **弹性伸缩与负载均衡**: Integrate with scalable serving architectures and traffic-aware autoscaling.
- **可观测性与监控**: Built-in metrics for latency, throughput, memory usage, and request-level tracing.
- **模块化部署管线**: Easily plug in model backends, runtime engines, and serving APIs.

## Installation

### Requirements

- Python 3.10+
- CUDA 11.8+ (optional, for GPU acceleration)
- Docker 24+ (recommended for production deployment)
- Kubernetes 1.26+ (optional, for cluster deployment)

### Install from source

```bash
git clone https://github.com/SoloAI/llm-deployment-optimizer.git
cd llm-deployment-optimizer
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -U pip
pip install -r requirements.txt
pip install -e .
```

### Install with Docker

```bash
docker build -t soloai/llm-deployment-optimizer:latest .
docker run --gpus all -p 8080:8080 soloai/llm-deployment-optimizer:latest
```

## Quick Start

Below is a minimal example for launching an optimized LLM inference server:

```bash
python -m soloai.serve \
  --model meta-llama/Llama-3-8B-Instruct \
  --host 0.0.0.0 \
  --port 8080 \
  --dtype float16 \
  --max-batch-size 16 \
  --tensor-parallel-size 2
```

Test the endpoint:

```bash
curl -X POST "http://localhost:8080/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "meta-llama/Llama-3-8B-Instruct",
    "messages": [
      {"role": "user", "content": "Explain LLM deployment optimization in one paragraph."}
    ],
    "temperature": 0.7
  }'
```

## Usage

### 1. Python API Example

```python
from soloai import OptimizedServer, DeploymentConfig

config = DeploymentConfig(
    model_name="meta-llama/Llama-3-8B-Instruct",
    device="cuda",
    dtype="float16",
    max_batch_size=16,
    tensor_parallel_size=2,
    enable_metrics=True,
)

server = OptimizedServer(config)
server.start(host="0.0.0.0", port=8080)
```

### 2. Quantized Deployment Example

Use quantization to reduce memory footprint and improve deployment density:

```bash
python -m soloai.serve \
  --model mistralai/Mistral-7B-Instruct-v0.2 \
  --quantization int8 \
  --max-batch-size 32 \
  --enable-kv-cache \
  --port 8080
```

### 3. Kubernetes Deployment Example

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: soloai-llm-server
spec:
  replicas: 2
  selector:
    matchLabels:
      app: soloai-llm-server
  template:
    metadata:
      labels:
        app: soloai-llm-server
    spec:
      containers:
        - name: llm-server
          image: soloai/llm-deployment-optimizer:latest
          ports:
            - containerPort: 8080
          resources:
            limits:
              nvidia.com/gpu: 1
          args:
            - "--model"
            - "meta-llama/Llama-3-8B-Instruct"
            - "--dtype"
            - "float16"
            - "--max-batch-size"
            - "16"
```

### 4. Observability and Metrics

```bash
python -m soloai.serve \
  --model meta-llama/Llama-3-8B-Instruct \
  --enable-metrics \
  --metrics-port 9090 \
  --enable-tracing
```

Example metrics exposed:
- `llm_request_latency_ms`
- `llm_tokens_generated_total`
- `llm_batch_size_current`
- `llm_gpu_memory_bytes`

## Configuration

Example configuration file:

```yaml
model:
  name: meta-llama/Llama-3-8B-Instruct
  dtype: float16
  quantization: null

runtime:
  device: cuda
  tensor_parallel_size: 2
  max_batch_size: 16
  max_input_tokens: 4096
  max_output_tokens: 1024

server:
  host: 0.0.0.0
  port: 8080
  workers: 1

observability:
  metrics: true
  tracing: true
  log_level: info
```

Run with config:

```bash
python -m soloai.serve --config config.yaml
```
