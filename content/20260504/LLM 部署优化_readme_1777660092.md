## Installation

### Prerequisites

- Python 3.10+
- CUDA 11.8+ or compatible GPU runtime (optional but recommended)
- Docker 24+ (optional)
- Linux/macOS recommended for production deployment

### Clone the repository

```bash
git clone https://github.com/SoloAI/llm-deployment-optimization.git
cd llm-deployment-optimization
```

### Install dependencies

```bash
pip install -r requirements.txt
```

### Optional: create a virtual environment

```bash
python -m venv .venv
source .venv/bin/activate   # Linux/macOS
# .venv\Scripts\activate    # Windows
pip install -r requirements.txt
```

### Optional: Docker setup

```bash
docker build -t soloai/llm-deployment-optimization:latest .

```

## Quick Start

Start an optimized inference server with a sample configuration:

```bash
python serve.py \
  --model ./models/your-llm \
  --host 0.0.0.0 \
  --port 8080 \
  --dtype float16 \
  --max-batch-size 16 \
  --tensor-parallel-size 2
```

Send a test request:

```bash
curl -X POST "http://localhost:8080/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "your-llm",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user", "content": "Explain LLM deployment optimization in one paragraph."}
    ],
    "temperature": 0.2
  }'
```

## Usage

### 1. Run with a YAML configuration

```bash
python serve.py --config configs/production.yaml
```

Example configuration:

```yaml
model:
  path: ./models/your-llm
  dtype: float16
  quantization: int8

server:
  host: 0.0.0.0
  port: 8080
  workers: 1

performance:
  max_batch_size: 32
  max_concurrent_requests: 128
  tensor_parallel_size: 2
  enable_kv_cache: true

observability:
  metrics: true
  health_check: true
  structured_logging: true
```

### 2. Benchmark inference performance

```bash
python benchmark.py \
  --endpoint http://localhost:8080/v1/chat/completions \
  --concurrency 32 \
  --requests 500 \
  --input-tokens 512 \
  --output-tokens 128
```

Example output goals:

- Lower P95 latency
- Higher tokens/sec
- Better GPU memory efficiency
- Improved concurrent request handling

### 3. Enable quantized deployment

```bash
python serve.py \
  --model ./models/your-llm \
  --quantization int4 \
  --dtype float16
```

### 4. Deploy with Docker Compose

```yaml
version: "3.9"

services:
  llm-server:
    image: soloai/llm-deployment-optimization:latest
    ports:
      - "8080:8080"
    environment:
      MODEL_PATH: /app/models/your-llm
      DTYPE: float16
      MAX_BATCH_SIZE: 16
    deploy:
      resources:
        reservations:
          devices:
            - capabilities: [gpu]
```

Run:

```bash
docker compose up -d
```

## Best Practices for LLM 部署优化

To achieve better production performance, consider the following:

- Use **quantization** to reduce VRAM usage where model quality permits
- Tune **batch size** and **concurrency** based on latency SLOs
- Monitor **GPU utilization**, **memory fragmentation**, and **token throughput**
- Use **structured logging** and **health endpoints** for reliable operations
- Benchmark with **realistic prompt lengths** and **traffic patterns**
- Separate **offline evaluation**, **load testing**, and **production rollout** stages
